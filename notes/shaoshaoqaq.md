**Week 1 Build Log —— 从钱包创建到 Monad 链上实践**

**1\. Week 1 学习目标**

本周主要目标是建立进入 Web3 世界的基础能力，包括：

-   理解区块链基础架构与核心概念；
    
-   创建并管理自己的链上身份；
    
-   完成第一次链上交互；
    
-   理解交易、智能合约和链上应用的基本运行方式；
    
-   探索 AI 在 Web3 学习和开发过程中的辅助作用。
    

**2\. 链上实践记录（Build）**

**2.1 创建课程专用钱包**

**完成内容**

-   使用 MetaMask 创建课程专用钱包；
    
-   学习助记词、私钥、公钥、钱包地址之间的关系；
    
-   建立安全意识：
    
    -   不使用主力钱包参与测试；
        
    -   不泄露助记词、私钥；
        
    -   区分公开信息（地址、交易 Hash）和敏感信息（助记词、私钥）。
        

**收获**

第一次认识到：

钱包不是传统互联网中的账号，而是用户进入链上世界的身份入口。

在 Web3 中，用户通过钱包控制资产和参与协议，而不是依赖中心化平台创建账户。

**2.2 配置 Monad Testnet**

**完成内容**

-   在 MetaMask 中添加 Monad Testnet 网络；
    
-   学习 EVM 网络配置方式；
    
-   理解：
    
    -   RPC Endpoint
        
    -   Chain ID
        
    -   Block Explorer
        

等基础概念。

**理解变化**

之前认为“切换网络”只是钱包里的一个设置。

实践后理解：

不同区块链网络实际上是不同的状态环境，钱包只是连接这些网络的工具。

**2.3 完成第一笔 Monad Testnet 链上交易**

**完成内容**

-   获取测试网资产；
    
-   完成第一次 Testnet 转账；
    
-   使用区块浏览器查询交易状态；
    
-   查看：
    
    -   Transaction Hash
        
    -   Block
        
    -   From
        
    -   To
        
    -   Gas Fee
        
    -   Transaction Status
        

**对交易机制的新理解**

交易并不是简单的“发送数字货币”，而是：

1.  用户通过钱包生成交易；
    
2.  钱包进行签名；
    
3.  节点接收交易；
    
4.  网络验证交易；
    
5.  交易被打包进入区块；
    
6.  区块状态同步到整个网络。
    

**2.4 Solidity 合约部署实践**

**完成内容**

在 AI 辅助下：

-   学习 Solidity 基础语法；
    
-   编写简单智能合约；
    
-   部署 Solidity 合约；
    
-   理解合约地址和交易调用关系。
    

**初步理解**

智能合约不是传统服务器程序，而是：

部署在区块链上的公开、可验证、按照规则自动执行的程序。

**3\. 遇到的问题与解决过程**

**问题 1：MetaMask 钱包创建流程不熟悉**

**遇到问题**

第一次操作时进入了“导入钱包”页面，不知道如何创建新钱包。

**解决方式**

通过 AI 辅助理解：

-   Import Wallet ≠ Create Wallet
    
-   助记词用于恢复钱包
    
-   创建钱包会自动生成助记词
    

最终完成课程专用钱包创建。

**问题 2：不知道如何理解链上交易**

**遇到问题**

刚开始只能看到交易完成，但不知道：

-   为什么需要 Gas；
    
-   为什么需要区块确认；
    
-   Transaction Hash 有什么意义。
    

**解决方式**

通过 AI 解释交易生命周期，并结合 Explorer 查看真实交易数据。

**问题 3：智能合约部署门槛较高**

**遇到问题**

Solidity 语法、部署流程、合约调用方式不熟悉。

**解决方式**

使用 AI：

-   解释代码逻辑；
    
-   辅助修改错误；
    
-   帮助理解部署步骤。
    

同时自己验证：

-   合约代码是否符合需求；
    
-   部署结果是否正确；
    
-   链上状态是否变化。

**5\. AI 不能替代的人工判断**

虽然 AI 提高了学习效率，但以下部分仍需要自己判断：

**安全判断**

例如：

-   是否可信网站；

-   是否泄露敏感信息。
    

**产品理解**

AI 可以解释概念，但需要自己判断：

-   一个协议解决什么问题；
    
-   用户为什么需要它；

**技术路线选择**

-   学 Solidity 还是数据分析；
    
-   进入开发、运营还是研究方向。
    

**6\. 对 Monad 和 Web3 的理解变化**

**Week 1 前**

对 Web3 的理解主要停留在：

-   加密货币；
    
-   NFT；
    
-   区块链投资。
    

**Week 1 后**

理解 Web3 更像是一套新的互联网基础设施：

-   区块链提供可信状态记录；
    
-   智能合约提供自动执行能力；
    
-   钱包成为用户身份入口；
    
-   应用可以直接建立在开放协议之上。
    

Monad 作为高性能 EVM 公链，其价值不仅是交易速度，而是在探索：

如何让开发者在保持 EVM 兼容性的同时，实现更高性能的链上应用。
