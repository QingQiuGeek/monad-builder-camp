### AI 智能体支付基础设施演进与商业潜力分析：以 FluxA 协议为例

**摘要**

随着 AI 智能体（AI Agent）从被动的聊天机器人演变为具备规划、推理、使用工具和执行多步任务能力的自主经济参与者，传统的金融支付网络已无法满足其需求。构建原生的 AI 支付基础设施，使智能体能够安全、自主地进行价值流转，正成为行业研究与投资布局的核心方向。本报告基于 Jack Xiao 的分享与 FluxA 项目资料，探讨 Agent Commerce 的支付痛点、风控架构及未来生态场景，并结合前沿公链与商业演进趋势提出投研感悟。

* * *

一、 行业痛点：传统商业支付与智能体经济的错配

智能体的商业逻辑并非传统的“一手交钱一手交货”，而是为了完成“意图发现（Intention Discovery）”所进行的一系列自动化调用与操作。这一特征使得传统支付体系在面对智能体时暴露出显著的短板：

-   **身份验证机制失效**：传统金融体系的 KYC（了解你的客户）依赖于人类的法定身份证件，而智能体缺乏合法的独立身份，无法直接开立银行账户。
    
-   **授权与安全性冲突**：传统支付需要人类手动点击和签名确认，这违背了智能体自主执行的初衷。此外，将敏感的银行卡信息作为明文输入到大模型的 Prompt 框中，存在极高的安全隐患。
    
-   **交易颗粒度与结算效率错位**：智能体调用 API 或获取数据通常涉及极小额的微支付（低至 0.001 美元），而传统信用卡网络的最低支付门槛通常在 0.50 美元以上，甚至连手续费都无法覆盖。同时，T+1 到 T+3 的传统清算速度完全无法适应智能体毫秒级的响应要求。
    

二、 FluxA 架构设计与底层风控模型

为解决上述痛点，由前阿里与蚂蚁集团高管打造的 FluxA 协议提出了一套专为智能体构建的底层支付网络。从机制设计与金融风控的角度来看，其核心架构包含以下几个关键模块：

-   **身份验证（Identity）**：为智能体分配基于区块链地址的独立身份，并利用 USDC 稳定币和 Base 链构建底层转账与结算链路。
    
-   **预算与风控策略（Budget & Policy）**：在金融场景中，防范大模型“幻觉”导致的资金滥用是核心风控诉求。协议允许设置严格的约束条件（Intent-based Policy），例如签发具有锁定预算上限和特定用途的单次有效智能体卡（Agent Card）。
    
-   **嵌入式授权与延迟结算（Embedded Mandate & Deferred Settlement）**：智能体在获得一笔预算授权后，可在限额内自由执行多步微交易操作，无需人类逐笔审批。交易完成后，系统会在底层链上进行集中确认与延迟结算，大幅提升了微交易场景下的处理效率和审计透明度。
    
-   **可撤销机制**：确保在交易出现异常或意图不符时，人类共管账户可以直接一键撤销交易，形成最终的风险兜底。
    

三、 商业落地模式与生态应用场景

FluxA 致力于成为智能体经济的金融引擎，实现无需将人类用户暴露于金融风险之下的自动收付款。其商业落地高度契合 Web3 的可组合性，主要覆盖以下四类场景：

-   **A2A (Agent-to-Agent)**：智能体之间为执行任务、数据交换或协作支付报酬。
    
-   **A2T (Agent-to-Tool)**：针对 API 和 MCP 工具提供按需付费（Pay-per-use）的微支付能力，免除长期订阅费用。配套的 Monetize 工具使得独立开发者能够轻松将自有服务货币化，完成商业闭环。
    
-   **A2M (Agent-to-Merchant)**：支持智能体自主预订航班、酒店以及 SaaS 订阅服务，未来还包括缴纳水电煤等生活化场景。
    
-   **Social (社交转账与打赏)**：支持智能体大规模分发社交红包及小费打赏。
    

四、 市场数据与行业展望

当前，FluxA 已在首波智能体商业化浪潮中取得了显著的市场牵引力。

-   **运营数据**：平台已注册的智能体钱包数量超过 110K+，日均峰值交易笔数达到 30K+，并将支付延迟大幅压缩至 0.28 秒（较传统体系提升 15 倍）。Jack Xiao 在直播中更透露目前注册数量已突破 13 万。
    
-   **生态建设**：目前已深度整合了传统金融体系与去中心化网络，生态合作伙伴包括 Coinbase、蚂蚁集团、MoonPay 等机构，以及 Claude、Qwen、Cursor 等开发者工具。
    
-   **研发趋势**：AI 支付赛道的兴起要求从业者不仅需要理解开发流程，更需要具备跨界的经济模型设计与风控架构能力。
    

五、 个人感悟与未来研究方向

-   **投资逻辑重构：从“人本金融”走向“机器经济”**
    
    AI Agent 支付基础设施不仅是工具链路的延伸，更是全新商业范式的开端。传统金融风控与交易网络的核心信任主体是“人”和“主体机构”，而在 Agent Commerce 时代，信任主体将不可逆转地向协议、代码与去中心化网络转移。FluxA 剥离了复杂的全栈开发门槛，通过结合稳定币（USDC）与去中心化账本，填补了这一巨大的价值流转空白。这不仅是 AI 赛道的应用拓展，更是 Web3 支付真正走向大规模落地（Mass Adoption）的破局点，为后续的行业研究与创投布局提供了核心锚点。
    
-   **基础设施演进：公链性能与高频微交易的化学反应**
    
    为了满足毫秒级确认和极低手续费，当前诸如 FluxA 的微支付大多选择跑在 Base 等 Layer 2 网络上。但展望未来，如果这类超高并发的 Agent 支付协议能够结合具备极高 TPS 与并行执行能力的高性能 Layer 1（例如 Monad），智能体之间的超高频数据交互与微交易容量将获得指数级突破。在深度参与高性能公链生态的研究实习中，金融架构、资本效率与去中心化程度的平衡，将是发掘下一代优质协议的关键。
    
-   **认知壁垒与行业影响力构建**
    
    在这个信息高度密集的周期，掌握 Web3 与 AI 融合的顶层商业逻辑具有极高的杠杆价值。将这类前沿协议的硬核机制，拆解提炼为兼具专业深度与传播度的投研观点并持续对外输出，是建立行业话语权的绝佳切入点。高质量的投研内容不仅能沉淀专业的分析框架，更有望在社交平台上形成强有力的辐射效应，吸引更多资深建设者与行业头部力量的深度互动。

# 以太坊协议层入门路径与开源贡献策略：基于 EPF 体系的非技术视角研究

## 摘要 (Abstract)

本文基于 JackCC 老师关于以太坊协议层的分享，系统梳理了普通开发者及投研人员进入以太坊底层的学习路径。有别于应用层（DApp/智能合约）的构建，协议层侧重于以太坊自身的运行机制（执行层、共识层、密码学等）。本文结合高频输出、社交媒体 IP 打造以及 GitHub 社区协作，提出了一套适用于非技术背景与研究型导向的 EPF参与模型。

* * *

## 一、 引言：跳出应用层，探究系统的“底层基本面”

在过去参与 DeFi 交互、二级市场交易以及空投生态的过程中，我的注意力主要集中在应用层（DApp）的商业模式上。然而，要真正深入 Web3，产出高质量的行业研究，必须理解以太坊这座“超级工厂”本身的运作规律。

相比于应用层，协议层的门槛确实更高，它包含了大量的规范、密码学和共识机制。但 EPF 的核心理念说明，参与协议层并非只面向“天才模式”，而是需要参与者具备“会学习、会复现、会提问”的能力。这与金融行业看重的数据拆解和研报撰写能力本质上是相通的。

## 二、 核心方法论：先建地图，再找入口

对于非技术人员，直接阅读底层代码如同没有财报基础就去盲看企业的流水账，很容易迷失。EPF 提倡的学习路径是建立在自驱动基础上的，核心在于“先补协议基础，理解自己想研究的问题”。

1.  **构建全景地图：** 第一阶段的目标不是背诵术语，而是弄清楚系统是如何运转的。必须先理解执行层（EL，涉及 EVM 和状态）和共识层（CL，涉及验证者、最终确定性）是如何协同工作的。
    
2.  **摸清交易流程：** 需要在宏观上理解一笔交易是如何进入区块、节点是如何同步和验证状态的。
    
3.  **寻找切入点：** 在了解整体架构后，再去评估哪个代码仓库与自己的研究方向相关，哪些问题是自己能够复现和理解的。对于习惯了分析行业基本面的研究者来说，可以借助 AI 辅助，将枯燥的英文规范（Specs）翻译成易懂的业务逻辑，用中文整理笔记后再补充英文术语。
    

## 三、 行动框架：新手的“四周启动路线”与微小贡献法则

为了避免陷入“总觉得自己还没准备好”的焦虑，EPF 框架提供了一个非常务实的“四周启动路线”，要求新手连续四周产出可见的证据：

-   **第一周（画地图）：** 利用 EPF Wiki 建立基本概念图，搞懂节点、交易等基础知识。
    
-   **第二周（跑起来）：** 尝试运行一个本地工具或轻量测试网，重点是记录下运行的命令和遇到的报错问题。
    
-   **第三周（读进去）：** 挑选一个具体的开源仓库，仔细阅读一个 Issue 或一小段规范。
    
-   **第四周（公开输出）：** 撰写一篇学习笔记，或者尝试提交一个小小的代码/文档修改（PR）。
    

在这个过程中，最重要的一点是打破“贡献必须是写出几百行核心代码”的迷思。在真正的开源协作中，整理文档、修复 Wiki 错误、补充中文翻译、甚至是把测试报错的环境写清楚，都属于极具价值的“微小贡献”。此外，在选择贡献方向时，应该选择一个“问题域”（比如数据证明或客户端的某个模块），而不是仅仅盯着某一个随时可能被别人修好的孤立 Issue。

## 4\. EPF 框架下的项目交付与贡献策略

EPF本质上是一个为期约五个月的项目制培养计划，而非标准化考试。

### 4.1 EPF 全周期阶段拆解

| 阶段 | 核心任务 | 策略侧重点 |
| 第一阶段 (准备期) | 熟悉协议知识与生态环境 | 建立基础认知框架，利用 AI 扫盲核心概念。 |
| 第二阶段 (深入期) | 深入阅读规范，参与社区讨论 | 在 GitHub 的 Issue/Discussion 中提出有价值的疑问。 |
| 第三阶段 (贡献期) | 围绕特定方向进行实质性输出 | 结合自身优势（如架构梳理、风险评估）进行研究输出。 |
| 第四阶段 (交付期) | 完成最终项目交付 | 交付一份高质量、逻辑严密的特定方向研究/贡献成果。 |

### 4.2 非技术视角的“微小贡献 (Micro-Contributions)”法则

不要陷入“必须写出复杂代码才能贡献”的误区。开源社区高度重视协作与审查成本：

-   **放弃“大活”执念：** 庞大的修改会带来极高的 Review 成本，容易被维护者搁置。
    
-   **从小处着手：** 优先修复文档错误、优化配置、修正拼写，或对复杂规范补充通俗易懂的逻辑说明。
    
-   **选择熟悉的路径：** 避开底层难度最高的客户端底层代码，选择进入门槛较低、更容易产生正反馈的规范层（Specs）文档或相关测试验证环节。通过微小但持续的 PR（Pull Request），快速走通社区协作的全流程。
    

* * *

## 结论 (Conclusion)

进入以太坊协议层并非纯粹代码极客的专属特权。通过 AI 辅助跨越技术鸿沟，将 GitHub 转化为高价值的一手投研信息源，并配合以周为单位的社交媒体公开输出，最终在 EPF 的项目制框架下从小微贡献切入，这是一套高度可行、且能最大化发挥非技术与投研分析优势的开源参与范式。

# 从生态连接者到公链交互体验的投研观察

## 一、 从 Builder 到生态连接者：DevRel 的核心商业价值

结合嘉宾 XiaoHai 的个人履历分享，可以清晰地看到一条典型的 Web3 建设者进阶路径：从底层的智能合约/前端开发工程师（[Crust.Network](http://Crust.Network), [Aro.Network](http://Aro.Network)），逐步走向技术治理研究员（LXDAO），最终成长为统筹大局的 DevRel 负责人

**投研视角的关键启发：**

-   **DevRel 是生态冷启动的胜负手：** 在评估一条新公链或 Web3 协议时，不能仅仅停留在 TPS、出块时间等白皮书数据上。DevRel（开发者关系）团队的软实力，即他们对外输出价值、降低技术门槛、连接不同开发者社区的能力，是项目能否在早期快速积累真实活跃开发者（Active Builders）的关键护城河。
    
-   **估值模型的新维度：** 繁荣的开发者生态是应用落地的基础，而丰富的应用直接决定了链上 TVL（总锁仓价值）的沉淀。因此，将项目方的 DevRel 战略与社区运营数据纳入核心基本面评估框架，对于精准捕捉早期 Alpha 资产具有重要指导意义。
    

## 二、 Monad 测试网交互复盘：开发者体验（DX）的降维打击

在日前的 Co-learning 环节中，完整观摩并梳理了 Monad 测试网的交互全流程，包括：网络配置、测试币申领、使用 Remix IDE 编写 `MessageBoard` 智能合约，以及连接 MetaMask 钱包完成最终部署。

**投研视角的关键启发：**

-   **零沉没成本迁移：** 整个交互过程最大的亮点在于其“无缝顺滑”。Monad 实现了对现有 EVM 工具链（如 MetaMask, Remix）的完全兼容。这意味着以太坊生态的成熟项目方和开发者，不需要学习新的编程语言或更换熟悉的开发环境，就能直接迁移到 Monad 上。
    
-   **商业潜力验证：** 极低的迁移门槛（Low Migration Cost）配合其底层极高的吞吐量预期。从行业研究的角度看，这种策略能够最大化地吸收外溢的 EVM 开发者与资金，极大地缩短了生态爆发的酝酿期。
    

## 三、 总结与下一步行动

这两场分享虽然涵盖了软性的社区运营与硬核的链上部署，但底层逻辑是统一的：**技术的成功最终取决于“人”的采用。** 优秀的 DevRel 团队负责吸引人，而极致兼容的底层架构（如 Monad）负责留住人。

**Next Steps：**

1.  **完善研究框架：** 在后续撰写研报时，正式将“开发者体验（Developer Experience）”与“社区运营/DevRel 活跃度”作为评估 Layer1/Layer2 项目的重要基本面指标。
    
2.  **个人 IP 沉淀：** 借鉴 DevRel 对外输出价值的思路，将本次观察到的“公链生态冷启动逻辑”提炼为短篇投研观点，尝试在社交平台上进行内容输出，建立自己的行业连接。

# 探索下一代高吞吐量 EVM：Monad 架构创新与商业潜力探析

**—基于 Web3 暑期实习计划开营分享Monad 101**

**摘要：** 随着稳定币、DEX 及预测市场等链上业务的爆发式增长，底层区块链基础设施的容量瓶颈日益凸显。本文基于 Monad DevRel 的开营分享内容，系统性梳理了 Monad 的核心技术愿景与架构创新。作为新兴的高吞吐量 EVM 兼容链，Monad 通过底层软硬件架构重构实现了 10,000 TPS 与 0.4 秒的出块时间，同时保持了去中心化特性。本文重点探讨了 Monad 如何在承载高频 DeFi 与实时链上交互方面打破现有公链的“扩容悖论”，为二级市场交易策略升级与 Web3 大规模应用提供坚实的底层支撑。

## 一、 行业痛点：需求跑赢容量，基础设施亟待破局

当前链上活动正处于爆发式增长阶段。从数据面来看，无论是稳定币的供应量、DEX 在现货市场的交易份额，还是预测市场的交易量以及机器对机器（M2M）的支付频次，各项核心指标的曲线均呈现出强劲的上扬趋势。然而，核心矛盾在于：**底层的链能否跟得上上层应用的扩张速度？**

回顾历史的压力测试，需求始终在跑赢容量：2017 年的 CryptoKitties 曾让以太坊陷入严重拥堵；DeFi Summer 期间的流动性挖矿引发了 Gas 费用的非理性飙升；随后 NFT 铸造与 Memecoin 的狂热交易更是屡次让链上网络几近瘫痪。正如分享中提到的核心观点：**“每一个新场景都撑爆了它所在的链。瓶颈不在于新应用，而在于为我们已有的应用提供基础设施。”** 面向未来，市场迫切需要能够支撑 Visa 级速度与成本的支付通道、面向实时交易的高保真 DeFi、全链游戏以及规模化的 AI 链上计算。

## 二、 性能突破：数据与架构双视角的 Monad 洞察

Monad 建立的核心愿景在于实现“可扩展性、安全性与去中心化”的有机统一，其主张“抬高上限，而不抬高门槛”。

### 1\. 核心性能数据对比

在保持完全兼容 EVM 的前提下，Monad 实现了两个数量级的容量提升。以下是其与 Ethereum L1 的核心指标对比：

| 指标 (Metric) | Monad | Ethereum L1 |
| 吞吐量 (Throughput) | 500M gas / sec | 5M gas / sec |
| 出块时间 (Block Time) | 0.4 s | 12 s |
| 持续 TPS | 10,000 | ~28 |
| 活跃验证者 | 200 | ~27,000 / block |

### 2\. 性能源于架构重构

Monad 的性能飞跃并非依赖昂贵的超级计算机，而是将普通商用硬件的性能“榨到极致”。其采用了从零开始的全新软件架构，并进行了大量的底层数据链路优化。在节点运行方面，Monad 维持了合理的内存与带宽要求，保障了完全的地理分布和数百个活跃验证者的参与，从而坚守了去中心化的底线。

### 3\. MonadDB 与并行处理机制

其中最为关键的基础设施创新之一是 **MonadDB**——一个专为 EVM 面向自定义的状态数据库。通过共享内存中的无锁环形队列，应用层与内核之间可以高效地传递任务。这种机制使得“并行访问”成为可能，极大地加速了智能合约的并行执行效率，从根本上解决了传统 EVM 串行处理导致的拥堵问题。

## 三、 生态迁移与商业版图：从 Builder 到 Trader 的新机遇

对于技术开发者与参与早期生态研究的投研人员而言，Monad 的核心吸引力在于其“无痛迁移”与“新场景解锁”。

### 1\. 无缝衔接现有以太坊技术栈

Monad 提供了与 Ethereum 完全一致的接口。开发者可以直接使用现有的技术栈原样上线，包括 Solidity 语言与相同的编译器，继续沿用 Foundry、Hardhat、viem 和 ethers 等熟悉的工具。更重要的是，它保持了相同的 EVM 模型（账户、Gas、交易排序）以及相同的 JSON-RPC 与 WebSocket 接口。这意味着项目方可以在零沉没成本的情况下，将高并发业务迁移至 Monad。

### 2\. 800 毫秒最终确认解锁的商业价值

Monad 带来的 **800 毫秒最终确认**不仅是一个技术指标，更是一个重塑商业逻辑的催化剂：

-   **高保真、高频 DeFi 与实时交易：** 在去中心化二级市场中，低延迟与确定性是执行复杂做市策略和高频套利的基础。0.4 秒出块与 800 毫秒确认将彻底消除滑点焦虑，让链上订单簿（Orderbook）和高频套利策略媲美 CEX 体验。
    
-   **链上状态的 AI Agent：** 结合高吞吐和低成本，AI 代理可以高频地在链上执行资产管理、风险对冲与自动化交互，形成智能化的链上经济体。
    
-   **全链游戏与微支付：** 体验媲美 Web2 支付通道的速度，使得游戏内的每一次细微操作均能上链确权，打开了全新的资产确权空间。
    

## 四、 结语

总结而言，Monad 不仅仅是“另一条高性能公链”，它是针对现有 EVM 生态“木桶短板”的一次系统性底层重构。对于参与 Web3 研究与建设的参与者而言，Monad 提供了一个极具潜力的蓝海生态。在接下来的 Builder Camp 乃至于更长远的生态发展中，如何利用其高吞吐量与并行执行机制，设计出更具金融效率的 DeFi 模型和更具用户粘性的实时应用，将是值得深入挖掘的重点研究方向。
