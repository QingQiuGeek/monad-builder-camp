# 2026-07-09
<!-- DAILY_CHECKIN_2026-07-09_START -->









通过学习 AI Agent 高危案例解析与安全构建，我认识到 AI Agent 的风险不只是“回答错误”，而是它可能连接外部工具并执行真实操作。

例如，Prompt Injection 可能让 Agent 被恶意指令劫持；工具调用失控可能导致数据被误删、邮件被误发或生产系统被破坏；当 Agent 具备支付、数据库、云盘等权限时，安全问题会进一步放大。

我最大的收获是：AI Agent 的安全设计必须前置，不能等出事后再补救。构建 Agent 时要坚持最小权限、人工确认、数据分级、日志审计和工具隔离。尤其是涉及删除、付款、发送邮件、修改生产环境等高风险操作时，必须设置二次确认和权限边界。

未来我在使用或设计 AI Agent 时，会更加关注“它能做什么”“它能访问什么”“它的操作是否可追溯”，而不是只关注模型是否聪明。

# 2026-07-06
<!-- DAILY_CHECKIN_2026-07-06_START -->











今天深入拆解了 Monad 的核心创新 **“Optimized Asynchronous Execution”（优化异步执行）**，彻底刷新了对“高性能链”的认知：

1.  **核心范式转移（先共识，后执行）**：传统链（如以太坊）是“边执行边共识”，导致执行时间被共识严重挤压；Monad 将其解耦，先快速敲定交易顺序（共识），后异步计算交易结果（执行）。这使执行时间可覆盖整个出块周期。
    
2.  **代价与补偿（延迟可见与预留余额）**：
    
    -   **代价**：交易状态滞后 **3个区块（D=3）**，约 **1.2秒**。用户看到“交易上链”只是“插队成功”，余额变动要等执行完毕。这不是比以太坊慢（实际上快了10倍），而是引入了**“薛定谔的猫”状态**——结果确定性在时间上被割裂了。
        
    -   **安全保障**：为了应对异步带来的“空账户堵塞”风险，Monad 强制每个账户预留 **10 MON** 作为 Gas 储备金，确保所有已共识的交易至少能扣掉 Gas 费，维持网络稳定。
        

结合三部曲的后两篇（以太坊上限 & 主权个体），Monad 扮演了**“文明基础设施承重墙”**的角色：

1.  **突破以太坊的“非 TPS 上限”**：  
    文章提到以太坊的上限不是 TPS，而是承载社会复杂性的能力。Monad 通过技术突破**（低硬件要求 + 高吞吐）**，正在降低验证者的门槛（消费级硬件可跑节点），这直接回应了“去中心化基础设施”的文明根基——**只有节点足够分散，UBI（全民基本收入）、可竞争主权等高级社会契约才能建立在可信的中立层之上。**
    
2.  **主权个体的“数字迁移”加速器**：  
    在 DeSoc（去中心化社会）愿景中，主权个体会携带 DID、ZK 凭证和 SBT，在多个 DAO 数字城市间自由迁徙（用脚投票）。
    
    -   **Monad 的价值**：跨 DAO 迁移身份和资产，需要极快的最终性和极低的滑点。Monad 的 400 毫秒出块 + 异步并行，让“换城市”的体验从“坐绿皮火车（以太坊主网）”升级为“坐高铁”。
        
    -   **新摩擦**：不过，迁移时若涉及账户状态（如预留的 10 MON 跨链清算），异步执行的滞后可能会导致短时间内的“双城身份模糊期”，这要求未来的主权身份栈（DID+ZK）必须能优雅处理这种时间差。
