### Q1：Web3 的远程工作方式和传统互联网公司有什么本质不同？

-   **无边界团队**：团队成员可能分布在 10+ 个国家，没有"办公室"概念，Discord 就是办公楼
    
-   **异步优先**：不要求同时在线，靠 GitHub/Discord/Notion 等工具异步协作
    
-   **开源透明**：代码公开、讨论公开、决策公开——你的每一行代码都是你的简历
    
-   **身份即钱包**：你的钱包地址就是你的链上身份，不需要 HR 系统录入工号
    

### Q2：前端工程师 / 智能合约工程师 / 后端工程师，你目标方向的技术栈分别是什么？

| 岗位 | 核心技能 |
| --- | --- |
| 前端 | React/Vue, TypeScript, Viem, Next.js |
| 智能合约 | Solidity, Foundry/Hardhat, ERC 标准 |
| 后端 | Node.js/Go/Python, GraphQL, Docker |

**你的目标**（前端 + 智能合约全栈）：需要同时掌握上面两张表的技能。

### Q3：什么是 Builder？和传统"程序员"有什么不同？

-   **Builder 不只是写代码**：Builder 是发现问题 → 用链上技术解决问题 → 把解决方案推向社区的人
    
-   **程序员 = 接受需求，Builder = 发现需求**：传统程序员等待 PM 分配任务，Builder 在社区里自己发现"这里需要一个更好的 DEX / 借贷协议 / NFT 工具"
    
-   **Proof of Work > 简历**：Builder 的 GitHub 仓库、链上部署的合约、社区贡献记录比任何学历证书都有说服力

## 🧠 核心概念复述（用自己的话写）

> 读完教材后，用你自己的话回答以下 3 个问题。不要复制粘贴！

### Q1：区块链解决了什么问题？为什么叫"区块"+“链”？

解决了去中心化的交易账本技术，区块就是交易记录以及他的哈希，链就是将区块串联起来

### Q2：以太坊跟比特币最大的不同是什么？"智能合约"到底是什么？

功能，共识机制，交易速度以及经济模型；智能合约是存储在区块链上的可执行的代码，执行交易，实现了无须信任的自动化规则,code is law

### Q3：你的钱包地址是怎么来的？私钥、公钥、地址三者是什么关系？

钱包地址来源于EOA，私钥相当于银行卡密码，十分重要，公钥就是私钥和地址的桥梁，用来证明身份，地址就相当于银行卡卡号
