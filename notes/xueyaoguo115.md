# **Web3 学习笔记 Day 4**

## 1.理解合约代码

### （1）整体结构

```solidity
contractNFTBadge{... 这里面所有代码都属于这一台"自动售货机"}
```

`contract NFTBadge { }` 就是这台机器的外壳——里面包住的所有变量和函数，都是"NFTBadge"这一个合约的组成部分,别的合约访问不到里面的东西,除非通过公开的函数

### （2）状态变量

```solidity
mapping(uint256=>address)public badgeOwner;
```

`mapping` 你可以直接理解成**一张查询表/字典**——左边输入一个东西，右边查出对应的另一个东西。

`mapping(uint256 => address)` 的意思是：**"给我一个徽章编号（uint256），我告诉你这个编号属于哪个地址（address）"**。

比如：

```
badgeOwner[0] = 0xAAA... (0号徽章属于这个地址)
badgeOwner[1] = 0xBBB... (1号徽章属于这个地址)
```

这就是咱们之前讲NFT时说的"账本上记的是这一个独一无二的编号目前归谁"——这行代码就是那句话的具体实现。

### （3）权限锁（谁能操作这台机器）

```solidity
modifieronlyOwner(){require(msg.sender== owner,"Not authorized: only owner can mint");_;}
```

`modifier` 是Solidity里一种"可复用的门禁检查"**，你可以理解成一道装在某些函数前面的**安检门\*\*。

-   `msg.sender`：Solidity里的一个内置变量，**永远代表"当前正在调用这个函数的人是谁"**——也就是发起这笔交易的钱包地址
    
-   `require(条件, "报错信息")`：**如果条件不满足，整笔交易立刻失败，并显示报错信息**（还记得咱们讲过的"失败交易也要消耗gas"吗？就是`require`不通过导致的典型场景）
    
-   `_;`：这个下划线是个占位符，意思是"安检通过后，才继续执行被保护的那个函数剩下的代码"

## 4.为什么合约要部署在区块链上，而不是只用区块链管钱

**区块链的核心能力不是"管钱"，而是提供一个没有中心化管理者、但所有人都信得过的公开记录系统**——钱只是这个系统最早、最好懂的应用场景。

-   比特币最初解决的是"钱"的问题，但底层能力其实是"公共账本，谁也改不了"
    
-   以太坊/Monad把这本账本变成**可编程的**（Solidity合约），能记录的不再局限于"余额"，而是任何需要"公开、不可篡改、没人能单方面修改"的信息
    
-   举例：NFT Badge记录的是"谁完成了任务"这件事本身——存数据库里可以被管理员悄悄改，存区块链上只有指定权限能新增、且写上去后连自己都改不了
    
-   同样逻辑适用于：投票合约（防篡改计票）、打卡合约（防补录假记录）、Dapp（没有公司能暗箱操作交易撮合）、DAO（没有老板能私自挪用资金）
    

**一句话**：钱是区块链的第一个杀手级应用，但只要账本能跑代码，任何"需要公开留痕、防篡改"的场景都可以往上搬。

# **Web3 学习笔记 Day 3**

## 1.哈希Hash

**哈希函数**——不管你塞进去的东西多大多小，吐出来的"指纹"永远是固定长度的一串字符。这个过程叫"哈希"，吐出来的结果就叫"哈希值“。

哪怕只改动输入里的一个标点，输出的哈希值会**天翻地覆地变化**，完全看不出规律。这个特性极其重要——它意味着**只要账本上任何一个字被偷偷改动，对应的哈希值立刻会变，篡改行为无所遁形**。

给你一个哈希值，你**没办法反推出原始输入是什么**

### 回到区块链本身：为什么"链"叫链

还记得咱们最开始说的"公共账本，谁也改不了"吗？**哈希正是实现"改不了"这个承诺的技术手段**。

区块链的"块"，是一批批打包好的交易记录。关键设计是：**每一个新区块，都把"上一个区块的哈希值"记录在自己内部**。

你之前贴的转账交易记录里，那个交易本身其实也有一个专属的"**交易哈希（Transaction Hash / TxHash）**"——就是那串很长的字符串，本质上就是"这笔交易的全部内容"经过哈希运算后得到的唯一指纹。你在Monadscan或者MonadVision里粘贴这个哈希去搜索，就能精确定位到这一笔、且只有这一笔交易——因为哪怕交易内容差一个字节，这个哈希都会完全不同。

## 2.NFT代币

NFT = **Non-Fungible Token（非同质化代币）**

区块链是一本"所有人都有一份、谁也改不了"的公共账本。

-   **普通加密货币（比如MON、比特币）**：账本上记的是"数量"——就像账本上写着"A地址有10个单位"，这10个单位彼此完全等价，可以随意拆分、合并
    
-   **NFT**：账本上记的是"这一个独一无二的编号，目前归谁所有"——比如"编号#4728这个东西，现在属于A地址"。这个编号**独一无二，不能拆成一半，也不能跟另一个编号互换成"等价的东西"**
    

## 3.EVM 是什么

**EVM = Ethereum Virtual Machine（以太坊虚拟机）**

还记得咱们讲Solidity合约时的比喻吗——"自动售货机"。EVM就是**运行所有这些自动售货机代码的那台"标准发动机"**。

你写的Solidity代码，不是直接被公共账本读懂的，而是先编译成一种EVM能理解的底层指令（叫bytecode），然后**EVM负责实际执行**这些指令、算出结果、更新账本。

### 什么是"EVM兼容"

**以太坊自己有一台EVM，但这台"发动机"的设计图纸是公开的**。于是很多新链（包括Monad）就直接**照着这套图纸，自己造了一台功能一模一样的发动机**，装在自己的公共账本系统里。

这就是"EVM兼容"的意思：**这条链虽然不是以太坊本身，但它的"发动机"跟以太坊用的是同一套标准**。

## 5.Solidity合约是什么

它是专门为以太坊（以及所有EVM兼容链，比如Monad）设计的编程语言，写出来的代码经过编译后，会被部署到公共账本上，变成一个能被任何人调用的"合约"。

**Solidity合约**：一旦部署到公共账本上，**代码基本没法修改**——因为账本的核心特性就是"不可篡改"，你的合约代码也是账本的一部分。这意味着：**上线前必须反复审计**，一旦有漏洞被黑客发现，钱包里的资产可能被直接转走，没有"下线修复"这个选项。这也是为什么之前提到的"代码审计"在Web3圈子里是个专门的高薪职业。

**智能合约，就是放到区块链上的那一小段程序。**它的特点是：

-   **自动执行：写好规则放上去，只要有人满足条件去触发它，它就自动照章办事，不需要任何人在旁边盯着**。
    
-   **改不了、关不掉：**一旦放上链（部署），它的规则就固定了，谁也没法偷偷改它、也没法把它关停（还是第 3 章那个"不可篡改")。
    

## 6.使用 AI 工具生成一个最小 Solidity 合约，并完成一次人工检查

```solidity
请帮我写一个最小化的 Solidity 智能合约，用于 Web3 训练营的学习练习。

需求：
- 功能是"NFT Badge"——一种不可转让的徽章，用来记录某个地址完成了某项任务/活动
- 只有合约部署者（owner）能够铸造徽章给别人，普通用户不能自己铸造
- 每个徽章有一个 tokenId，记录归属地址，以及一个 URI 字符串（用来指向徽章的图片/元数据）
- 不需要实现完整的 ERC721 标准（不需要 transfer、approve 等功能），因为这是不可转让的荣誉徽章
- 使用 Solidity ^0.8.20
- 代码尽量精简，变量和函数命名清晰易懂，适合初学者阅读
- 生成代码后，请用简单语言解释一下这份合约的结构：有哪些状态变量、哪些函数、权限控制是怎么实现的
```

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract NFTBadge {
    string public name = "Web3 Summer Camp Badge";
    string public symbol = "CAMP";

    address public owner;
    uint256 public nextTokenId;

    mapping(uint256 => address) public badgeOwner;
    mapping(uint256 => string) public tokenURI;
    mapping(address => uint256) public balanceOf;

    event BadgeMinted(address indexed to, uint256 indexed tokenId, string uri);

    modifier onlyOwner() {
        require(msg.sender == owner, "Not authorized: only owner can mint");
        _;
    }

    constructor() {
        owner = msg.sender;
    }

    function mintBadge(address to, string memory uri) public onlyOwner {
        require(to != address(0), "Cannot mint to zero address");

        uint256 tokenId = nextTokenId;
        badgeOwner[tokenId] = to;
        tokenURI[tokenId] = uri;
        balanceOf[to] += 1;

        nextTokenId += 1;

        emit BadgeMinted(to, tokenId, uri);
    }
}
```

### AI的解释

> 这个合约有三组状态变量：`badgeOwner`（记录每个tokenId归谁）、`tokenURI`（记录每个徽章指向的元数据链接）、`balanceOf`（记录每个地址一共拥有几个徽章）。核心函数只有一个——`mintBadge`，被 `onlyOwner` 这个修饰器保护，意味着只有部署合约的那个地址能调用它。每次铸造成功后会触发 `BadgeMinted` 事件，方便外部程序（比如前端界面）监听到"谁在什么时候获得了徽章"。这个合约刻意没有实现转让功能，因为设计目标是"记录荣誉"，不是"可交易资产"。

## 7.Gas费两个单位：Wei、Gwei

ETH 可以像"元、角、分"一样拆得很细。因为 Gas 费常常很小，用"元"表示是 0.0000021 这种，太难看，于是发明了小单位：

-   **Gwei**：表示 Gas 价格时最常用的单位，可以理解成"分" —— 说"油价 20 Gwei"就像说"油价 2 毛"。
    
-   **Wei**：最最小的单位，像"厘"。

## 2.概念理解——钱包到底是什么

**MetaMask钱包里并没有存着你的"钱"**。

区块链是一个"所有人都有一份、谁也改不了"的公共账本，你的余额、资产，其实是**记在这本公共账本上的一行数据**，账本本身存在全世界成千上万台电脑里，不存在你的MetaMask里。

那MetaMask里到底装的是什么？——**一把钥匙**。

具体说，是一对"钥匙"：

-   **私钥（Private Key）**：只有你自己知道的一串密码，谁拿到这串密码，谁就能操作账本上属于这个地址的资产。**这个丢了/泄露了，没有客服能帮你找回**（还记得咱们讲过的"去中心化没有客服"吗）
    
-   **公钥/钱包地址**：相当于账本上"你那一格"的门牌号，可以公开给别人，让别人往这个地址转账
    

所以MetaMask更准确的定位是：**一把能打开公共账本上"你那一页"的钥匙串**，它本身不存钱。

为什么要选网络

这是个很关键的问题，答案是：**"区块链"不是一本账本，而是好多本互相独立的账本**。

以太坊主网是一本账本，Monad测试网是另一本完全独立的账本，Polygon、Arbitrum……每一条链都是一本单独的公共账本，**彼此不共享数据、互相不认识**。

MetaMask这把"钥匙"本身是通用的——同一串私钥，理论上能打开好多本不同的账本（这也是为什么很多链都兼容以太坊的钥匙格式，也就是之前提到的"EVM兼容"）。但**你必须先告诉MetaMask"我现在要查哪本账本"**，它才知道去哪儿查你的余额、往哪儿发送你的交易。

## 3.概念理解——区块链浏览器是什么

区块链浏览器就是**查阅这本公共账本的网页界面**——你不用自己去读原始的区块链数据，直接打开这个网站，输入一个钱包地址、交易哈希、或者合约地址，就能像查银行流水一样看到：

-   这个地址收发过哪些交易
    
-   某笔交易到底成功了没有、花了多少Gas费
    
-   一个智能合约的代码内容是什么
    
-   当前网络的整体状态（比如最新区块高度、总交易数）

（2）概念理解

## 2.区块链知识

**区块链：**"所有人都有一份、谁也改不了、公开透明"的记账本。每一个区块包括两个部分：交易记录等一些信息, 和之前的区块摘要。一条链的任意一条边只可以连接两个区块。组合在一起就是区块链。

特性：不可篡改；公开透明、匿名；快速交易

分布式网络帮助区块链实现了去中心化和真正的不可篡改

区块链的组成部分：去中心化的网络和区块链；维持网络运行的代币激励（矿工）

公链vs联盟链vs私链：

| 区块链类型 | 节点加入方式 | 数据可见性 | 管理模式 | 适合场景 |
| --- | --- | --- | --- | --- |
| 公链 | 任何人自由加入 | 所有人可见 | 去中心化（大家投票） | 加密货币、公共存证 |
| 联盟链 | 需联盟成员邀请/审批 | 仅联盟成员可见 | 多中心化（董事会决策） | 供应链、金融协作 |
| 私链 | 由老板严格审批 | 仅内部成员可见 | 中心化（老板说了算） | 企业内部管理、审计 |

Dapp（去中心化应用，Decentralized App）**核心特征：**

-   **数据主权归用户**：用区块链存储身份和资产
    
-   **无需信任中介**：智能合约自动执行规则

## 4.Monad的介绍

让我开始思考如何将虚拟货币的交易与我们日常生活中的信用卡交易进行对比。让我印象深刻的是：过去的压力测试如何塑造了 Monad 的设计。CryptoKitties、DeFi Summer、NFT 铸造潮、Memecoin 交易——每一次都暴露同一个瓶颈：以太坊的**单线程串行执行**模型。所有交易排队处理，一旦需求突增，Gas 费飙升、网络拥堵是必然结果，不是意外。

Monad 的技术方案基本是针对这个瓶颈逐条打补丁：

-   **Optimistic Parallel Execution（乐观并行执行）**：交易并行处理，事后解决冲突，而不是像以太坊那样默认串行
    
-   **MonadBFT**：基于 HotStuff 优化的共识算法，专注缩短出块与最终确认时间
    
-   **MonadDB**：为区块链工作负载定制的存储层，让普通硬件也能跑节点，兼顾性能与去中心化
    

宣传的目标是 全兼容以太坊虚拟机（EVM），开发者可以直接部署现有的以太坊智能合约，无需重写代码或学习新的执行环境，同时把吞吐量和延迟做到远超以太坊现状。

Monad 在这套逻辑里扮演什么角色：它不是在发明新概念，而是在解决"**公共账本记得太慢**"这个问题。以太坊这个账本很安全、很公开，但记账速度慢（每秒能处理的交易少），一旦很多人同时来记账，就得排队、加钱插队（也就是 Gas 费飙升）。Monad 想做的，就是让这个公共账本**记得更快、成本更低**，同时保持"公开透明、没有老板"这两个核心特性不变。
