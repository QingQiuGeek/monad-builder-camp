# 安全合规

### 国内法律红线

国内 Web3 法律底线：

我国明确禁止发币融资、虚拟币交易、挖矿，只要项目涉及代币募资、多层级拉人头返利、链上赌博、虚拟币跨境换汇，参与开发、运营、推广的所有人都可能担刑责，会触犯非法集资、传销、开设赌场、洗钱等罪名。同时 Web3 很多项目境外注册，用代币发工资、私下场外换币出金风险极高，容易银行卡冻结甚至涉案，入职前一定要核查项目模式，避开违规业务。

### 全球监管趋势与入职避坑要点

海外各地监管规则：

欧盟、美国、香港都出台加密监管法案，统一要求平台做实名核验、交易溯源，稳定币、交易所管控越来越严，全球监管整体趋严。另外找 Web3 工作要留心用工风险，很多境外项目不给员工交社保，只用虚拟货币发薪资不受劳动法保护，一旦欠薪、资产归零很难维权，入职前尽量协商一部分人民币薪资，保留资金来源凭证。

### 网络诈骗类型 + 简单防护办法

普通人最容易中招的网络安全骗局：

比如仿冒官网、假面试软件、空投钓鱼、剪贴板木马，骗子会套取助记词、私钥盗走钱包资产，域名细微改字、陌生安装包、私聊链接全是高危陷阱。

防护清单：只用官方会议工具、不随便连接陌生网站钱包、私钥离线手写保存、转账完整核对地址，遇到可疑链接和软件直接拒绝，从源头减少被盗风险。

# 智能合约开发

## 构成

普通网站：前端页面 + 后台服务器 + 数据库，全是一家公司管。

区块链 Dapp：没人统一管控，分成 4 个核心零件，缺一不可：

1.  前端，用户看见的页面，必须靠中间的钱包中转链接区块链。
    
    1.  只查余额、看 NFT：免费，不用花钱；
        
    2.  转账、发留言、买 NFT：要钱包签名，扣手续费 Gas，交易上链。
        
2.  智能合约 - 核心
    
    1.  存在链上、自动执行的代码，相当于项目 “**永久不变**的后台规则”。
        
    2.  比如发代币、NFT、去中心化交易所的买卖规则，全写在这里，一旦**部署不能随便改**。以太坊用 Solidity 语言写。
        
3.  数据检索器
    
    1.  区块链原始数据杂乱，前端直接读取会很慢。
        
    2.  检索器专门监听合约的所有操作记录（比如谁转了 NFT），整理成干净表格存普通数据库，前端直接查表，页面加载飞快。
        
4.  区块链+去中心化存储
    
    1.  区块链：存合约代码、所有人余额、每一笔交易记录；
        
    2.  IPFS 这类工具：存图片、视频、文档（NFT 图片都放这，不上链省手续费）。
        

## 构建步骤

1.  **需求规划**
    

先想好做什么：留言板 / 发代币 / NFT，选用哪条链（以太坊、Polygon 等），设计页面怎么操作。

2.  **写智能合约**
    

用 Solidity 写规则，写完大量测试，检查有没有盗币漏洞，优化手续费。

3.  **写检索器**
    

抓取链上操作记录，整理成数据库方便前端读取。

4.  **做前端页面**
    

写网页，接入钱包，对接合约，实现查数据、发起交易。

5.  **RPC 节点对接**
    

网页靠 RPC 节点和区块链通信，后面单独细说。

6.  **部署上线**
    

先部署到**测试网**（代币无真实钱，随便造），测试没问题再上主网，最后维护更新。

## 环境工具

node 、git

开发框架：

-   **Foundry**：速度最快，现在行业主流，用来编译、测试、本地模拟链、部署合约；自带 anvil 本地虚拟链，不用花钱就能测试。
    

-   **Hardhat**：上手简单，新手友好，配套教程多。
    

在线 Remix

## RPC 柜员

把整条区块链当成银行总系统，**RPC 节点就是遍布各地的 ATM 机**。

-   网页不能直接连区块链，必须找 ATM（RPC 节点）传话；
    
-   查余额、发交易、看区块数据，全部通过 RPC 完成。
    

用于一个出入端口，可以查数据、可以用于广播交易记录

## Solidity 合约语言

-   状态变量：永久存在区块链上的数据（所有人余额、留言记录），修改手续费很贵；
    
-   函数：合约能执行的操作，分 4 种权限：
    
    -   public：任何人都能调用；
        
    -   private：只有合约内部能用；
        
    -   view：只查数据，不修改，免费；
        
    -   payable：能收 ETH，转账、充值用；
        
-   事件（Event）：记录关键操作（转账、发留言），检索器、前端靠事件抓链上动态；
    
-   修饰符：权限锁，比如 onlyOwner，只有合约创建者才能调用管理员功能。
    

### 合约安全-很重要

-   **重入攻击（最经典漏洞）**
    
    错误写法：先转账，再修改用户余额 → 黑客反复取钱，把合约钱掏空。
    
    正确写法：先改余额，再转账，从根源杜绝。
    
-   **权限漏洞**
    
    管理员功能不加限制，任何人都能提取合约全部资金，必须加 onlyOwner 锁。
    
-   **数字溢出**
    
    0.8 版本以后系统自带防护，老版本会出现数字越界，被人无限领奖励。

# 赛道详情案例

区块链从最初的加密货币比特币已经形成了平台（以太坊推动了很大），然后逐步催生了覆盖金融、组织治理、艺术和社区等等的生态系统。下面是四个比较经典关键的领域方向。

DeFi（去中心化金融）、NFT（非同质化代币）、DAO（去中心化自治组织）与 MEME（模因币）四大领域

## 去中心化金融

传统银行、支付宝、券商都是有个公司管着你的钱，人家可以冻结账户、限额转账、半夜没法取钱。DeFi 就是把银行搬到区块链上，没有老板、没有平台管控，代码定好规则自动运行，谁也没法私自扣你资产。没人垄断掉信息，不用看别人脸色。

## 飞同质化代币

普通数字货币（比如比特币）1 个和另 1 个完全一样，能随便拆分互换，就像人民币。

NFT 是独一无二、编号唯一、不能互换的凭证，相当于区块链上带防伪码的正版证书。

一张图片、一段视频、一首歌做成 NFT 后：

-   谁是原版持有者全网可查，没法盗图冒充版权；
    
-   后续每次转手卖出，原作者能自动分到一笔版权费，不用靠平台中介。
    

常见用途：头像藏品、数字门票、游戏道具、艺术品确权。

这较为有利于版权的自动监管了。其实目前的版权全都中心化，有对应的监管平台，但平台又不会太主动去看侵权的问题，只有走法律被动走。但我们这种情况可以规避掉这种局限性。

## 去中心化自治组织

这有意思了，谁能拒绝权利的诱惑？如何实现完全民主？

  
普通公司：老板 + 管理层拍板决定花钱、做项目、招人。

DAO：没有老板，手里持有这个组织代币 / NFT 的所有人都是股东，大事必须全员投票，票数够了智能合约自动执行，没人能独断专行。

举几个例子：

1.  Nouns DAO：每天拍卖一个 NFT，卖来的钱进公共金库，大家投票决定这笔钱用来投项目、做广告、搞公益；
    
2.  乡村 DAO：村里搞农业、文旅，发行社区代币，村民持有代币就能投票决定村子怎么发展、分红怎么分。
    

缺点：人多意见杂，决策慢，容易被大额持仓大户操控投票。

人人都是股东，但涉及关键决策，太多声音是否是好事呢？权重搞了还是会形成小的董事会吧。

## MEME币 土狗

不靠技术、不靠产品，纯靠段子、表情包、玩梗、网友热度炒起来的币，比如狗狗币、佩佩蛙。本质就是**靠网络情绪和共识炒作**，没有实际落地用途。优点：牛市里热度一来价格能短期翻几十上百倍；风险极大：项目方可以偷偷套现跑路，热度一过价格直接暴跌归零，绝大部分人会亏钱，纯投机赌博。

纯纯吃限时流量是吧。

# 以太坊

以太坊在比特币后面出现。貌似比特币不是一个平台，而是更强调其货币属性。

以太坊相比于比特币那块，更注重于平台的建设、支持智能合约非中心化应用开发。他们在定位、共识机制上都有不同。

比特币里总量永久固定 2100 万枚依靠 PoW 挖矿发行，区块十分钟出块速度缓慢，核心目标就是打造去中心化稀缺数字货币；而以太坊搭载图灵完备编程语言 Solidity，依靠智能合约实现自动化条件执行，跳出单纯货币范畴打造可运行各类程序的去中心化平台，原生代币 ETH 没有固定总量上限，在 EIP-1559 销毁机制与合并升级后转为小幅通缩，区块出块仅 12 秒更适配高频交互场景。

所以以太坊能衍生出多元的业态，比如去中心化金融、数字藏品等。这可以说是两个分支

## 共识机制演进

起初也是和比特币一样工作量证明，就是大概谁的算力贡献多谁就得到奖励多。这会产生巨额点亮消耗且效率低下。

后来进阶到PoS权益证明体系和工作量证明双线并行。再到后面标志性的the merge合并，直接就是关掉了挖矿转了信标链条。违规成本大幅度提高，能耗下降99.

## 扩容升级

the merge后核心目标就围绕降低使用成本还有吞吐量以及安全性。整体都是为了这三个指标服务了。比如落地EIP-4844升级，就是Blob交易协议，以往二层网络向主网提交批量交易数据需要占用常规区块存储空间，手续费高昂，该升级新增专门的临时数据存储格式，数据到期自动清理，无需长期占用主网资源，直接让一些主流 L2 手续费降低 70% 至 90%；其次是分片技术调整，以太坊放弃原先 64 条分片链并行处理交易的方案，改为聚焦数据分片，主网只负责数据可用性与最终结算，所有实际交易交由二层 Rollup 处理；另外 ZK-Rollup 零知识证明扩容技术作为重点方向，能够在链下批量打包上千笔交易后生成一条简洁密码学证明提交主网核验，无需逐笔校验交易，进一步压缩验证耗时与费用。

## 层级划分

清晰分为了应用层、协议层和扩展层。网络里有L1主网 L2扩容网和侧链。协议层承载共识客户端 底层协议基础设施。扩展层就同意收纳所有二层扩容方案了。

# 区块链概念

## 从特性入手

-   不可篡改
    
    -   怎么实现的？去中心化，数据之间具有粘连，一条绳上的蚂蚱，荣辱与共。具有连坐性，一个地方产生了交易等，所有地方都会同时记录，保证了数据稳定性，当有超过51%的点改了这，才认为是真改了。边际下讲就是你改了这个必须全改。
        
    -   去中心化来维护，全国每人个人都有备份，你有本事就改完
        
    -   那么谁愿意去做这个节点？谁甘心做白水牛马？所以要给好处（链上的币）。这就形成了挖矿。
        
-   公开透明-匿名
    
    -   最危险的地方就是最安全的地方？最好的鉴权就是不鉴权？链上的信息都是公开的谁都可以看，但是链上地址不捆绑任何个人信息，除非你自己暴露。
        
-   快速交易
    
    -   这个不太清楚，为什么一被打包，记录了交易记录就算完成就比其他快？难道是因为中心化的数据库交互慢？这应该是根本上的快。而后在此基础上优化，加入分布式等等。
        

## 区块链流转运行

关键步骤包括用户发起交易（智能合约）、交易信息同步广播到其他结点、结点验证合法性、打包成块、块上链、发放奖励。

也是由区块链特性来展示这个流程，有交易就需要备份到所有结点，通过**共识机制哪些个算法**保证安全，一致。

这里保留下问题，就是这里的新块的触发是什么？是十分钟就会打包新块？还是到了容量限制会打包新块？每次交易也就是处理数据都会涉及发放奖励吗？  
  

# 公-联盟-私

公联盟私逐步变成独裁中心化，数据与治理权集中在部分个体甚至个人手上，决策自然会越来越一言堂，会变得越来越快，相应的也会出现信息差、对齐有偏差等等问题。这其实也能追溯到什么扁平化管理的问题，信息超过两层传递就会出现大误差了。  
说歪了，公链就是大公社人人平等，用共识机制投票做决策， 慢但安全民主。联盟的就比如一些金融机构银行、董事会，几个关键决策参与者利益占有者进行主要决策，那就决策慢了，数据集中了，处于安全与速度的平衡。到了私有链，可以说是封建制度，秦始皇一言堂，决策相当之迅速，数据相当之集中且不透明。隐私什么的都会保护在一处。

# web3 web3.0

今天第一次知道web3.0，和web3关系不是很大但能结合。web2中平台垄断，数据隐私不能掌握自己手中，到web3中去中心化的区块链革命。web3.0主要体现在数据层面，数据更结构化更语义化且有条理有图。

# 总结

整体而言就是web3将个人信息隐私数据牢牢给到用户自身掌握，去中心化大家伙审查免掉了中心的审计等，增强安全性。但缺点就是投票共识机制决策会减缓执行效率，生态治理与易用较差。
