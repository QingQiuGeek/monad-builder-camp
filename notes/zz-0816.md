## 一、两种主流模式（选型第一步）

### 模式 1：代理支付（用户钱包托管，C 端个人 Agent，国内主流）

Agent**没有独立资金账户**，仅作为用户的支付代理人，资金全部走用户本人支付宝 / 微信 / 银行卡，适合个人助手、购物、订票 Agent（支付宝 ACT 协议、微信 AI 开放平台）**凤凰网科技**。

-   逻辑：用户一次性授权 Agent 代理支付，设置消费规则；Agent 发起支付请求，支付平台校验授权范围后扣用户账户资金。
    
-   优点：合规简单、资金安全、不用单独开户；缺点：依赖国内支付服务商开放 AI 代理接口。
    

### 模式 2：自主钱包支付（Agent 自有资金，B 端 / 海外 / 链上 M2M）

Agent 拥有独立托管钱包（法币钱包 / 稳定币钱包 USDC），自有预算自主付费，适合企业采购 Agent、API 付费、跨智能体交易（AWS Bedrock、Stripe Link、x402）**Amazon Web…**。

-   逻辑：企业给 Agent 钱包充值预算，内置 spending policy 消费规则，自动支付服务商、API、云资源。
    
-   优点：高度自治、机器间小额高频支付；缺点：需资金托管资质、海外 / 区块链方案为主。
    

## 二、五层核心技术架构（完整实现链路）

### 1\. 身份与授权层（信任根基，合规核心）

支付合法前提：**每一笔 Agent 支付必须绑定用户可核验、可撤销、可存证的授权凭证36氪**。

（1）Agent 可信身份

-   国内：支付平台分配独立 AgentID，OAuth2.0/OpenID Connect 机器鉴权，绑定用户实名账户；
    
-   海外 / 链上：W3C DID 去中心化身份，Agent 唯一数字身份，不可伪造。
    

（2）可验证授权凭证（Mandate/VDC）

用户签名生成加密防篡改授权文件，是 Agent 花钱的 “电子许可合同”，包含 5 项强制约束**稀土掘金**：

1.  金额：单笔上限、日 / 月总预算；
    
2.  场景：允许商户 / 品类（仅外卖、禁止转账提现）；
    
3.  时效：授权有效期、仅用户离线可支付；
    
4.  审批阈值：小额自动支付、大额推送人工确认；
    
5.  可撤销：用户一键终止 Agent 支付权限，即时生效。
    

主流标准：

-   国内：支付宝 ACT 2.0 智能体商业信任协议、AHA 互联接口；
    
-   国际通用：AP2 协议（谷歌 + 万事达 + PayPal 共建）、x402 HTTP 微支付协议；
    
-   链上：VC 可验证凭证、EIP-712 签名、ERC4337 账户抽象可编程钱包。
    

授权两种形态

1.  预授权（批量自动）：用户提前设置规则，Agent 小额高频免确认；
    
2.  单次意图授权（高风险）：每笔大额交易推送用户确认，生成单次 Cart Mandate。
    

### 2\. 规则风控引擎（控制层，防止乱花钱）

部署在 Agent 与支付通道中间，**每笔支付请求先过规则校验，不满足直接拦截letagentpa…**：

-   自然语言转规则：用户输入 “每日 API 花费不超过 50 元，禁止夜间消费”，LLM 自动转为可执行 JSON 策略；
    
-   实时校验维度：剩余预算、单笔限额、商户黑白名单、交易频次、时段；
    
-   分级处理：小额自动放行、超阈值人工审批、异常交易直接阻断 + 告警。
    

### 3\. 钱包与签名层（资金载体，密钥安全隔离）

分两类钱包方案，密钥全程隔离，Agent 永远触碰不到私钥：

方案 A：绑定用户现有钱包（国内 C 端）

支付宝 AI 开放平台、微信 AI 助手：Agent 通过授权令牌调用用户钱包，签名由支付平台 TEE 可信执行环境完成，Agent 仅传递交易参数，无密钥权限**Alipay+**。

方案 B：Agent 独立托管钱包（海外 / B 端）

1.  法币托管钱包：Stripe Link、Coinbase CDP、AWS Bedrock Payment Connector，服务商托管私钥；
    
2.  链上稳定币钱包（USDC）：Circle 钱包、ERC4337 智能合约钱包，TEE 硬件加密签名；
    
3.  安全隔离：密钥存储在 TEE / 硬件密钥，Agent 仅发起支付指令，签名由钱包服务独立执行。
    

### 4\. 支付执行通道层（打通资金链路）

根据场景选择对应支付接口，Agent 标准化调用：

1.  **国内互联网支付（个人 Agent 首选）**
    
    支付宝 AI 开放平台 Skill 工具、微信 AI Agent 支付 API，基于 ACT 协议，支持自动下单扣款、账单同步、退款自动处理；
    
    流程：Agent 解析购物意图→组装支付参数→携带 Agent 授权凭证调用支付网关→平台校验用户授权→扣用户余额 / 银行卡→返回支付结果给 Agent。
    
2.  **海外法币通道（企业海外 Agent）**
    
    Stripe Agent Pay、Visa Intelligent Commerce、PayPal Agent API，基于 AP2 协议，支持信用卡自动代扣、企业预算管控。
    
3.  **机器微支付 M2M（API / 模型付费）**
    
    x402 协议：HTTP 原生支付，服务商返回 402 Payment Required，Agent 自动用钱包签名支付凭证放入请求头重试，实现按调用付费，无需预充值套餐。
    
4.  **区块链自主结算（跨 Agent 交易）**
    
    ERC4337 账户抽象钱包、USDC 稳定币，支持批量交易、免 Gas、可编程消费限制，适合纯机器自治场景。
    

### 5\. 审计与对账层（追溯、售后、监管）

全链路日志永久存证，满足监管对账与纠纷举证：

-   每笔交易绑定：AgentID、用户授权凭证、商品明细、时间、预算消耗、签名记录；
    
-   自动对账：Agent 后台实时展示消费账单，支持导出、退款申请、异常交易回溯；
    
-   纠纷举证：加密授权凭证可提交支付机构 / 法院，证明支付为用户授权行为。
    

## 三、端到端完整支付流程（以国内购物 Agent 举例）

1.  用户授权：绑定支付宝，设置规则「单笔≤200 元，日上限 1000 元，仅实物消费，超 500 元需确认」，生成加密 ACT 授权凭证；
    
2.  Agent 任务规划：收到指令 “买 30 元咖啡”，拆解下单 + 支付工具调用；
    
3.  风控校验：引擎检查 30 元未超单笔 / 日预算、商户在白名单，自动放行；
    
4.  发起支付：Agent 携带 AgentID + 授权凭证 + 订单参数调用支付宝 AI 支付网关；
    
5.  平台核验：校验授权有效期、额度、场景合规，TEE 完成用户钱包签名扣款；
    
6.  结果回传：支付成功返回订单号，Agent 同步订单状态；超阈值则推送手机确认，用户同意后再扣款；
    
7.  审计记录：全链路日志存入存证系统，用户可随时查看、撤销授权。
    

## 四、落地三大关键约束：安全 + 合规

### 1\. 安全红线

-   Agent**不可持有用户支付密钥 / 银行卡明文**，所有签名由支付服务商 TEE 隔离执行；
    
-   授权不可无限期、不可无上限，必须强制限额与有效期；
    
-   支持一键撤销全部支付权限，撤销即时生效，未完成交易终止。
    

### 2\. 国内合规硬性要求

1.  实名绑定：Agent 支付必须关联已实名的个人 / 法人支付账户，禁止匿名支付；
    
2.  委托授权书面化：授权规则可视化展示，用户主动确认，禁止静默默认授权；
    
3.  交易留痕：所有 Agent 支付流水至少保存 5 年，满足反洗钱监管；
    
4.  禁止高风险场景：Agent 不可自主转账、提现、大额理财，仅限定消费场景。
    

### 3\. 海外监管要点

-   AP2/x402 需配套 KYA（Know Your Agent）代理身份核验；
    
-   企业 Agent 钱包需资金托管资质，区分自有资金与客户资金隔离。
    

## 五、开发者快速落地两种路径

### 路径 1：国内个人消费 Agent（最简，支付宝 / 微信）

1.  入驻支付宝 AI 开放平台，创建 Agent，开通 AI 支付 Skill；
    
2.  前端实现用户授权弹窗，生成 ACT 标准化支付委托；
    
3.  Agent 内置 MCP 工具调用支付 API，接入风控规则引擎；
    
4.  对接账单、退款、授权撤销接口，完成审计后台。
    

### 路径 2：海外企业 / API 自治 Agent（AWS/Stripe/Circle）

1.  开通 Coinbase CDP / Stripe Link 托管钱包，为 Agent 分配独立预算会话；
    
2.  集成 AP2/x402 支付协议连接器，封装 paidFetch 自动处理 402 付费请求；
    
3.  配置 Spending Policy 预算规则，接入 TEE 签名服务；
    
4.  搭建审计看板，同步每笔支付凭证与结算记录。
