# Web3 三大方向完整细分详解（技术 / 运营 / 研究）

每个板块包含：核心定位、细分岗位、日常工作、必备知识、工具、入行门槛、优缺点、适合人群，内容完整适配你当前学习智能合约的背景。

## 一、Web3 技术方向（底层建造者，你现在正在学习这条线）

### 整体定位

整个 Web3 行业的基建核心，所有公链、合约、钱包、DApp、交易所产品全部由技术团队开发；负责把白皮书里的规则变成可链上运行的代码，保障资金安全、链上交互稳定。

### 细分赛道 & 完整工作内容

1）智能合约开发（你当前 Mini Demo 所属赛道，需求量最大）

1.  编码开发
    
    使用 Solidity 编写 ERC20、ERC721、ERC1155、质押、借贷、时间锁、AMM 等合约；基于 OpenZeppelin 安全库开发，自定义业务逻辑。
    
2.  本地调试与测试
    
    Hardhat / Foundry 搭建本地私有链，编写单元测试、边界用例；复现大额转账、权限漏洞、重入攻击等风险场景。
    
3.  部署与多链适配
    
    本地[localhost](http://localhost)、Sepolia、Monad、Arbitrum、以太坊主网部署；配置 hardhat.config 多网络 RPC、私钥环境隔离。
    
4.  合约优化与 Gas 节约
    
    简化存储变量、减少外部调用、优化循环逻辑，降低用户交互手续费。
    
5.  对接前端交互
    
    提供 ABI、合约地址，配合前端实现铸币、转账、质押、提取收益等交互功能。
    
6.  上线前自查
    
    排查常见漏洞：重入、整数溢出、权限失控、tx.origin 鉴权、随机数漏洞。
    

**必备技能**

Solidity、Hardhat/Foundry、TS/JS、EVM 底层逻辑、Gas 机制、EIP 规范、Git、区块浏览器。

**日常工具**

VS Code、MetaMask、Terminal、GitHub、Monad 测试网浏览器、Remix。

2）DApp 前端开发（页面层，连接钱包与合约）

1.  钱包接入：MetaMask、Coinbase Wallet、Rabby 连接逻辑，签名交易、切换网络。
    
2.  页面功能：铸造 NFT、质押挖矿、兑换代币、资产查询、交易历史页面。
    
3.  链上数据读取：通过 Viem/Ethers.js 读取合约余额、解锁时间、质押数量。
    
4.  交易状态监听：等待区块确认、失败报错提示、Gas 费用自定义面板。
    

**必备技能**

React/Vue、Viem/Ethers、TS、Web3 交互逻辑、HTML/CSS。

3）链上后端 / 数据索引开发

1.  搭建节点服务，同步链上全部交易、区块、合约事件。
    
2.  写索引程序抓取转账、铸币、质押记录存入数据库。
    
3.  对外提供 API，给前端、数据分析平台调用链上数据。
    
4.  处理批量查询、大户持仓统计、代币发行量统计。
    

4）区块链底层开发（高门槛，公链 / 跨链）

使用 Rust / Go 开发 L1 公链、Layer2、跨链桥、共识算法；修改 EVM 虚拟机、自定义 Gas 模型、并行交易（如 Monad 并行执行逻辑）；搭建节点集群、P2P 网络。

5）Web3 安全审计（高薪刚需岗位）

通读整套合约源码，自动化工具 + 人工审计；输出漏洞报告，区分高危盗币漏洞、中危逻辑缺陷、低危优化建议；指导开发修复漏洞，复测验证；跟进历史黑客攻击案例，建立漏洞知识库。

### 技术方向入行门槛

-   入门门槛：中等偏上，需要学习代码、命令行、区块链底层概念；
    
-   成长周期：3–6 个月可完成 Mini Demo、独立编写基础合约；1–2 年可独立承接 DeFi 合约开发；
    
-   薪资：三大方向里上限最高，合约开发、安全审计薪资显著高于运营、研究。
    

### 优势与劣势

优势：不可替代、岗位缺口大、远程机会多、技术沉淀通用（以太坊兼容链全部通用）；

劣势：前期学习成本高，需要持续跟进新 EIP、新公链技术；一旦代码上线出错，会直接造成用户资产损失，责任重。

### 适合人群

能接受敲代码、喜欢逻辑推理、愿意反复调试命令行、对链上底层运行原理感兴趣的人。

## 二、Web3 运营方向（用户与生态连接器，无代码门槛）

### 整体定位

项目对外的窗口，承接开发团队的产品，面向普通用户、社区、合作方做推广、维护、活动；目标是提升项目热度、留存用户、扩大生态合作，不需要写代码，但必须看懂基础区块链概念。

### 细分赛道 & 完整工作内容

1）社区运营（行业最基础、需求量最大）

1.  社群日常维护
    
    管理海外社群：Discord、Telegram、X（Twitter）；回答新手基础问题：钱包导入、测试币领取、合约交互步骤、转账失败、Gas 设置。
    
2.  社群秩序管理
    
    管理机器人、管理员团队，清理广告、骗子链接、诈骗空投，防范社群钓鱼。
    
3.  用户反馈收集
    
    整理用户对 DApp、合约功能的吐槽、需求，同步给技术团队迭代优化。
    
4.  线上活动落地
    
    举办 AMA 项目问答、白名单抽奖、答题任务、社交转发空投、链上交互活动。
    
5.  DAO 治理协助
    
    组织社区投票、讲解治理提案、引导用户参与链上投票。
    

2）内容运营 & 科普写手

1.  产出多形式科普内容
    
    图文教程、短视频脚本、长文研报简化版、合约交互分步指南；把复杂智能合约、Gas 机制、Monad 公链特点翻译成普通人能看懂的文字。
    
2.  多语言内容翻译
    
    中英双语白皮书、活动规则、新手操作手册。
    
3.  平台分发
    
    X、Medium、B 站、小红书等渠道定时发布内容，提升项目曝光。
    

3）市场活动 / 增长运营

1.  策划拉新活动：交易大赛、NFT 免费铸造、流动性挖矿激励、任务平台联动。
    
2.  数据复盘：统计活动新增钱包地址、链上交互量、社群新增人数，判断活动效果。
    
3.  KOL 合作对接：联系海外加密博主、分析师做推文、直播 AMA。
    

4）商务生态运营

1.  跨项目合作：和 NFT 项目、DeFi 协议、公链（Monad/Sepolia）互换资源、联合活动。
    
2.  交易所对接：对接中心化 / 去中心化交易所洽谈代币上线、流动性合作。
    
3.  生态拓展：拓展节点服务商、安全审计公司、链上工具平台合作。
    

5）舆情与合规运营

实时监控全网负面言论、黑客事件、监管新闻；规避违规宣传，调整话术，防止项目被限流。

### 运营必备知识（不用写代码）

1.  基础概念：钱包转账、Gas、智能合约、测试网 / 主网、空投、NFT、质押；
    
2.  工具：Discord 后台管理、推文排版、剪辑工具、表单统计、区块浏览器基础查询；
    
3.  加分项：英语读写流利，海外社群沟通无障碍。
    

### 入行门槛

-   入门门槛：低，零基础 1–2 个月可上手基础社群维护；
    
-   成长周期：半年可独立策划完整活动；1 年以上可负责整体市场增长。
    

### 优势与劣势

优势：不用写代码、入行快、社交属性强、适合擅长沟通的人；

劣势：替代性高，基础社群运营竞争激烈；薪资上限低于技术；行业热度波动大，熊市会大量缩减运营岗位。

### 适合人群

擅长沟通、喜欢社交、会策划活动、擅长文案写作、英语不错、不想学习编程的人。

## 三、Web3 研究方向（行业分析师，数据与逻辑研判）

### 整体定位

Web3 行业 “智库”，通过链上数据、行业资讯、项目拆解、政策分析输出专业判断，服务投资机构、项目方、媒体；核心是分辨项目好坏、预判赛道发展趋势，介于技术和运营中间。

### 细分赛道 & 完整工作内容

1）赛道宏观研究（行业分析师）

1.  持续跟踪各大板块：L1 公链（Monad、以太坊）、Layer2、DeFi、NFT、GameFi、AI+Web3、稳定币。
    
2.  梳理行业竞争格局：对比各公链 TPS、Gas 成本、生态项目数量、代币经济模型。
    
3.  跟踪全球监管政策：欧美、亚洲各国加密法规，分析政策对市场的影响。
    
4.  输出周度 / 月度行业报告，总结板块热点、创新技术趋势。
    

2）链上数据研究（数据分析师）

1.  使用专业工具查询链上原始数据：Dune Analytics、Nansen、Flipside；
    
2.  分析核心指标：TVL 锁仓量、日活交互钱包、代币交易量、大户持仓、质押率、铸币销毁数量；
    
3.  分辨真实用户与机器人刷量，判断项目真实活跃度；
    
4.  撰写数据看板、数据分析短文，解释资金流入流出逻辑。
    

3）一级项目尽调（投资研究员，机构岗）

针对新项目做完整拆解评估：

1.  技术层面：合约是否开源、有无审计、底层逻辑有无漏洞；
    
2.  经济模型：代币分配、通胀速率、解锁周期、质押收益设计；
    
3.  团队与生态：团队背景、合作机构、社区规模、开发者数量；
    
4.  风险评估：梳理合约风险、政策风险、团队跑路风险；
    
5.  输出尽调报告，给机构提供投资 / 观望建议。
    

4）二级市场行情研究

跟踪加密货币大盘行情，解读涨跌背后的链上、消息面逻辑；快速产出突发事件点评（公链升级、大额盗币、政策出台）。

### 研究方向必备能力

1.  基础区块链知识：看得懂合约部署逻辑、Gas、链上交易、代币模型；
    
2.  数据能力：Excel 图表、简单 SQL 查询链上看板、数据归纳总结；
    
3.  文字能力：逻辑清晰，能输出长篇专业分析报告；
    
4.  加分项：能看懂基础合约代码、读懂审计报告。
    

### 入行门槛

-   入门门槛：中等，不需要写合约代码，但需要大量记忆行业概念、学会数据工具；
    
-   成长周期：3–6 个月能独立写简单数据分析；1 年以上可独立完成完整赛道报告。
    

### 优势与劣势

优势：工作偏脑力分析，不用大量社交、不用高强度写代码；行业认知全面，同时看懂技术、市场、资金逻辑；

劣势：纯研究岗位数量少于技术和基础运营；熊市机构缩减预算，岗位收缩明显；薪资居中，上限低于合约开发。

### 适合人群

喜欢阅读、擅长数据分析、逻辑严谨、擅长总结归纳、对行业宏观发展感兴趣，不想高强度敲代码、也不擅长频繁社群社交的人。

Web3三大核心角色：研究、运营、开发 + 方向选择指南

✅核心定位

团队的认知大脑、策略参谋、信息分析师，负责看懂赛道、协议机制、代币经济、宏观周期，输出研究结论支撑产品、融资、投资与决策。

✅主要工作

1\. 赛道/协议深度拆解：DeFi、NFT、GameFi、L2、RWA、ZK等底层机制、代币模型、经济博弈、风险分析

2\. 数据研究：链上数据、TVL、交易量、持仓地址、MEV、合约行为、竞品对标

3\. 撰写研报、周报、白皮书摘要、治理提案、投研报告、行业复盘

✅必备技能

• 能力：逻辑建模、定量/定性分析、信息溯源、深度写作、PPT复盘、基础Python数据分析

• 加分：金融/经济背景、代码基础、英文阅读、DAO治理理解

✅性格&适配特质

• 适合：擅长深度思考、长期阅读、挖掘底层逻辑、做复盘写作、抗压做周期研判、耐心做数据验证

• 不适合：坐不住、讨厌长篇报告、不喜欢慢节奏深度研究、追求高频即时反馈

✅发展路径

研报研究员 → 赛道主管研究员 → 投资分析师/VC研究员/协议战略/DAO治理顾问

三、Web3 开发（Development / Engineering）

✅核心定位

团队的技术基石、产品建造者，负责把协议/产品逻辑变成可运行、安全、可用的链上代码与应用，保障产品功能、性能与安全。

✅主要工作

一、什么是 Monad

核心定义

Monad（单子）是函数式编程里的一种抽象数据类型，本质是一个带类型构造器 + 2 条核心规则的包装容器

用来解决：顺序副作用、异步流程、错误处理、空值、状态传递这类纯函数不好处理的问题，保证代码可组合、可链式调用。

3个组成要素

1\. 类型构造器 M：把基础类型 T 包裹成 Monad 类型（例：Maybe<T>、Promise<T>、Either<T,E>）

2\. unit / of / pure 函数：把普通值 T 包装进 Monad 容器 M

// JS示例

Promise.of(5) // 把普通值包装进Promise Monad

3\. bind / flatMap / chain 函数：接收一个返回 Monad 的函数，做嵌套展开、链式运算，避免回调地狱

三大 Monad 定律（必须遵守）

1\. 左单位元：pure(x).bind(f) === f(x)

2\. 右单位元：m.bind(pure) === m

3\. 结合律：m.bind(f).bind(g) === m.bind(x => f(x).bind(g))

常见例子

• Maybe/Option：处理空值 null/undefined，防止报错

• Either/Result：做错误处理（成功/失败分支）

• Promise：异步流程 Monad

• IO：封装读写、日志等副作用操作

• State：封装可变状态

一句话记忆：Monad = 可链式运算、遵守固定规则的上下文包装器，用来优雅处理副作用和复杂流程

三、数据库与文件存储

1\. 文件存储（File Storage）

• 本质：以原始二进制文件（txt/图片/视频/文档/备份）形式，保存在磁盘/对象存储（S3、OSS、本地硬盘、NAS）

◦ 特点：保存非结构化/大体积数据；直接读写原始字节；无固定表结构、无SQL；适合静态资源、媒体、大附件、归档备份

◦ 优点：成本低、适合超大文件、版本存档、静态资源分发

◦ 缺点：难以检索、事务差、并发读写易冲突、无索引、一致性差，不适合高频查询/账务数据

◦ 典型：对象存储OSS/S3、NAS、本地磁盘、CDN资源

2\. 数据库（Database）

• 本质：结构化/半结构化数据管理系统，带索引、查询引擎、事务机制（SQL/NoSQL）

◦ SQL（MySQL/PostgreSQL）：表结构、ACID事务、强一致性、适合账务、用户信息、订单、核心业务数据

◦ NoSQL（MongoDB/Redis）：文档/键值/列式，高并发、灵活结构，适合缓存、实时数据、海量非结构化业务数据

◦ 特点：支持索引检索、复杂查询、事务、并发控制、数据约束，保证正确性，适合业务核心状态

◦ 缺点：存超大媒体文件效率差、成本偏高

3\. 最佳搭配方案（现代应用）

• ✅ 数据库：存元数据、业务状态、用户信息、订单、关系数据（小结构化数据）

◦ 例：图片url、视频id、商品信息、余额、用户资料

• ✅ 文件/对象存储：存原始大文件：原图、视频、音频、备份、静态资源

• ✅ 缓存（Redis等）：加速高频查询

核心区别

• 文件存储：存内容本体，主打大容量静态数据

• 数据库：存索引/元数据/业务状态，主打可检索、可事务的动态业务数据

一、区块链基础概念

区块链是Web3的底层技术基石，理解它是入门的第一步。

​核心定义：区块链本质上是一个去中心化的、公开的、不可篡改的分布式数据库。它由一系列按时间顺序连接的数据块（区块）组成，每个区块都包含了一定时间内的交易信息 。

​关键特性：

​去中心化：网络中没有中心服务器，所有节点（参与者的电脑）地位平等，共同维护数据，避免了单点故障和中心化控制 。

​不可篡改：一旦数据被记录，几乎无法修改。这是通过密码学哈希函数和链式结构共同保证的 。

​透明公开：链上的所有交易记录对所有节点公开，任何人都可以查询和验证 。

​公链、私链与联盟链：

​公链：完全开放，任何人都可以参与，如比特币、以太坊 。

​私链：由单一组织控制，权限受限，常用于企业内部 。

​联盟链：由多个组织共同管理，介于公链和私链之间，常用于商业合作 。

​Web3 vs Web2：Web2是中心化的互联网，用户数据由大公司控制；而Web3是去中心化的互联网，用户拥有自己的数据和身份，实现了从“平台拥有”到“用户拥有”的范式转变

二、以太坊概览

以太坊是Web3生态中最重要的公链平台，被誉为“世界计算机”。

​什么是以太坊：与比特币主要作为数字货币不同，以太坊是一个可编程的区块链平台。其核心创新是引入了智能合约，允许开发者在链上构建和运行去中心化应用（DApps） 。

​核心概念：

​智能合约：一段部署在区块链上的自动执行代码，一旦条件触发，合约条款就会自动执行，无需第三方介入 。

​以太币（ETH）：以太坊网络的原生加密货币，用于支付交易手续费和计算服务 。

​Gas机制：在以太坊上执行操作（如转账、部署合约）所需支付的费用，用以补偿网络验证者的计算成本 。

​以太坊虚拟机（EVM）：一个去中心化的全球计算机，负责执行智能合约代码，确保所有节点运行结果一致 。

​关键升级：

​从PoW到PoS（The Merge）：2022年，以太坊完成了“合并”，共识机制从工作量证明（PoW）转向权益证明（PoS）。这使能耗降低了99%以上，并提升了网络的可扩展性和安全性 。

​Layer 2 扩展方案：为了解决主网拥堵和高Gas费问题，出现了Layer 2（第二层）解决方案，如OP Rollup和ZK Rollup，它们在主链之外处理交易，再将结果提交到主链，大幅提升了交易速度和降低了成本 。

三、行业赛道全览

Web3的应用场景非常广泛，形成了多个核心赛道 。

​DeFi（去中心化金融）：旨在重塑传统金融体系，提供去中心化的借贷、交易、稳定币等服务。代表项目有Uniswap（去中心化交易所）、Aave（借贷协议） 。

​NFT（非同质化代币）：代表独一无二的数字资产所有权，广泛应用于数字艺术、收藏品、游戏道具等领域。代表项目有CryptoPunks、Bored Ape Yacht Club 。

​DAO（去中心化自治组织）：一种由社区成员共同管理的组织形式，通过智能合约和投票机制进行决策，实现了组织的去中心化治理 。

​GameFi与SocialFi：将区块链技术融入游戏和社交平台，赋予玩家和用户经济价值，实现“边玩边赚”或“边社交边赚 ”。

​基础设施：包括数据存储（如IPFS）、预言机（如Chainlink，为区块链提供外部数据）、跨链桥等，为整个生态提供底层支持 。

四、Web3工作方式

Web3的工作方式与传统互联网有很大不同，强调去中心化、远程协作和社区驱动

​必备工具与平台：

​沟通协作：Discord、Telegram、Twitter是Web3社区的核心沟通工具 。

​开发工具：GitHub用于代码托管和协作，Hardhat、Foundry等是常用的智能合约开发框架 。

​钱包：MetaMask是最流行的浏览器钱包，用于与DApp交互和管理数字资产 。

​远程协作最佳实践：Web3项目通常采用全球分布式团队，异步沟通、文档驱动和高度自治是常见的工作模式 。

​社区文化与行业黑话：Web3社区文化开放、透明，强调贡献和所有权。了解“GM/GN”（早安/晚安）、“WAGMI”（我们都会成功）、“DYOR”（自己做研究）等黑话有助于融入社区

​岗位全景：Web3的岗位分为技术岗和非技术岗 。

​技术岗：包括智能合约开发（需精通Solidity）、前端/后端开发、安全审计、区块链底层开发等 。

​非技术岗：包括社区运营、内容创作、市场推广、产品经理、数据分析等 。
