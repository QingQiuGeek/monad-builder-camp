1\. Web3 实习怎么找

Web3 现在行情一般，实习岗位没有以前好找，尤其是技术岗和运营岗都更依赖 内推 / 社群 / 线下活动。

比较有效的方式：

TG 求职群 / Web3 招聘群

可以先去观察岗位 JD，了解不同岗位到底在做什么。

线下活动 / 黑客松 / 会议

很多机会来自线下认识的人。Web3 圈子很看重信任，见过面、一起做过项目、在社群里活跃过，更容易被推荐。

内推比海投更有效

但内推的前提是自己要有项目、能力或个人标签，不然别人没有理由推荐。

2\. Web3 运营岗需要什么

运营不是单一岗位，里面可以分很多方向：

用户增长

负责拉新、转化、提升用户数量。

社群运营

维护群聊氛围、组织活动、提高用户活跃度。

活动运营

策划线上线下活动，比如 AMA、分享会、任务活动、空投活动。

内容运营

做 Twitter / 小红书 / 抖音 / TG 内容，输出项目观点或行业内容。

对新人来说，可以先从 自己起号 开始：

找一个定位

持续发内容

分享 Web3 / AI / 项目学习记录

有一点流量或内容积累后，可以写进简历

简历里可以写：

自己运营过的账号

做过的内容数据

参加过的活动

社群贡献

黑客松经历

项目经历

对 Web3 某个方向的理解

3\. Web3 技术路线

技术方向可以先从基础开始：

Solidity 基础语法

先学会智能合约怎么写。

DApp 开发

学习前端如何连接钱包、调用合约、完成链上交互。

项目实操

Web3 更看重项目经验，不只是理论。学校课程如果只讲哈希算法、比特币结构，但不做项目，对就业帮助有限。

比较重要的是：

能不能做出东西，而不是只会背概念。

4\. Web3 找工作的核心逻辑

Web3 圈子比较小，很多机会不是公开投简历来的，而是通过：

社群混脸熟

线下认识人

黑客松组队

朋友推荐

助教 / 老师 / 同学介绍

持续输出内容

所以求职重点不是单纯投简历，而是建立信任和可见度。

可以理解为：

能力 + 项目 + 圈子曝光 = 更容易拿到机会

5\. 个人 IP / 内容输出的重要性

Twitter、小红书、抖音、TG 都可以成为展示自己的地方。

内容方向可以根据个人定位来做：

技术型定位

分享开发过程、合约学习、DApp 项目、踩坑记录。

运营型定位

分享活动复盘、社群增长、项目观察、用户增长思考。

Research 型定位

分享项目分析、赛道观察、数据分析、Web3 趋势判断。

学生成长型定位

分享从 0 学 Web3、参加训练营、做项目、找实习的过程。

重点是先让别人知道：

这个人正在认真进入 Web3，并且有持续产出。

6\. Web3 安全意识

新人很容易被钓鱼网站骗，尤其是空投、领福利、钱包授权类网站。

常见风险：

点陌生链接

连接钱包

授权恶意合约

钱包资产被自动转走

U 卡 / 礼品卡被盗

交易所、邮箱、身份信息被关联攻击

所以新人必须注意：

陌生链接不要点，钱包不要乱连，授权一定要看清楚。

Web3 里钱一旦被链上转走，追回非常困难。

7\. 区块链专业和就业现实

即使是区块链专业，最后真正做区块链相关工作的人也不多。

很多人会转向：

普通程序员

AI

销售

主播

考研

考公

其他互联网岗位

原因是区块链行业岗位本身不多，而且对项目经验、圈子资源和实战能力要求比较高。

所以学历或专业不是最关键，真正重要的是：

项目经验、实操能力、社群连接和持续学习能力。

8\. 黑客松项目案例：喵星智能体

他们之前做过一个 AI + Web3 + 宠物硬件 项目，叫“喵星智能体”。

核心想法：

通过摄像头、声音识别、AI 分析猫咪行为和叫声，判断猫咪需求，比如饿了、想吃东西、状态异常等。

项目流程大概是：

采集猫咪声音 / 行为数据

通过摄像头、麦克风等设备收集信息。

AI 分析猫咪意图

把猫叫声转成声纹向量，再结合 RAG 和模型判断猫咪可能想表达什么。

多轮确认需求

如果模型不确定，会继续通过互动判断猫咪需求。

主人审批

当系统判断猫咪想买东西，比如猫粮或零食，会推送给主人确认。

链上支付 / 存证

通过预言机和智能合约，把审批、支付、履约流程连接起来。

9\. 这个项目为什么要用区块链

有人问：这个项目用互联网也能做，为什么要用区块链？

可以理解为，区块链在这里不是为了炫技，而是用于：

支付流程

链上存证

订单记录

权限审批

透明履约

多方共享可信数据

不过这也提醒一个点：

Web3 项目不能为了上链而上链，必须说明区块链解决了什么现实问题。

10\. 对学生最有用的启发

Web3 找机会不能只靠投简历。

更现实的路径是：

先学习基础 → 做一个项目 → 参加社群 / 黑客松 → 输出内容 → 认识人 → 获得内推或合作机会。

对于学生来说，最应该积累的是：

一个能讲清楚的项目

一段持续学习记录

一个内容账号或作品集

一些社群参与经历

一点行业方向判断

基础的安全意识

1️⃣会议总结：普通开发者如何进入以太坊协议层

**主讲人**：JackCC（EPF 第7期华语区唯一入选者，全球仅10人）

**会议时间**：2026-07-07 20:03

###一、核心观点

\- **以太坊协议层 ≠ 智能合约/DApp 开发**：协议层关注的是以太坊底层机制——EIP 提案研究、客户端实现、共识层/执行层规范、扩容（如 Gas 降低、零知识证明）等，而非上层应用。

\- **以太坊生态的核心竞争力是开发者社区**：以太坊是全球开发者数量最多的公链，基金会通过 EPF 等项目吸引新人参与，体现去中心化精神。

\- **EPF 面试不看算法八股，看重热情和研究潜力**：不会考手撕算法或密码学难题，更多聊你对以太坊的了解、对哪些 EIP 感兴趣、是否有持续贡献的意愿。

\- **以太坊技术更新极度保守谨慎**：一个 EIP 提案从提出到上线客户端可能需 1-2 年、多轮 review，作为承载全球大量资金的系统不可能激进跟进新技术（如 AI agent）。

\- **协议层不适合功利求职者**：大部分人是以副业形式参与，经济回报不确定；如果目标是稳定高薪，Web2 路线更确定。

\- **英语是 Web3 的"第一编程语言"**：面试全英文、日常讨论技术英语、国际化协作场景多，但不需要雅思口语级别，能清晰表达想法即可。

\- **多客户端、多语言是以太坊去中心化的基石**：不能让单一语言/客户端占据太大份额，防止单点故障影响整个网络。

\### 二、关键事实

\- EPF（Ethereum Protocol Fellowship）：面向所有对以太坊内部机制感兴趣的人，分为 permissioned 和 permissionless 两种，未获许可也可参加，最终看项目成果。

\- JackCC 非计算机科班出身（电子信息背景，研究生方向深度学习），擅用 Python，从零到能看懂 GitHub 讨论大约需要 2-3 个月。

\- 协议层 Research 方向主要围绕 EIP 提案展开，如近期 EIP-7688 将 EL-CL 连接层数据结构替换为 SSZ。

\- 以太坊基金会正在缩减，出现了新的组织如 ETHLabs（5 位以太坊研究员发起，获 5 亿美金支持）。

\- EPF 中有很多未获许可但已工作 10 年以上的资深开发者，他们不需要资助仍在贡献。

\- 有学员通过 EPF 项目后以实习生身份进入执行层规范团队，说明存在实习机会，但机会较少。

\- EPF 参与者中印度人非常多，印度程序员英语表达更自信。

\- EPF Discord 链接：[https://discord.gg/8RPnPGEQtJ；JackCC](https://discord.gg/8RPnPGEQtJ；JackCC) 的 X：[https://x.com/JackCC60。](https://x.com/JackCC60。)

\### 三、可执行建议

\- **入门路径**：

\- 先通读以太坊 EIP 提案和 GitHub 代码库（全部开源），结合 Python 版规范代码上手（难度较低）

\- 推荐学习计算机网络和数据结构作为基础知识储备

\- 准备好英文阅读和基本口语能力，优先掌握技术讨论场景下的专业英语

\- **参与策略**：

\- 不要一开始就追求"正式工作"，先以兼职/兴趣身份持续在一个方向上贡献（提 PR、参与讨论）

\- 持续跟进至少半年到一年，做出他们当前关注方向的实际成果，自然会有人主动联系

\- 申请 EPF 时准备一份聚焦协议层工作的简历/项目集，放在 GitHub 上

2️⃣会议总结：Co-Learning 第二次分享

**会议主题**：用 AI Coding 构建 Web3 Mini Demo — 从合约到前端的完整闭环

**分享人**：ETH Coooder / 41

**日期**：2026-07-07 20:05-21:18

\### 一、核心观点

\- **AI Coding 是 Web3 开发的新范式**：用 Codex/Cursor 等工具，无需 Remix 手动操作，即可完成合约编写、编译、部署到测试网的全流程，开发效率大幅提升，甚至非技术人员也可参与。

\- **Remix 仍有学习价值**：初学者应先通过 Remix 走一遍完整流程（编写→编译→部署→获取 ABI），理解各环节后再转向 AI Coding，否则“完成了任务但不知道做了什么”。

\- **私钥安全是 Web3 生存底线**：私钥/助记词代表钱包所有权，泄露即资产归零`.env` 文件绝不能上传 GitHub，中转站 API 有偷私钥风险，开发钱包和真实钱包必须严格隔离。

\- **智能合约 + 前端 = Web3 Demo 的最小闭环**：ABI/JSON 文件是合约与前端之间的接口契约，前端通过它调用链上合约。一个 Mini Demo 0 只需一个合约 + 一个前端即可。

\- **区块链应被理解为后端的补充而非替代**：受 Gas 费和延迟制约，不能将所有传统后端逻辑上链；实际系统通常是传统后端与智能合约互为补充。

\- **Co-Learning 定位是互助交流，不是单向授课**：鼓励自主学习后用 AI 探索，遇到问题来共学时间或群里提问。

\---

\### 二、关键事实

\- 三周 Share 节奏：第一周打理论基础，第二周对应课程设计，第三周请嘉宾分享产品化/商业化思路。

\- 推荐钱包：\*\*Rabby\*\*（测试网模式清晰，比 MetaMask 体验好）；Phantom 兼容 Solana 等多链。

\- AI Coding 工具推荐：Codex（主讲人使用）；Claude Code 也可；Plus 套餐日常够用，Pro 适合专职开发。

\- ABI/JSON 文件由合约编译自动生成，相当于接口文档，前端需据此调用合约函数。

\- 部署合约消耗 Gas（测试网上演示消耗约 0.02 ETH 测试币）。

\- Monad 测试网浏览器地址`testnet.monadscan.com`。

\- BuildAnything 学习平台课程需按解锁顺序学习，可自定节奏。

\- 打卡时间零点是按用户所选时区计算，可在设置中调整。

\- Claude 封号问题：注意节点和付款方式一致性；Codex 有概率因网络环境触发风控；建议开启 2FA 降低风控概率。

\- 曾有黑客松参赛者因 GitHub 上泄露 `.env` 中的私钥，奖金刚到账就被转走。

\---

\### 三、可执行建议

**对参与者：**

\- 注册一个开发专用钱包（Rabby 推荐），与真实资产钱包彻底隔离

\- 按 Remix 教程走一遍完整部署流程（编写→编译→部署→获取 ABI），理解每一步的意义

\- 之后切换到 AI Coding 工具（Codex/Claude Code），尝试用自然语言描述需求让 AI 生成并部署合约

\- 完成 Mini Demo 0：写一个合约 + 做一个前端页面，前端能调用链上合约

\- `.env` 放私钥时务必加入 `.gitignore`，不上传 GitHub；不要用中转站类平台操作含私钥的项目

\- 零代码基础者可从 B 站搜“编程是什么”类 20 分钟入门视频，或让 AI 用熟悉的知识举例解释代码

\- Monad 生态项目 Gallery 可在 Monad 官网上找到，研究 Uniswap 等开源项目学习产品设计

\- 有问题可在群里提问或私聊助教/老师，不需要等到 Co-Learning 时间

**对组织者：**

\- 后续可将 Remix 部署教程以图文形式发到群里

\- Co-Learning 时可在开头用 2-3 分钟快速提及当日技术内容的商业应用场景，帮助运营同学建立概念框架

\- 建议提供录屏回放，方便不同时区或跟不上节奏的同学课后复习
