## 今日理论核心学习笔记

### 模块 1 Monad 核心架构：不止 “更快”，底层机制全面革新

1 基础性能指标（核心竞争力）

-   吞吐量 10000 TPS、区块间隔 400ms、交易最终确认 800ms，完全兼容 EVM，原生支持 Solidity；
    
-   并行执行 + MonadBFT 共识 + RaptorCast 传输 + MonadDB 存储五大核心优化，在不牺牲去中心化前提下解决以太坊拥堵、Gas 昂贵痛点。
    

2 Monad VS 以太坊关键底层差异

1.  合约容量限制
    
    以太坊合约最大 24KB，Monad 提升至 128KB，初始化字节码上限 256KB，大型 NFT、DeFi 合约无需拆分部署；
    
2.  Gas 计费核心区别（最关键开发差异）
    
    以太坊：按**实际消耗 Gas**扣费；
    
    Monad：直接收取用户设置的**Gas Limit 全额费用**，设计目的抵御 DoS 攻击，异步执行模式下防止恶意交易占用区块资源；
    
    计算公式：实际扣费 = Gas Limit × Gas 单价；
    
    ⚠️ 开发风险：合约执行失败、交易回滚，依然会扣除全部 Gas Limit 费用，前端必须固定已知操作的 Gas 值，禁止依赖 eth\_estimateGas 自动估算。
    
3.  交易与内存池机制
    
    无全局公共 Mempool，交易仅转发给少数下一区块出块节点，降低全网广播带宽压力；不支持 EIP-4844 Blob 交易；
    
4.  EIP-7702 账户委托特殊规则
    
    EOA 开启账户委托后，账户余额不得低于 10 MON，且禁止使用 CREATE/CREATE2 部署合约；解除委托后无余额限制；
    
5.  历史数据存储策略
    
    高 TPS 导致海量区块数据，全节点不存储完整历史状态，高频查历史事件必须使用索引器（Allium/Envio/Goldsky 等），不能反复调用 eth\_getLogs；
    
6.  预编译与 Opcode 优化
    
    重定价各类操作码，新增 secp256r1 预编译合约，原生支持 WebAuthn/Passkey 链上签名验证。
    

3 Monad Gas 完整规则

-   采用兼容 EIP-1559 双层定价：Base Fee 基础费 + Priority Tip 矿工小费；
    
-   最低基础费固定 100 MON-gwei；单区块总 Gas 上限 200M，单笔交易 Gas 上限 30M；
    
-   基础费调整算法：上涨慢、下跌快，避免区块空闲时 Gas 居高不下，提升用户交互体验；
    
-   开发最佳实践：转账、铸币等 Gas 消耗固定的操作，前端硬编码 Gas Limit，减少 RPC 请求延迟，规避估算失败导致高额 Gas Limit 漏洞。
    

4 高性能 DApp 开发标准实践

1 RPC 与读取优化

-   批量 Multicall 聚合多笔读取请求，减少 RPC 轮询；
    
-   高频历史数据、事件查询必须接入链上索引器，避免频繁 eth\_getLogs 拖慢程序；
    
-   批量并发发送多笔交易，本地维护 Nonce 计数器，防止交易序号错乱。
    
    2 数据存储分层
    
    链上仅存储核心资产、业务状态；图片、长文本等大容量资源使用 IPFS / 中心化对象存储；索引器同步事件至数据库供前端快速查询。
    
    3 部署与运维选型
    
    静态资源：Vercel/CloudFront；后端服务：AWS Lambda、容器 ECS；链上数据索引：Envio、Goldsky、The Graph Subgraph。
    

5 10000TPS 带来的全新产品可能性

以太坊受限于低 TPS，链游、高频 DEX、小额微支付、批量 NFT 铸造、实时 RWA 产品无法流畅运行；

## 今日理论核心学习笔记

### 模块 1 Solidity 基础核心概念

1.  语言定位
    
    Solidity 是面向 EVM 的静态类型高级编程语言，语法借鉴 C/Python/JS，专门用于编写智能合约；0.8 及以上版本内置整数溢出自动校验，是开发最低安全基线。
    
2.  合约标准文件结构
    

-   SPDX 许可证声明：开源合规标识，必须置于文件首行；
    
-   Pragma 编译版本声明：限定编译器适配区间，避免版本兼容报错；
    
-   Contract 主体：合约业务载体，包含状态变量、构造函数、自定义方法、事件等；
    

3.  核心基础组件
    

-   状态变量：永久存储在区块链，消耗 Gas；
    
-   构造函数`constructor`：合约部署时仅执行一次，用于初始化参数；
    
-   函数可见性修饰：`public`任意账户调用 / `external`仅外部 EOA 调用（Gas 更省） / `internal`合约 & 子合约可用 / `private`仅当前合约；
    
-   状态修饰：`view`只读不消耗 Gas / `pure`无读写完全免费 / `payable`可接收原生 MON；
    
-   Event 事件：链上日志载体，前端索引器通过监听事件抓取合约数据；
    

4.  关键概念对应关系
    

-   合约源码：人类可读 Solidity 代码；
    
-   ABI（应用二进制接口）：源码编译后生成 JSON 接口，定义所有函数名、入参、返回值，钱包 / 前端靠 ABI 识别合约调用方式；
    
-   合约地址：部署交易生成唯一 0x 地址，是链上合约唯一标识；
    
-   Read 函数（view/pure）：仅读取数据，无需签名、不产生交易哈希；
    
-   Write 函数（修改状态 /payable）：改变链上存储，钱包签名广播，生成唯一 txHash（交易哈希），区块浏览器可永久查询执行记录。
    

### 模块 2 Remix IDE 全功能开发流程

Remix 是浏览器端一站式 EVM 开发 IDE，无需本地环境，适配 Monad 等全部 EVM 兼容链，核心四大面板：

1.  File Explorer：新建 / 管理.sol 合约文件；
    
2.  Solidity Compiler：选择匹配版本编译，自动检测语法报错；
    
3.  Static Analysis（静态扫描）：Slither 工具检测重入、权限缺失等安全漏洞；
    
4.  Deploy & Run Transactions：本地虚拟机调试 / 连接 MetaMask 部署至测试网。
    
    部署核心环境区分：
    

-   Remix VM：本地模拟链，无真实测试币，仅调试语法；
    
-   Injected Provider - MetaMask：关联课程专用 Monad 测试钱包，真实上链生成交易。
    

### 3 Monad 测试网合约部署标准流程

1.  前置条件：MetaMask 已添加 Monad Testnet（Chain ID:10143）、持有水龙头领取 MON 测试代币；
    
2.  在 Remix 编写极简 Gmonad 问候合约，指定编译器版本匹配；
    
3.  编译通过，无警告后切换部署环境为 MetaMask 注入钱包；
    
4.  填写构造函数初始化入参，点击 Deploy，钱包确认 Gas 费用；
    
5.  交易上链后生成部署 txHash，面板显示已部署合约地址；
    
6.  交互：调用无消耗`greeting`读函数查看文本，调用`setGreeting`写函数发起新交易，生成新 txHash；
    
7.  复制合约地址、全部交易哈希至 Monadscan 浏览器核验区块、Gas、事件日志。
    

### 4 AI 辅助合约开发规范（核心学习目标）

1.  AI 使用逻辑：仅做代码生成辅助，禁止直接复制部署；

## 今日理论核心学习笔记

### 模块 1：钱包底层逻辑与安全分级使用

1.  钱包本质
    
    MetaMask 属于浏览器扩展 EOA 外部账户钱包，仅作为密钥管理工具，不托管资产；资产永久存于区块链，私钥 / 助记词唯一所有权凭证，丢失无法找回。
    
    核心区分两类钱包：
    

-   主力钱包：存放自有真实资产，不参与测试、空投、陌生 DApp 交互；
    
-   课程实验钱包：本次新建独立钱包，仅用于 Monad 测试网实操，无真实资金风险，隔离资产避免钓鱼损失。
    

2.  关键密钥体系
    

-   助记词（BIP39）：12/24 单词，钱包唯一恢复凭证，**禁止截图、云端存储、聊天发送**，仅手写纸质离线存放；
    
-   私钥：助记词派生字符串，等同于钱包全部资产，绝不对外分享；
    
-   公钥 / 0x 地址：公钥加密生成公开收款地址，可随意分享，无资产风险。
    

3.  账户分类
    

-   EOA 外部账户：MetaMask 对应账户，可主动发起交易，带 Nonce 自增计数器；
    
-   CA 合约账户：无私钥，仅能被 EOA 调用，部署后生成固定链上地址。

### 模块 3：交易完整结构与 Gas 计费体系

1.  一笔标准交易核心字段
    
    `from`发送地址、`to`接收 / 合约地址、`nonce`交易序号（防重复）、`value`转账代币数量、`gasLimit`最大算力、`maxFeePerGas`最高单价、`maxPriorityFeePerGas`矿工小费、`input`合约调用数据、签名、唯一交易哈希 txHash。
    
2.  EIP-1559 计费规则
    
    总手续费 = 消耗 Gas ×（Base Fee 基础销毁费 + Priority Tip 小费）
    
    Base Fee 随区块拥堵自动浮动，交易完成后销毁，降低代币通胀；小费用于激励验证者优先打包。未使用 Gas 全额原路退回账户。
    
3.  交易生命周期
    
    钱包签名→全网广播→进入交易池→验证者打包上链→区块确认→浏览器永久存证，交易不可逆。
    

### 模块 4：境内合规硬性红线（实操必守）

1.  监管底线
    
    境内禁止虚拟货币 ICO、IDO、现货交易、场外 OTC 换汇；测试网 MON 仅用于学习实验，无金融流通属性，严禁炒作、变现。
    
2.  从业者法律风险
    
    参与代币融资、多级返利链游、跨境虚拟币换汇，可能涉嫌非法集资、传销、非法经营、洗钱罪；技术开发人员参与非法项目同样认定共犯，无法以 “仅写代码” 免责。
    
3.  实习 / 项目筛选标准
    
    规避带有提现、返利、投资承诺、面向国内用户的链上项目，仅参与纯技术测试、开源学习类生态活动。
    

### 模块 5：高频网络攻击与实操防护规范

1.  钓鱼攻击（最高发风险）
    
    仿冒水龙头、假 Monad 官网、虚假空投链接、TG/Discord 私信诱导连接钱包签名，签名授权可直接转移钱包全部资产。
    
2.  木马剪贴板劫持
    
    恶意软件自动替换复制的钱包地址，转账前必须完整核对全部字符，不能仅对比前后几位。
    
3.  实操强制安全流程
    

-   钱包仅 Chrome 官方扩展商店安装，拒绝第三方安装包；
    
-   陌生链接不点，所有官方站点手动输入域名或收藏书签；
    
-   任何要求输入助记词 / 私钥的页面 100% 骗局；
    
-   实验钱包仅领取测试代币，不授权陌生合约；
    
-   重要账号开启 2FA 双重验证，密码独立不复用。

今日理论核心学习笔记

模块1：区块链底层体系 1. 基础定义 区块链是去中心化分布式账本，由有序区块串联构成，单区块包含批量交易数据+上一区块哈希数字指纹），依托全网多节点同步账本实现不可篡改、链上数据公开、地址匿名三大核心特性。 全网采用分布式记账模式，单一节点篡改区块无效，需要控制全网51%节点才能伪造数据，篡改成本极高。 2. 区块链三类分类（去中心化程度由高至低） - 公链：任何人可自由接入成为节点，链上数据全网可见，代表BTC、ETH、Monad，适用于加密货币、NFT公共存证； - 联盟链：需机构邀请审批才可加入，数据仅联盟内部开放，多用于银行、供应链协同； - 私链：由企业管理员审批准入，数据完全内部私有，用于企业审计、内部数据管理。 3. 区块链1.0代表：比特币（BTC） 采用PoW工作量证明共识机制，总量固定2100万枚主打价值存储；仅支持简单交易脚本，无法运行复杂应用，区块出块速度约10分钟，扩展性较弱。

模块2：以太坊（区块链2.0）核心架构 1. 核心创新 首创智能合约与EVM以太坊虚拟机，搭载图灵完备编程语言Solidity，可开发DeFi、NFT、DAO等各类DApp去中心化应用，定位全球共享分布式计算机，原生代币为ETH。 2. 核心升级The Merge 2022年完成合并升级，网络共识从PoW切换为PoS权益证明，用户质押32ETH即可成为验证者，网络能耗降低99.95%；验证者可获得区块奖励与Gas手续费，作恶质押资产会被销毁Slashing。 3. Gas燃料机制 Gas是链上计算服务费，Gas Limit为单笔交易最大消耗算力，Gas Price为单位算力单价；EIP-1559升级后拆分Base Fee基础销毁费+Tip矿工小费，网络拥堵时Gas价格自动上浮。 4. 两类链上账户 - EOA外部账户：由私钥控制，通过助记词备份恢复，对应MetaMask钱包地址，可主动发起转账、合约调用； - CA合约账户：部署智能合约自动生成地址，无私钥，仅能被EOA外部账户触发执行逻辑。 5. 以太坊分层扩容体系 - L1主链：底层结算层，保障全网最终安全； - L2二层Rollup：链下批量打包交易上传L1，大幅降低Gas成本，分为ZK-Rollup、Optimistic Rollup； 补充生态信息：Polygon zkEVM将于2026逐步下线，Polygon资源转向Polygon PoS侧链与AggLayer跨链基础设施； - Sidechain侧链：独立运行区块链，通过跨桥与L1互通，典型代表Polygon PoS。 6. 配套升级标准EIP：以太坊改进提案，用于网络迭代升级，如EIP-1559、EIP-4844。
