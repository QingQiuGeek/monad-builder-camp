# **Day 2 学习笔记：工具准备与 Builder 身份**

## 一、工具准备：搭建远程工作基础设施

工具不是随机罗列，而是对应 Web3 工作的不同场景层。理解每层“为什么存在”比记住名字更重要。

| 场景层 | 工具 | 核心价值 |
| --- | --- | --- |
| 信息信号 | X (Twitter)、Telegram | X 是行业信息g广场，Telgram 是社群讨论阵地。 |
| 社区协作 | Discord | 频道结构与权限系统天然适配去中心化组织——理解频道治理逻辑比“会用”更重要。 |
| 代码与身份 | GitHub、MetaMask | GitHub 是技术身份证明。MetaMask 是链上身份。务必从官方渠道安装 MetaMask，克隆版钱包是资产被盗的常见入口。 |
| 文档知识 | Notion / Google Docs | 关键结论沉淀到文档，标注时区与预期响应时间。 |
| 效率倍增 | Cursor / Claude Code / Codex / ChatGPT | AI 不只是“帮写代码”，它重塑 Builder 工作方式：AI Learning（消化协议文档）→ AI Coding（辅助开发）→ Build Log（记录日志）。 |

## 二、Proof of Work：让学习过程成为资产

Proof of Work 是 Web3 中具颠覆性的概念——传统简历是自我声明，而 PoW 是**可验证的、公开留存的、累积式的**。你的 GitHub 记录、链上交互历史、社区贡献、公开笔记，构成比简历更有说服力的凭证系统。

常见的认知错误是“等我学好了再输出”。但 PoW 的逻辑恰恰相反：**输出本身就是学习的一部分**。公开记录踩过的坑、解决的问题，这些“过程性内容”的价值往往高于结论——它们展示的是你的思维方式和成长轨迹。在远程协作中，PoW 还有一个隐性功能：**降低信任成本**——你被信任的基础不是你说了什么，而是你做过什么，且这些需要被公开记录、可追溯。

## 三、远程协作核心习惯

-   **异步沟通优先**：不依赖实时回复，关键信息沉淀到文档/Issue，标注时区与响应预期。学会用文字完整表达意图。
    
-   **目标导向而非时长导向**：价值由产出定义，不是在线时长。与整个计划的“以输出倒逼输入”一脉相承。
    
-   **决策痕迹留存**：每次会议记录决策、负责人、截止时间，让不在场的人也能对齐认知。
    
-   **掌握行业语言**：DYOR、FOMO、FUD、L1/L2、EVM、Rugpull、RPC、ABI……这些不是门槛，而是高效沟通的压缩包。建议从今天起建立自己的术语表。

这三件事背后有一个统一的设计逻辑：**以输出倒逼输入**。阅读是输入，但如果没有产出作为锚点，阅读就会变成“看过即忘”。Build Log 的意义正在于此——它强迫你把模糊的感受转化为清晰的文字，把隐性的理解显性化。

## 二、Web3 是什么？— 从互联网的三次演进说起

### 2.1 Web1 → Web2 → Web3 的范式迁移

理解 Web3 最有效的方式，不是直接定义它，而是把它放在互联网演进的历史脉络中。

**Web1（约 1990-2005）** 是“只读互联网”。用户是纯粹的内容消费者——浏览网页、阅读信息，但几乎无法参与内容创造。这就像一张电子报纸：你能看，但不能写。

**Web2（约 2005-至今）** 是“可读写互联网”。社交媒体、博客平台、视频网站让每个人都能发声。但问题随之浮现：你创造的内容存储在平台的服务器上，平台随时可以删除你的账号、修改规则、拿走你的数据。Web3 实习手册用了一个精准的比喻——**“就像租房子，你可以装饰（发内容），但房东（平台）随时能收回钥匙（封号）”**。你拥有的是使用权，不是所有权。

**Web3** 的核心主张是“可读写 + 可拥有”。通过区块链技术，用户真正拥有自己的数据、资产和身份。手册的类比是：**“像自己买地盖房（数据自托管），用智能合约管理水电费（自动结算）”**。你不再是租客，而是产权所有者。

### 2.2 一个必须澄清的常见误解

Web3 实习手册特别指出：**Web3 ≠ Web 3.0**。这两个概念经常被混用，但它们指向完全不同的东西：

-   **Web 3.0** 是 Tim Berners-Lee 提出的“语义网”（Semantic Web）愿景，核心是让机器理解网页内容的含义——通过 RDF、OWL 等标准给数据贴上结构化标签，让搜索引擎、推荐系统更智能。它仍建立在中心化架构之上。
    
-   **Web3** 是以太坊联合创始人 Gavin Wood 在 2014 年提出的概念，核心是去中心化——用区块链、智能合约、加密货币重构互联网的信任层和价值层。
    

| 维度 | Web2 | Web 3.0（语义网） | Web3（去中心化互联网） |
| --- | --- | --- | --- |
| 控制权 | 平台垄断 | 部分开放 | 用户自治 |
| 数据存储 | 中心服务器 | 混合存储 | 区块链 / IPFS |
| 支付系统 | 信用卡 / 支付宝 | 集成支付 | 加密货币 |
| 典型技术 | JavaScript | RDF / OWL | 智能合约 |
| 代表企业 | 腾讯 / 阿里 | W3C / DBpedia | Uniswap / ConsenSys |

### 2.3 为什么选择 Web3？

Web3 实习手册前言给出了几个关键理由：DeFi（去中心化金融）重新定义金融服务，NFT 为数字创作者开辟新的商业模式，DAO（去中心化自治组织）探索全新的协作方式。Web3 正在各个领域带来从“平台中介”到“协议直连”的范式变化。

但手册同时保持了诚实的态度——**“虽然目前 Web3 由于种种原因饱受争议和质疑，甚至时常有负面新闻，但这并不是 Web3 的全部”**。这种不回避问题的立场比单纯画饼更有说服力：一个领域如果只能讲好故事而不能直面批评，那它的价值确实存疑；但如果它能解释清楚争议从何而来、哪些是技术成长的必经阵痛、哪些是真正的结构性缺陷，那才值得深入。

## 三、区块链基础概念 — 信任机器的运作原理

### 3.1 区块链是什么？

Web3 实习手册给出了一个严谨的定义：**区块链是一种去中心化的分布式账本技术，用于在网络节点之间安全、透明且不可篡改地记录事务数据。** 每条链由一系列按时间顺序相连的“区块”组成，每个区块内部包含多笔交易数据及元数据。

用费曼的方式拆解这个定义：

-   **账本**：就是一本记录“谁给谁转了什么”的记账本。
    
-   **分布式**：这本账本不是存在一个中心服务器上，而是复制了无数份，分散在网络中的每个节点上。每个节点都有完整的、相同的账本副本。
    
-   **去中心化**：没有一个“行长”或“管理员”能单方面修改账本。任何修改都需要网络中多数节点达成共识。
    
-   **不可篡改**：每个区块都包含前一个区块的哈希值（一种加密指纹），形成链条。如果有人试图篡改历史记录，后续所有区块的哈希都会变化，被全网立刻发现。
    

### 3.2 区块链如何运行？— 六步完整流程

手册描述了一条区块链从用户发起到最终确认的完整流程：

1.  **用户发起交易**：用户通过钱包应用发起转账、智能合约调用等操作。
    
2.  **交易广播**：交易信息被广播到整个网络中的各个节点。
    
3.  **节点验证**：网络中的矿工节点验证交易的合法性（余额是否足够、签名是否正确等）。
    
4.  **打包成块**：通过共识机制（如工作量证明 PoW 或权益证明 PoS），矿工将验证过的交易打包成新的区块。
    
5.  **链接上链**：新区块被添加到区块链上，更新全网的账本状态。
    
6.  **奖励发放**：成功出块的节点可获得协议奖励（如区块奖励 / 质押奖励）和交易手续费。
    

这个流程揭示了一个关键点：**区块链的安全性和可信性不是靠“信任某个机构”来保证的，而是靠“让作弊在经济上不划算”来保证的。** 篡改账本需要控制全网 51% 以上的算力或质押量，其成本远高于篡改所能带来的收益。

### 3.3 公链 vs 联盟链 vs 私链

区块链并非只有一种形态。手册用生活化的类比区分了三种类型：

-   **公链（Public Blockchain）= 公共公园**：任何人都可以自由进入、参与维护。规则由大家共同制定。比特币、以太坊都是公链。
    
-   **联盟链（Consortium Blockchain）= 多公司联合的董事会**：由多家机构共同管理一个共享数据库，只有联盟成员才能参与决策。适合供应链、金融协作场景。
    
-   **私链（Private Blockchain）= 私人俱乐部**：由单一组织控制，谁可以加入、谁能查看账本都由“老板”决定。适合企业内部管理。
    

| 类型 | 节点加入方式 | 数据可见性 | 管理模式 | 适合场景 |
| --- | --- | --- | --- | --- |
| 公链 | 任何人自由加入 | 所有人可见 | 去中心化 | 加密货币、公共存证 |
| 联盟链 | 需联盟成员邀请/审批 | 仅联盟成员可见 | 多中心化 | 供应链、金融协作 |
| 私链 | 由老板严格审批 | 仅内部可见 | 中心化 | 企业内部管理、审计 |

Web3 世界中我们主要打交道的是**公链**——它的开放性和无需许可性是整个去中心化生态的基础。

### 3.4 比特币：区块链的第一个应用

比特币是区块链技术的第一个成功应用，理解它有助于理解整个体系的激励逻辑。

手册提出了一个关键问题：**“去中心化和分布式网络很好，可是为什么会有人愿意提供这项服务呢？”** 答案是——因为他们能得到奖励：比特币。

这揭示了区块链运行的经济基础：节点服务提供商（矿工 / 验证者）通过提供计算和存储服务获得代币奖励。这些奖励由两部分组成：**协议奖励**（新生成的代币，即“区块奖励”）和**交易手续费**（用户支付的交易费用）。你经常听说的“燃料费”（Gas Fee）属于交易手续费的一部分，并不等于全部节点奖励。

比特币的局限性也很明显：每个区块打包需要约 10 分钟，交易确认较慢；每个区块存储数据有限；匿名性使其容易被黑产利用。这些正是以太坊等新一代区块链试图解决的问题。

## 四、以太坊概览 — 从“数字黄金”到“世界计算机”

### 4.1 以太坊是什么？

如果说比特币是“区块链 1.0”——专注于货币属性，那么以太坊就是“区块链 2.0”——通过智能合约和可编程性，把区块链从“账本”升级为“计算机”。

以太坊由 Vitalik Buterin 在 2013-2014 年间提出，目标是构建“下一代加密货币与去中心化应用平台”。它的核心创新是**智能合约（Smart Contract）**：存储在区块链上的可执行代码，能够在满足预设条件时自动执行操作，无需人工干预。这一特性使得以太坊不仅是数字货币的载体，更是构建 DeFi、NFT、DAO 等生态系统的基础设施。

截至 2025 年，以太币（ETH）是全球市值排名第二的加密货币，仅次于比特币。以太坊的原生代币 ETH 用于支付网络上的一切操作费用。

### 4.2 以太坊 vs 比特币：关键差异

| 维度 | 比特币（Bitcoin） | 以太坊（Ethereum） |
| --- | --- | --- |
| 目标与定位 | 去中心化数字货币，强调安全、稳定、稀缺性（总量 2100 万枚） | 去中心化平台，支持智能合约和 Dapps，定位“区块链 2.0” |
| 编程能力 | 脚本语言有限，仅支持简单交易验证逻辑 | 图灵完备编程语言（如 Solidity），可开发复杂智能合约 |
| 共识机制 | 工作量证明（PoW），矿工通过算力竞争记账权 | 权益证明（PoS），通过 The Merge 实现能源效率优化 |
| 交易速度 | 每 10 分钟生成一个区块，确认较慢 | 区块时间约 12 秒，确认更快 |
| 经济模型 | 总量固定，强调抗通胀 | 供应灵活，通过 EIP-1559 等机制可能呈通缩趋势 |

一个关键区别：比特币是“数字黄金”——储值为主；以太坊是“世界计算机”——运行应用为主。你用比特币“存”，用以太坊“用”。

### 4.3 The Merge：从 PoW 到 PoS

2022 年，以太坊完成了被称为 **The Merge（合并）** 的里程碑升级，从工作量证明（PoW）转向权益证明（PoS）。

-   **PoW（工作量证明）**：矿工通过消耗算力解题来竞争记账权，谁算得快谁出块。问题是能耗巨大——以太坊 PoW 时期的年耗电量与一个中等国家相当。
    
-   **PoS（权益证明）**：验证者通过质押 ETH 来获得出块资格，系统随机选择验证者。不再需要拼算力，能耗降低 99.95% 以上。作弊的惩罚是被罚没质押的 ETH——经济激励替代了算力竞争。
    

The Merge 之后，以太坊的年通胀率降至约 0.2%-0.8%，在交易活跃时甚至呈现轻微通缩趋势（因为 EIP-1559 机制会销毁一部分交易手续费）。

### 4.4 以太坊生态分层架构

以太坊生态系统由多层架构组成：

-   **L1（Layer 1，主网）**：以太坊基础链，提供安全性和共识。所有交易的最终结算都在 L1 完成。
    
-   **L2（Layer 2，二层扩展方案）**：如 Rollups（Optimistic Rollups、ZK Rollups），在链下处理大量交易，再将结果批量提交到 L1。大幅降低交易费用、提升吞吐量。
    
-   **侧链（Sidechains）**：独立运行的区块链，与以太坊兼容但有自己的共识机制。如 Polygon PoS 链。
    

这种分层设计解决了一个核心矛盾：**去中心化、安全性、可扩展性不可能同时最优化（区块链不可能三角）**。L1 优先保证去中心化和安全性，把可扩展性交给 L2 处理。

### 4.5 以太坊的三大核心机制

手册将以太坊的运作拆解为三个关键机制，这三个机制是理解所有链上操作的基础：

机制一：账户系统 — 你的数字身份

以太坊有两种账户类型：

-   **外部账户（EOA,Externally Owned Account）**：由私钥控制，就是你的“钱包账户”。由一对密钥（私钥 + 公钥）生成，私钥严格保密，公钥通过加密算法生成唯一地址（如 `0xAbc...123`）。EOA 可以主动发起交易。
    
-   **合约账户（CA,Contract Account）**：由智能合约代码控制，不可主动发起交易，只能被 EOA 触发。比如你部署一个 NFT 市场合约，区块链会生成一个 CA 地址。当你点击“购买 NFT”时，你的 EOA 向 CA 发送交易，CA 的代码自动执行出货逻辑。
    

每个账户包含四个关键字段：nonce（交易序号，防重放）、etherBalance（ETH 余额）、storageRoot（存储根哈希）、codeHash（代码哈希，EOA 为空，CA 为合约代码）。

机制二：Gas 模型 — 交易的燃料费

在以太坊上进行的任何操作都需要支付 Gas——就像开车需要消耗汽油一样。Gas 的计算公式：

-   **Gas Limit**：你愿意为这笔交易支付的最大 Gas 量（类似于“最多跑多少公里”）。
    
-   **Gas Price**：每单位 Gas 的价格（类似于“每公里油费”）。
    

举例：假设 Gas Limit 设为 150,000，Gas Price 为 100 Gwei，总费用 = 150,000 × 100 Gwei = 0.015 ETH。如果实际消耗 120,000 Gas，剩余的 30,000 Gas × 100 Gwei = 0.003 ETH 会退还给你。

**EIP-1559 升级后的 Gas 费结构**发生了重要变化。Gas Price 被拆分为两部分：

-   **Base Fee**：基础费，由网络根据拥堵程度自动调节，**这部分会被销毁**（从流通中永久移除）。
    
-   **Priority Fee（小费）**：给验证者的小费，出价越高交易被打包的速度越快。
    

Gas 的存在有两个目的：一是激励节点维护网络运行，二是防止恶意用户通过无限循环代码攻击网络（每一步计算都要花钱，攻击成本极高）。

机制三：以太坊虚拟机（EVM）— 代码的执行引擎

EVM（Ethereum Virtual Machine）是以太坊的“大脑”，是专门用来运行智能合约的虚拟计算机。它运行在每个节点上，确保整个网络在处理代码时结果一致、可信任。

EVM 的核心意义在于**确定性**：同一段智能合约代码在所有节点上执行后必须得到完全相同的结果。这是去中心化共识的前提——如果不同节点算出不同结果，就无法达成共识。为此，EVM 被设计为一个完全隔离的沙箱环境，合约代码无法访问网络、文件系统或其他进程，只能读写链上状态。

## 五、链上身份：钱包、地址、交易

### 5.1 钱包（Wallet）

钱包不是“存钱的容器”，而是**管理私钥的工具**。它的核心功能是：安全地存储你的私钥，并用私钥对交易进行签名。钱包里存的不是你的代币——代币永远在区块链上；钱包存的是你控制这些代币的“钥匙”。

钱包分为两类：

-   **热钱包**：联网状态（如 MetaMask 浏览器插件、手机 App），方便但风险更高。
    
-   **冷钱包**：离线设备（如 Ledger、Trezor 硬件钱包），安全性高但不方便频繁操作。
    

### 5.2 地址（Address）

地址类似银行账号，是你在链上的公开身份标识。它由公钥经过哈希运算生成，以 `0x` 开头，如 `0xAbc...123`。地址是公开的——你可以把它分享给任何人来接收转账。但与地址绑定的私钥必须绝对保密：**谁拥有私钥，谁就拥有该地址下的所有资产。**

### 5.3 交易（Transaction）

交易是在链上更改状态的行为——转账、调用智能合约、部署合约都是交易。一笔交易从发起到最终确认的完整流程：

1.  用户在钱包中填写交易参数（目标地址、金额、Gas Limit、Gas Price）。
    
2.  钱包用私钥对交易进行数字签名。
    
3.  签名后的交易广播到以太坊网络。
    
4.  节点验证交易合法性（签名是否正确、余额是否足够、Nonce 是否连续）。
    
5.  验证者将交易打包进新区块。
    
6.  区块上链，交易被确认。
    

一旦交易被打包进区块并上链，就**不可逆转**。这是 Web3 与 Web2 的一个根本区别：Web2 里你转账转错了可以找客服撤回，Web3 里没有任何人能帮你撤回一笔已确认的交易。

### 5.4 安全意识：助记词管理

这是 Day 1 最容易被忽视、出问题后果最严重的环节。助记词（Seed Phrase / Mnemonic）是私钥的人类可读形式，通常是 12 或 24 个英文单词。掌握助记词等于掌握该钱包下所有地址的私钥。

**绝对规则**：

-   助记词永远不要截图、不要存在备忘录/云端笔记/邮箱里。
    
-   永远不要在任何网页或 App 中输入助记词——正规钱包永远不会要求你输入助记词。
    
-   离线手写记录，存放在安全的物理位置。
    
-   任何人索要助记词都是诈骗，没有例外。

# **Web3 实习计划学习笔记：从认知到 Builder 的五周旅程**

## 整体概述：这套计划的设计逻辑

五周的内容是一条经过设计的递进路径：**认知 → 选向 → 构建 → 验证 → 职业转化**

第一周解决“Web3 是什么”，跨过门槛入门；第二周解决“我在 Web3 里做什么”以选定赛道；第三周解决“怎么做出东西”，从零到一交付产品；第四周用黑客松的高压环境检验前三周的积累；第五周把作品变成机会，完成从学习者到从业者的身份转换。

其设计的核心信念很明确：**以输出倒逼输入**。每一周都要求具体产出——不是“学完”，而是“做出”。第一周要完成第一笔链上交易，第三周要交付可运行的 MVP，第四周要路演 Demo，第五周要产出作品集和简历。学习不是目的，构建才是。

另一个值得注意的设计是**跨轨协作**。第二周把学员分入 Tech、Ops、Research 三条赛道，但第三周立刻要求跨赛道组队——技术写代码、运营做增长、研究做分析，三者配合才能交付一个完整项目。这模拟的是真实 Web3 团队的运作方式：没有谁能只靠单一技能把产品做起来。

### 第一周：Enter Onchain World — 跨入链上世界

这是整个旅程的“传送门”。从 Web2 到 Web3 的跨越，不在于读了多少文章，而在于你是否真正在链上完成过一次交互。

**1\. Web3 基础认知**

四个理解透的概念：

-   **钱包（Wallet）**：不是存钱的工具，而是你在链上的身份标识。没有钱包，你在 Web3 里不存在。
    
-   **Gas**：链上每一次操作都要付的“燃料费”。理解 Gas 就理解了为什么链上交互有成本、为什么需要优化合约。
    
-   **地址（Address）**：链上资产的存储位置，由公钥派生而来。它是公开的，但私钥必须绝对保密。
    
-   **区块链（Blockchain）**：去中心化的账本。它的核心价值不在于“去中心化”这个词本身，而在于无需信任第三方即可达成共识。
    

**2\. 链上实践**

理论认知之后必须动手。本周的实操路径是：

-   创建钱包，学习安全地管理私钥和助记词（这是最容易被忽视、出问题后果最严重的一步）
    
-   在测试网（Testnet）上熟悉链上环境——零风险地犯错
    
-   完成第一笔链上交易，走通从发起到确认的完整流程
    

**3\. AI × Builder**

本周引入了 AI 工具的三个使用层次：

-   **AI 学习**：用 AI 快速消化复杂的 Web3 协议文档，降低认知门槛
    
-   **AI Coding**：用 AI 辅助编写智能合约或前端代码，加速开发
    
-   **Build Log**：用记录开发日志的方式培养建设者思维——遇到什么问题、怎么解决的，都是未来最值钱的经验
    

### 第二周：Tech / Ops / Research — 选定赛道，构建能力壁垒

如果说第一周是横向铺开认知，第二周就是纵向深挖能力。三条赛道对应 Web3 行业的三种核心角色，本周的任务是选定方向并开始构建自己的能力壁垒。

### 三大赛道拆解

**技术赛道（Tech）**

核心是“把想法变成代码”：

-   智能合约：Web3 的底层逻辑载体，理解并亲手写合约是技术人的立身之本
    
-   工具链：掌握开发、测试、部署过程中必备的工具
    
-   开发实战：理论到代码的转化，动手写出来才算数
    

**运营赛道（Ops）**

核心是“让东西被看见、被使用”：

-   社区协作：在去中心化团队中高效沟通，这是 Web3 特有的协作范式
    
-   活动运营：策划执行活动，驱动社区活跃度
    
-   内容产出：高质量内容的创作与传播，在注意力稀缺的环境里建立项目认知
    

**研究赛道（Research）**

核心是“看懂行业，给出判断”：

-   行业研究：宏观把握 Web3 发展脉络
    
-   项目分析：对特定项目做深度拆解，理解商业模式与技术架构
    
-   趋势洞察：识别前沿方向，保持行业敏感度
    

### 本周产出要求

三条赛道各有对应的交付物：技术笔记、运营方案、研究报告。这确保了“学”和“做”绑定，不是停留在“听懂了”的层面。

### 第三周：Build / Ship / Iterate — 做出你的第一个 Web3 项目

### 本周定位

前两周积累的认知和技能，在这一周要变成一个真正能跑的东西。核心动作是三个词：构建、发布、迭代。

### 三阶段工作流

**1\. 项目开发（Build）**

重点是一个字：**MVP（最小可行性产品）**。不用试图在第一版就把所有功能做完，核心是验证最基础的逻辑能不能跑通。这周的理念是“完成比完美重要”——先让项目跑起来。

**2\. 部署上线（Ship）**

技术环节包括：

-   智能合约部署到区块链网络
    
-   前端集成，打通钱包与网页的交互
    
-   测试，确保合约逻辑和前端交互无误
    

**3\. 反馈迭代（Iterate）**

上线不是终点，而是起点：

-   收集真实用户的体验反馈
    
-   根据反馈做针对性优化
    
-   持续改进，修补漏洞、增加特性
    

### 本周产出

-   可运行的 MVP
    
-   演示 Demo（视频或文档）
    
-   基于当前版本和反馈的迭代计划
    

### 第四周：Hackathon Week — 用作品证明成长

### 本周定位

前三周是“练”，第四周是“战”。黑客松把所有人推入高压环境，在有限时间内完成从创意到路演的全流程。这是对前几周积累的集中检验。

### 黑客松四阶段

**1\. 团队协作 · 创意落地**

组队、碰撞创意，学习如何在 Web3 生态中找到真实痛点，把抽象想法转化为可执行的项目方案。这一步考验的是问题嗅觉和团队配合。

**2\. Build · Ship**

全力构建 MVP，在有限时间内交付一个可演示的 Demo。高强度的工程化能力在这里被压到极限。

**3\. Demo 展示 · 评审**

路演展示，向导师和伙伴讲清楚项目价值。这一步考验的是表达能力——做出来了还要讲明白，这是很多技术人容易忽视的能力。

**4\. 冲击大奖 · 社区曝光**

优秀项目角逐奖金（以 USDT 形式发放）和生态资源支持，同时获得社区曝光。这一步让人理解 Web3 生态的激励机制：好的项目不仅能赢奖金，更能通过社区资源获得持续赋能。

### 黑客松的核心逻辑

**Build → Ship → Win**。不只是“做出来”，还要“发出来”，最终才能“赢”。三者缺一不可。

### 第五周：Career Port — 从 Builder 到 Professional

### 本周定位

前四周让你成为了 Builder，第五周的目标是让你成为 Professional。核心命题是：**把你的作品，变成未来的机会。**

### 四项核心任务

**1\. 作品集整理**

打磨 Demo、优化代码仓库（Repo）、完善文档。在 Web3 领域，代码和实际产品就是最好的简历。不仅要功能完整，还要注重代码规范和文档可读性——这些细节向招聘方传递的是你的工程素养。

**2\. 简历优化**

突出项目经历，聚焦于“在 Web3 项目中解决了什么具体问题、用了什么协议栈、个人有什么独特贡献”。避免流水账式的罗列。

**3\. 机会对接**

利用内推机会、参与企业交流、争取导师推荐。Web3 社区高度依赖社交资本，很多职位信息是不公开的（Hidden Job Market），通过与导师和企业的直接对话才能触达。

**4\. 职业启航**

面向未来，确立自己在 Web3 生态中的长期定位——协议开发者、安全审计师、DApp 架构师，方向需要在这个阶段开始清晰。

### 核心理念

**FROM BUILDER TO PROFESSIONAL**。从学习者到建设者的蜕变已经完成，下一站的定义权掌握在自己手中。

## 整体复盘：这五周旅程将教会我什么

**1\. 输出导向的学习才是有效的**

这五周里，没有任何一周的终点是“学完”。第一周要完成链上交易，第二周要产出笔记/方案/报告，第三周要交付 MVP，第四周要路演 Demo，第五周要产出作品集。每一周都以具体产出收尾，这倒逼你把知识真正用起来，而不是停留在“听过、知道”的层面。

**2\. 跨轨协作是 Web3 的真实工作方式**

技术、运营、研究三条赛道看似独立，但在真实项目中三者缺一不可。第三周的跨轨组队是整个计划的关键设计——它让你提前体验真实团队的协作摩擦，学会用别人听得懂的方式表达技术问题，也学会理解运营和研究的语言。

**3\. 实战是检验能力的唯一标准**

第四周的黑客松把前三周的积累推上考场。在高压环境下做技术取舍、在有限时间里完成交付、在路演中讲清楚项目价值——这些能力只有实战能练出来，纯学习永远练不出。

**4\. 职业转化需要主动经营**

第五周的核心信息是：作品做出来了，不等于机会会自动找上门。你需要主动整理作品集、优化简历、对接人脉、建立个人品牌。在 Web3 这个高度依赖社区和声誉的行业里，“被看见”和“做出来”同等重要。
