    -   带有 `view` 或 `pure` 的函数（如 `getMessage`）是**读操作**。

    -   `msg.sender`（谁调用的）和 `block.timestamp`（什么时候调用的）是智能合约获取外部信息的桥梁，不需要作为参数传进来，合约内部直接就能用。
        
3.  **数据位置（Data Location）**：
    
    -   复杂类型（如 `string`, `struct`, `array`）在作为函数参数或返回值时，必须标明是放在 `memory`（内存，临时）还是 `storage`（存储，永久）。为了省 Gas，函数内部处理时尽量用 `memory`。

# **使用Github**

## **一、Github Flow：按照 GitHub 流程开展项目协作**

1.  创建分支：简短的描述性分支名称
    
2.  进行更改：
    
    -   给每个提交一个描述性的信息；
        
    -   每次代码提交应包含一个独立且完整的更改 （例如：如果要重命名变量并添加一些测试，则将变量重命名放在一个提交中，将测试放在另一个提交中。 以后，如果想保留测试但恢复变量重命名，则可以恢复包含变量重命名的特定提交）
        
    -   为每个不相关的更改集创建单独的分支
        
3.  创建拉取请求：
    
    -   在完成更改之前需要早期反馈或建议，可以将拉取请求标记为草稿
        
    -   提供更改的概述和解决的问题
        
4.  解决审查评论：评论者可以评论整个拉取请求或将评论添加到特定行或文件
    
5.  合并拉取请求：拉取请求获得批准后，合并拉取请求（合并分支）。
    
6.  删除分支：合并拉取请求之后，删除分支，表明分支的工作已经完成。
    

## **二、连接到Github：**

### **2.1 通过浏览器在 GitHub.com 上操作**

GitHub 是一款基于 Web 的应用，可用于将文件托管在存储库中、协作处理以及跟踪随时间推移对文件的更改

**入门**：较好的开始方式是通过浏览器在 GitHub.com 上进行更改。

由于以下原因，直接使用 GitHub.com 通常是最快捷的参与存储库工作的方式。

-   正在直接使用 GitHub 托管存储库，因此无需将存储库副本下载到计算机并使此副本保持同步。
    
-   如果已登录到 GitHub，则有权访问你拥有必需权限的任何存储库，因此无需在计算机中设置任何其他身份验证。
    
-   由于可以在用户界面中提交更改，因此无需使用命令行或记住任何 Git 命令。
    

**在浏览器中进行更复杂的更改：**

-   如果需要能够快速打开或创建文件的编辑器，可以在任意存储库中按`.`键打开github.dev 编辑器。 这是一个基于 Web 的轻量级编辑器，包含许多 Visual Studio Code 的功能，例如搜索栏和用于 Git 命令的按钮。
    
-   如果想要继续使用浏览器但需运行命令、创建项目的测试版本或安装依赖项等，可以在 Codespace 中打开存储库。 Codespace 是具有存储和计算能力的远程开发环境。 其包含编辑器和集成终端，并预装了处理项目（包括 Git）可能需要的常用工具。
    

### **2.2 从桌面工作：**

要从桌面使用存储库，需要将存储库的副本下载（或“克隆”）到计算机，然后将所做的任何更改推送到 GitHub。

从桌面连接到Github的工具：

-   命令行：充分控制和较高灵活性。需要安装Git并熟悉基本的git命令；还可以安装使用 GitHub CLI，该命令行接口可用于对 GitHub 执行多种操作，例如创建拉取请求或存储库分支。
    
-   GitHub Desktop 等可视化 Git 客户端：可以使用可视化界面
    
-   编辑器：如果要在同一个位置进行操作，通常可以通过编辑器执行大部分操作。 VS Code 等编辑器包含常见 Git 命令的集成终端和按钮，可供编辑文件并从同一个位置将更改推送到GitHub。 还可以通过安装扩展直接处理 GitHub 上的拉取请求和问题。

Gas vs 手续费：

为了方便理解，我们可以把区块链网络想象成一家**快递公司**：

-   **Gas** 相当于你包裹的 **“重量”**（代表交易占用的计算资源）。
    
-   **Gas 价格** 相当于快递的 **“每公斤单价”**。
    
-   **手续费** 相当于你最终支付的 **“总快递费”**。
    

Gas是区块链中用来衡量**“执行某项操作需要消耗多少计算资源”** 的单位。

-   **为什么需要它？** 区块链上的计算机（验证者/节点）需要花费算力来处理你的交易。Gas 就是用来量化这种工作量的。同时，它也能防止有人恶意发送死循环程序把网络搞瘫痪（因为运行程序需要花钱买 Gas，没钱了程序就会自动停止）。
    
-   **Gas 消耗量是固定的吗？** 对于简单的操作，消耗量是固定的。比如最普通的代币转账，无论转 1 个 MON 还是 100 万个 MON，系统规定固定消耗 **21,000 Gas**。但如果去和复杂的智能合约交互（比如在去中心化交易所换币、铸造 NFT），因为计算复杂，可能需要消耗 10万 甚至 100万 Gas。
    

手续费：是为了把这笔交易记录到区块链上，**实际支付给网络的代币金额**（也就是真金白银）。

在早期的区块链中，手续费全都被矿工/验证者赚走了。但现在的区块链（包括 Monad 采用的 EVM 机制）引入了**销毁机制**，所以手续费被分成了两部分：

NaN.  **销毁费用（Base Fee）**：系统根据当前网络拥堵程度自动计算出的“基础底价”。这部分钱会被**直接打入黑洞（销毁）**，永远消失。这会让代币总量变少，起到通缩作用。
      
NaN.  **验证者小费（Priority Fee / Tip）**：为了让验证者优先打包你的交易，你额外给的一点“小费”。这部分钱是真正落入验证者（打包这笔交易的节点）口袋里的。
      
NaN.  **总费用** = 销毁费用 + 验证者小费。
      

交易失败也会扣除gas费用的原因：当你在区块链上发起一笔交易（尤其是和智能合约交互时），会发生以下过程：

NaN.  **执行代码**：验证者节点开始运行智能合约的代码。这个过程需要一步步进行计算，每计算一步，就会扣除相应的 Gas。
      
NaN.  **触发失败条件**：在计算过程中，代码发现某个条件不满足。比如：“你的余额不足”、“你没有权限”、“合约里的代币已经卖光了”。
      
NaN.  **状态回滚（Revert）**：因为条件不满足，区块链会**撤销**这笔交易带来的所有状态改变（比如你的钱不会转出去，代币也不会给你）。这就叫“交易失败”。
      
NaN.  **Gas 不退还**：虽然交易结果被撤销了，但前面第1步中节点已经付出的计算过程是无法撤销的。因此，已经消耗的 Gas 费必须从你的钱包中扣除，支付给验证者。
      

总结：一旦提交了交易，就不可避免的会消耗区块链上节点的算力，使用了计算资源，所以无论交易最终结果是成功还是失败，都会产生费用

# 一、Web3 vs Web3.0 vs Web2的范式革命

## 1.1 Web2（当前互联网）

**核心特征：**

- 中心化控制：数据存储在科技巨头的服务器（如 Google、Facebook）
- 用户角色：内容生产者，但不拥有数据
- 商业模式：广告驱动，平台抽取佣金
- 典型应用：微信、抖音、亚马逊

比喻理解：就像租房子，你可以装饰（发内容），但房东（平台）随时能收回钥匙（封号）

## 1.2 Web3.0（语义网）

**核心特征：**

- 语义标记：使用RDF、OWL等标准描述数据关系
- 结构化数据：信息按照标准格式组织，便于机器理解
- 知识图谱：构建实体间的语义关系网络
- 典型技术：本体工程、语义查询语言（SPARQL）、链接数据

**关键区别：**

1. 不是区块链技术，而是传统互联网的数据组织升级
2. 主要不依赖AI，而是通过标准化数据格式实现
3. 与web3可结合（语义标记 + 区块链存储）

## 1.3 Web3（去中心化互联网）

核心特征：

- **数据主权归用户**：用区块链存储身份和资产

- **无需信任中介**：智能合约自动执行规则

- **核心组件**：

- **真正拥有数字资产**：你的 NFT 头像、游戏道具真正属于你，平台无法删除或收回

## 1.7 相关解释：

### 1.7.1 语义网（Semantic Web）：

又被称为智能网。

语义网不是指一个全新的网络，而是对当前万维网的一种扩展，其核心目标是：**让互联网上的数据变得“机器可读”且“机器可理解”**。

语义网是通过在网页数据中添加**结构化的元数据（Metadata）**，明确地告诉机器数据代表什么，以及数据与数据之间的关系。

语义网的核心技术（W3C标准）：

语义网的实现依赖于一系列由W3C（万维网联盟）制定的标准技术，通常被称为“语义网技术栈”：

- URI（统一资源标识符）：给互联网上的每一个事务（人、地点、概念）一个全球唯一的“身份证号”
- RDF（资源描述框架）：语义网的数据模型。它使用 **“主语-谓语-宾语”的三元组**来描述事实。例如：`[北京] (主语) - [是首都] (谓语) - [中国] (宾语)`
- OWL（Web本体语言）：用于定义复杂的本体（Ontology），即概念之间的分类和逻辑关系。它让机器具备了逻辑推理的能力（比如：如果机器知道“麻雀是鸟”，且“鸟都有羽毛”，它就能推理出“麻雀有羽毛”）
- SPARQL：一种专门用于在语义网中查询RDF数据的编程语言。

语义网的实际应用：

- **知识图谱（Knowledge Graphs）：** 比如 Google 搜索时右侧弹出的信息卡片，或者百度的实体百科。它们背后都是语义网技术，将海量数据关联成了结构化的知识网络。
- **智能搜索引擎：** 现在的搜索引擎不仅能匹配关键字，还能理解你的搜索意图（比如搜索“姚明身高”，它能直接给出“2.26米”的答案，而不是给你一堆包含这两个词的网页）。
- **Linked Data（关联数据）：** 维基百科的 DBpedia 项目将维基百科的内容转化为了语义网数据，供全球的研究者和开发者免费调用和关联。
- **智能助手与物联网：** 比如 Siri、小爱同学等 AI 助手，需要理解复杂的上下文和实体关系，这离不开语义网的技术支撑。

### 1.7.2 本体（ontology）：

**本体就是“对特定领域之中某套概念及其相互之间关系的形式化表达（formal representation）”**

官方定义为：“共享概念模型的明确的形式化规范说明”

本体的核心要素：一个完整的本体通常包含五个核心组件：

1. **类 / 概念(Classes / Concepts)**：领域内的事务类别。例如：人、公司、汽车、疾病
2. **属性(Properties / Attributes)**：对象（和类）所可能具有的属性、特征、特性、特点和参数。例如人有年龄和性别；汽车有最高时速
3. **关系 (Relations)**：概念之间是如何互动的。这是本体最强大的地方。
   - 分类关系（Is-a）：麻雀是一种（is-a）鸟
   - 组成关系（part-of）：发动机是汽车的一部分
   - 其他语义关系：马斯克创立了spaceX；阿司匹林用于治疗头痛
4. **实例 (Instances / Individuals)**：现实世界中具体的个体。例如：“姚明”是`人`这个类的一个具体实例；“特斯拉 Model 3”是`汽车`的一个具体实例。
5. **公理 / 约束 (Axioms / Constraints)**：永远为真的逻辑规则。例如：“一个人”不能同时是“一家公司”；“年龄”必须大于等于 0；“素食主义者”不吃“肉类”。

本体与普通数据库、字典的区别：

- **普通字典/词表**：只告诉你词的意思和同义词（比如“西红柿”=“番茄”），没有逻辑结构。
- **关系型数据库**：只关心数据的存储和表格之间的外键关联（比如“员工表”的部门ID关联“部门表”的ID），它不知道“部门”和“员工”在现实世界中的深层语义。
- **本体**：不仅包含词汇和关联，还包含 **逻辑推理规则**。它赋予了数据灵魂，让机器能够像人类专家一样去理解这个领域。

本体在语义网中的位置：

- RDF（资源描述框架）就像是砖块，用来描述简单的事实（比如 `[北京] [是首都] [中国]`）。
- 本体（通常用OWL语言编写）就像是建筑图纸和物理法则。它定义了“什么是首都”、“什么是国家”，以及“一个国家只能有一个首都”这样的复杂逻辑。没有本体，语义网就只是一堆散落的数据，无法进行高级的智能推理。

现实中的应用场景：

- [x] **医疗健康（SNOMED CT）**：这是全球最大的医学本体。它把全世界所有的疾病、症状、药物、手术统一编码并建立逻辑关系。这使得不同国家的医院系统能够无缝对接，AI 也能根据症状推理出可能的疾病。
- [x] **电商与零售**：亚马逊或淘宝后台庞大的“商品类目树”本质上就是一种本体。它规定了“智能手机”属于“电子产品”，具有“屏幕尺寸”、“内存”等属性，从而支撑精准的商品搜索和推荐。
- [x] **知识图谱（Knowledge Graph）**：各大互联网公司（Google, 百度, 阿里）构建的知识图谱，底层都依赖本体来规范实体和关系的定义。
- [ ] **大语言模型（LLM）的结合**：现在的 AI 经常会产生“幻觉”（胡说八道）。将大模型与特定领域的“本体”结合（即神经符号AI），可以用本体的严谨逻辑去约束大模型的输出，让 AI 在医疗、法律等严肃领域回答得更准确。

总结：本体就是人类把某个领域的专家知识，翻译成机器能看懂的、带有严密逻辑规则的“概念地图”。有了这张地图，机器就不再是只会死记硬背的计算器，而是具备了一定“常识”和“推理能力”的智能体。

本体语言OWL（Web Ontology Language）：是一种用于编写本体声明（ontological statements）的语言。OWL的发展继承了RDF和RDFS以及一些早期的本体语言项目，包括本体推理层（Ontology Inference Layer，OIL）、DARPA智能体标记语言（DARPA Agent Markup Language，DAML）以及DAMLplusOIL。OWL旨在应用于万维网之上；而且，其构成要素（类、属性和个体）均被定义为RDF资源，并采用URI加以标识。

区块链的定义：

区块链是一种去中心化的分布式账本技术，用于在网络节点之间安全、透明且不可篡改地记录事务数据。每条链由一系列按照时间顺序相连的“区块”组成，每个区块内部包含两个部分：交易记录等一些信息，以及之前的区块摘要。（区块按照顺序依次串联）

区块（Block）：记录数据的基本单位，一个完整的区块由两大部分组成：区块头和区块体

区块头：包含了区块的元数据（Metadata），主要用于验证、链接和维护区块链的安全。通常包含以下核心信息：

1\. 前一区块的哈希值：区块链能够形成“链”式结构的关键。如果有人篡改了历史区块中的数据，会导致该区块的哈希值改变，进而使得后续所有区块的“前一区块哈希值”对不上，从而被网络拒绝。

2\. 时间戳：记录该区块被创建（或打包）的具体时间。证明了区块内的数据在某个特定时间点已经存在，是区块链溯源和共识机制的重要参考。

3\. 梅克尔根（Merkle Root）：是区块体中所有交易数据经过哈希运算后生成的最终摘要。通过梅克尔根，节点可以快速且安全地验证区块内包含的交易数据是否被篡改。任何一笔交易的微小改变，都会导致梅克尔根发生巨大变化。

4\. ......

区块体：包含了该区块实际要记录的核心业务数据，通常包括：

1\. 交易数据列表：包含了在这个区块打包周期内发生的所有有效交易记录。例如用户之间的转账记录、智能合约的调用数据等。

2\. coinbase交易（区块奖励）

区块头负责“安全和链接”（通过哈希值、时间戳、随机数等保证数据不可篡改并首尾相连），区块体负责“存储数据”（记录真实的交易明细）。

区块链的特性：

1\. 不可篡改：不可改变历史信息，因为每个区块都包含了上一个区块的摘要并加以串联。

2\. 公开透明、匿名：在区块链上的信息全部公开透明。每个人都可以顺着区块和链找到历史上所有的记录来查看你的钱包余额。但是没人知道这是你的钱包。

3\. 快速交易：只要交易记录被打包在区块链上，交易就自动完成。

特性的一些解释：

公开透明：账本是完全公开的。任何人都可以追溯一个钱包地址的完整历史，查看它确切持有多少加密货币，并追踪它发送或接收的每一笔交易。

匿名：并不代表完全隐形，而是指现实身份与链上活动被分离开了。现实身份被一个\*\*密码学钱包地址\*\*所替代。在区块链上，用户的身份不是姓名等真实信息，而是一个钱包地址——一个长字符串。

这里第二条特性提到的“没人知道这是你的钱包”，意思是：虽然所有人都能看到这个特定地址的余额和交易历史，但他们并不知道在现实世界中，究竟是谁在控制这个地址。

背后的原理：通过密码学原理和系统设计共同实现的：

\- 公钥密码学（非对称加密）：当创建一个区块链钱包时，系统会生成一对密钥：\*\*私钥\*\*（自己秘密保存，用于签名和授权交易）和\*\*公钥\*\*（公开的验证工具和地址生成器）。

\- 地址生成：钱包地址是通过密码学哈希函数（如SHA-256和RIPEMD-160）从公钥 数学推导出来的，因为哈希函数是单向的，从钱包地址反向推导出公钥，或者推导出用户的真实身份，在计算上是不可能的。

\- 无需注册：与传统的银行账户不同，创建区块链钱包不需要提供姓名、手机号或物理地址。用户可以在自己的设备上几秒钟内本地生成一个。因此，区块链网络中没有一个中心化的数据库将“地址 X”与“张三”联系起来。

快速交易：“快速”，主要是相对于“跨境汇款”而言的（传统跨境汇款可能需要3-5个工作日，而区块链只需要几十分钟）。它强调的“自动完成”，更多是指\*\*流程上的无缝衔接和无人工干预\*\*，而不是物理时间上的绝对“快”。

自动完成——核心含义是指：交易的验证、记账和结算全过程，完全由计算机代码和共识协议自动执行，没有任何人工干预，也不需要中心化机构审批（如银行）

1\. 无需人工审核与清算（去中心化）

区块链的“自动”：当发起交易后，交易会被广播到全网，网络中的节点（矿工/验证者）运行的计算机代码会自动检查你的签名是否正确？（用公钥）余额够不够？手续费给没给？只要代码验证通过，交易就会被 **自动打包** 进区块。没有银行的人工流程。

2\. 全天候运行

区块链网络是由全球成千上万台计算机组成的，机器不需要睡觉。无论在哪里、什么时候，只要网络畅通，交易就会被自动、持续地处理。

3\. 账本自动更新且不可逆（最终确认）

一旦交易记录被打包进区块链，全球所有节点的账本会自动同步更新。你的余额自动减少，对方的余额自动增加。这个过程是 **不可逆** 的，一旦完成，没有任何人能够手动去“撤回”或“冻结”这笔交易。

公钥：

最核心的作用是：验证交易签名——证明”这笔交易确实是由私钥的主人授权的“，且”数据没有被篡改“。

\- 原理：当用户发起一笔转账时，用户的钱包软件会用 **私钥** 对这笔交易数据盖上一个“数字印章”（这叫数字签名）

\- 公钥出场：这笔交易和签名会被广播到全网。网络中的其他节点（矿工/验证者）会拿出你的\*\*公钥\*\*，去核对这个“数字印章”。如果核对成功，就证明这笔交易确实是你本人发起的，且中途没被篡改。

\- 通俗理解：私钥就像私人印章，公钥就像你在公安局备案的印模。别人拿到你的文件，只要用印模（公钥）一盖，能完全吻合，就知道这文件确实是你盖的，别人伪造不了。

区块链+分布式网络——新的特性（确保历史交易记录没有被篡改且可信）：

分布式网络记账模式：一个区块链网络中有非常多的节点（矿机）来记账，每个节点都会记录完整的、相同的区块链信息。

去中心化：区块链网络通常分布在全球，每个节点都将会存储一份相同的区块链数据。没有人能够控制全部的节点，因此这份区块链数据将会一直存在。

不可篡改：区块链网络通常分布在全球，一个人控制大部分节点几乎不可能，因此即便有人修改了部分节点的区块链数据，只要被修改记录的节点不超过 51%，这个改动将不会被认可。

比特币（BTC）：比特币是作为奖励提供给去中心化和分布式网络服务的服务商

节点奖励机制：网络节点服务提供商可以得到奖励。不同的网络服务提供商可以得到不同的代币奖励，比如：比特币。

比特币具备货币属性。

区块链的核心组成部分：

\- 去中心化的网络和区块链：区块链将会有一条链来记录全部的信息，这条链将存在对应的去中心化网络中。 去中心化的网络，将由无数节点提供服务来维持网络运行。节点通过计算验证交易获得代币奖励。

\- 维持网络运行的代币激励：去中心化的网络由无数节点提供服务来维持网络运行。以 PoW 链为例，这些操作常被称为“挖矿”，参与者一般被称为矿工。
