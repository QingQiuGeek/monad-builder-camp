**今日学习笔记：EPF 与以太坊协议层入门**

今天这场分享主要讲的是 **EPF，也就是 Ethereum Protocol Fellowship**。它和我们前几天做的测试网转账、部署留言板合约不太一样。之前学的是“链上应用层”，比如钱包、智能合约、DApp；而 EPF 更偏底层，研究的是 **以太坊协议本身怎么运行、怎么升级、怎么被维护**。

一句话理解：

```
DApp / 智能合约开发：在以太坊上盖房子
以太坊协议层开发：研究和改造以太坊这块地基
```

**一、EPF 是什么**

EPF 聚焦的是以太坊协议层开发，内容包括：

| 方向 | 说明 |
| --- | --- |
| 执行层 | 交易执行、EVM、状态变化、客户端实现 |
| 共识层 | 验证者、区块确认、共识机制 |
| 网络协议 | 节点之间如何通信 |
| 密码学 | 签名、验证、数据结构、安全性 |
| EIP 提案 | 以太坊升级和改进方案 |

它不是教你写普通智能合约，也不是做钱包、NFT、DeFi 页面，而是更接近“参与以太坊本身的研究和维护”。

**二、EPF 的参与方式**

EPF 有两种参与方式：

| 类型 | 特点 |
| --- | --- |
| 官方许可参与 | 需要申请和面试，通过后可能有津贴 |
| 无许可参与 | 即使没通过，也可以自己学习、提交成果、参与 GitHub 讨论 |

官方项目一般每年三四月份开始自学通道，入选后会有大约 5 个月的实践周期。核心讨论基本都在 GitHub 上公开进行，这一点很重要：以太坊生态很多东西不是藏在内部文档里，而是公开在 GitHub、EIP、规范文档和社区讨论里。

**三、推荐学习路径**

今天最重要的学习建议是：不要一上来就冲贡献，先搭知识框架。

比较适合新手的路径：

1.  先看以太坊基础概念。
    
2.  再读以太坊官方 Wiki 和规范文档。
    
3.  用 AI 辅助理解英文文档。
    
4.  每周整理学习笔记。
    
5.  选择一个小方向长期跟进。
    
6.  从小 issue、小 PR 开始参与。
    
7.  慢慢进入 EIP、客户端、协议讨论。
    

辅助资料可以看：

| 资料 | 用途 |
| --- | --- |
| 以太坊官方文档 / Wiki | 建立正统理解 |
| Ethereum Specs | 理解协议细节 |
| EIP 提案 | 看以太坊如何升级 |
| Go 客户端代码 | 理解成熟客户端实现 |
| 肖海老师《从0到1学习以太坊》 | 中文入门辅助 |
| 区块链文明三部曲 | 降低理解门槛 |
| Discord 协议学习社区 | 遇到问题问 mentor |

**四、贡献实践方法**

分享里提到一个很实际的点：新手不要一开始就做大 PR。

更好的方式是：

```
先选定一个问题域
↓
找到相关仓库和 issue
↓
做小范围修改
↓
提交 2-3 个有关联的小 PR
↓
体现持续理解和连续贡献
```

小 PR 更容易被 review，也更适合新手建立反馈循环。不要到处乱找浅层问题，否则很容易做着做着断掉。

**五、面试和申请重点**

EPF 面试不是传统技术面试，不会重点考算法、八股、手撕代码。

更看重：

| 重点 | 说明 |
| --- | --- |
| 对以太坊的兴趣 | 为什么想参与协议层 |
| 对某个模块的理解 | 比如执行层、共识层、EIP |
| 学习记录 | 是否持续整理和输出 |
| 英文交流能力 | 面试是英文形式 |
| 研究潜力 | 不一定要求很强代码背景 |

所以对新手来说，最好的准备方式不是刷题，而是选一个小方向深入理解，并能用自己的话讲清楚。

**六、我今天真正理解到的点**

今天对我最有价值的理解是：

**Web3 不是只有发币、NFT、DApp 和链上交互。真正底层的东西，是协议如何运行、如何升级、如何被开发者共同维护。**

之前我部署留言板合约，是在“使用以太坊这台机器”；而 EPF 研究的是“这台机器本身为什么能运转，以及未来要怎么改进”。

这让我对 Web3 的分层更清楚了：

```
钱包层：MetaMask、Rabby
应用层：DApp、NFT、DeFi
合约层：Solidity、智能合约
协议层：执行层、共识层、客户端、EIP
基础研究层：密码学、网络、数据结构
```

**七、职业与现实判断**

协议层开发听起来很硬核，但它不是一条特别功利的职业路线。

分享里提到：

1.  协议层全职岗位较少。
    
2.  很多人是业余时间参与。
    
3.  经济回报不一定稳定。
    
4.  更适合真正对去中心化、开源协议、底层技术感兴趣的人。
    
5.  以太坊最大的优势不是短期价格，而是开发者生态。
    

这句话我觉得很关键：

```
公链最重要的是开发者。
```

因为一条链有没有长期生命力，不只是看币价，而是看有没有持续建设的人。

**一句话总结**

今天这节课让我意识到：Web3 入门不是只会部署一个合约，而是要逐渐看懂链的分层结构。EPF 代表的是更底层的以太坊协议学习路径，它难度高、周期长，但能帮助人真正理解“区块链不是一个产品，而是一套由全球开发者共同维护的开放协议”。

**二、完整实操流程**

| 步骤 | 做什么 | 目的 |
| --- | --- | --- |
| 1 | 安装 MetaMask 钱包 | 创建链上身份 |
| 2 | 创建新钱包并保存私钥/助记词 | 获得钱包控制权 |
| 3 | 添加 Mod / Monad 测试网 | 连接到指定测试链 |

**三、核心概念笔记**

| 概念 | 解释 |
| --- | --- |
| 钱包 | 链上账户工具，比如 MetaMask。它不是“存钱软件”，更像你的链上身份入口。 |
| 私钥 | 钱包控制权。谁有私钥，谁就能控制资产。不能发给任何人。 |
| 公钥/钱包地址 | 可以公开，用于收款、查询链上记录。 |
| 测试网 | 用来练习和开发的链，测试币没有真实价值。 |
| 水龙头 | 免费领取测试币的网站。通常每天只能领一次。 |
| 测试币 | 测试网上的 Gas 费燃料，不能当真钱。 |
| 交易哈希 | 每笔链上交易的唯一编号。可以在区块浏览器查询。 |
| 区块浏览器 | 查询地址、交易、合约、Gas、时间戳等链上信息的工具。 |
| Gas 费 | 链上操作的手续费，公式大致是 Gas 消耗量 × Gas 单价。 |
| 智能合约 | 部署在区块链上的程序，满足条件后自动执行，部署后通常不可随意修改。 |
| Remix | 在线 Solidity 合约编译和部署工具。 |

* * *

**四、Gas 费怎么理解**  
Gas 就是链上操作的“燃料费”。

简单理解：

```
Gas 费 = 操作复杂度 × 当前油价
```

不同操作费用不同：

| 操作 | Gas 消耗 |
| --- | --- |
| 普通转账 | 较低 |
| 部署合约 | 较高 |
| 调用写入函数，比如留言 | 会消耗 Gas |
| 查询只读函数 | 通常不消耗 Gas 或不需要发交易 |

留言字数越多，理论上链上存储数据越多，Gas 可能越高。测试网费用很小，但如果放到真实公链上，成本可能明显增加。

* * *

**五、智能合约留言板的意义**  
留言板合约虽然简单，但它覆盖了智能合约最基础的几个动作：

| 动作 | 代表能力 |
| --- | --- |
| 写留言 | 向链上写入数据 |
| 查留言 | 读取链上数据 |
| 记录钱包地址 | 识别是谁操作的 |
| 记录时间戳 | 记录什么时候操作 |
| 触发事件 | 让前端或区块浏览器能追踪交互 |
| 防止空留言 | 合约里的基础校验逻辑 |

所以它不是为了“做一个真正有用的留言板”，而是让你理解：  
**智能合约就是一段放在链上的程序，每次写入都会留下不可篡改的记录。**

* * *

**六、新手最容易错的地方**

1.  把私钥发给别人。  
    这是最严重错误，等于把钱包送人。
    
2.  以为测试币有价值。  
    测试币只用于练习和测试，不能兑换真钱。
    
3.  在区块浏览器里找交易按钮。  
    区块浏览器主要是查询工具，不是交易所。
    
4.  钱包网络选错。  
    如果你在 Monad / Mod 测试网操作，钱包也必须切到对应测试网。
    
5.  没有测试币就想部署合约。  
    部署合约、留言、转账都需要 Gas，所以必须先领水。
    
6.  以为合约部署后可以随便改。  
    链上合约部署后通常不可篡改，所以正式部署前要先测试。
