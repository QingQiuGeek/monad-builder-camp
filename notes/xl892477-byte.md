    学习总结：从合约代码到 DApp 初体验
    

1\. 核心理论：理解“自动售货机”模型

-   智能合约：本质上就是放在区块链上的“自动售货机”。它通过代码预设规则，自动处理交易，不需要人工干预。
    
-   交互逻辑：
    
    -   Read Function (读取函数，如 get)：这是“查阅”，因为不修改区块链上的任何状态，所以是免费的，不需要消耗 Gas。
        
    -   Write Function (写入函数，如 set)：这是“操作”，因为你在改变区块链上的数据，需要矿工进行记录，所以必须消耗 Gas 手续费，并且会生成一个唯一的 Transaction Hash（交易收据）。
        

2\. 实践操作：我的技术清单

-   环境搭建：学会了使用 Remix IDE。掌握了从编写代码、点击按钮编译（生成字节码），到将其部署到模拟网络（Remix VM）的全过程。
    
-   代码实践：编写了名为 SimpleStorage 的合约，深入理解了以下核心语法：
    
    -   pragma：声明编译器版本。
        
    -   uint256：定义存储数字的类型。
        
    -   public：定义函数可被外部访问。
        
    -   view：标记函数为只读，优化执行效率。
        
-   关键术语掌握：
    
    -   合约地址 (Contract Address)：这是合约在区块链上的“门牌号”，任何人都通过这个地址找到你的合约。
        
    -   ABI (Application Binary Interface)：这是合约的“说明书”。它以 JSON 格式告诉前端网页，合约里有哪些函数可以调用。
        
    -   DApp 架构：理解了一个完整的去中心化应用是由“前端界面（HTML/JS）+ 翻译官（Ethers.js）+ 链上合约”三部分构成的。
        

3\. 我遇到的困难与思考

-   概念转换：最初对于“为什么修改数据要付钱”感到困惑，但在理解了“写入操作需要全网共识并更新账本”后，对 Gas 的逻辑有了深刻认识。
    
-   环境配置：意识到“环境一致性”是初学者最容易踩的坑。必须确保钱包（如 MetaMask）连接的网络与部署合约的区块链网络处于同一个频道，否则无法交互。

### 7.7号Web3 实习作业总结报告

一、 学习目标达成

1.  **钱包安全：** 创建了独立的课程专用 Phantom 钱包，并已安全备份助记词，严格遵守安全准则。
    
2.  **网络配置：** 在 Phantom 钱包中成功配置并启用了 Monad Testnet 网络。
    
3.  **测试资产：** 成功从官方水龙头（Faucet）获取了测试币（MON），并通过区块浏览器确认余额到账。
    
4.  **浏览器分析：** 掌握了通过 Block Explorer 查询地址余额及验证交易记录的方法。
    
5.  **交互实操：** 完成了钱包连接及转账指令的发送操作，通过实践掌握了交互流程。
    

二、 遇到的主要问题及解决方案

1.  **网络添加障碍：** 尝试添加网络时提示 Chain ID 已存在，后确认网络已存在，通过网络列表直接切换解决。
    
2.  **水龙头领取失败：** 领取时提示连接服务器失败，通过避开高峰期重试解决。
    
3.  **代币显示异常：** Phantom 钱包首页未自动显示 MON 代币，通过开启“开发者设置”中的“Testnet Mode”并在管理代币列表中手动匹配解决。
    
4.  **交易指令积压 (Pending)：** 在进行最后一步转账交互时，交易在钱包中长时间处于 Loading 状态，尚未完全确认上链。经分析，这是由于当前测试网网络波动、RPC 节点响应延迟及网络拥堵导致的典型“交易卡顿”现象。
    

三、 当前状态说明

目前我的钱包已处于 Monad Testnet 网络状态，且浏览器显示地址 确实持有测试资产。因测试网高负载导致最后一笔转账指令处于 Pending 状态，该现象已作为本次实操观察到的真实交互结果记录在案。

四、 学习心得体会

本次实践让我深刻认识到 Web3 并非即时响应的系统。每一笔操作的背后，实际上是与去中心化节点的反复通信。虽然交易因网络拥堵处于 Pending 状态，但这恰恰让我直观地理解了 Gas 机制与网络吞吐量的运行规律。通过观察区块浏览器，我明白了链上数据才是唯一的真相——即便钱包插件的界面存在延迟，浏览器也能准确反映资产状态。这次练习不仅提升了我的动手能力，更培养了我排查并理解去中心化系统故障的耐心。
