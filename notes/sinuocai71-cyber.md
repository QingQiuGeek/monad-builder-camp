### 3\. AI Agent：从问答工具升级为自主任务执行系统

（1）基础定义

AI Agent = 运行在特定环境下的**目标驱动系统**

运行逻辑：基于目标、上下文、约束条件选择行动 → 执行后观察环境结果 → 更新自身状态，循环执行直至完成目标或人工介入

完整执行链路：**目标 → 感知 → 规划 → 行动 → 评估**

1.  目标：明确最终要交付的结果
    
2.  感知：读取当前环境、已有信息
    
3.  规划：拆解分步执行方案
    
4.  行动：调用各类工具落地操作
    
5.  评估：校验执行结果是否达标
    

（2）Agent 完整组成公式

`Agent = 大模型 + 上下文 + 记忆模块 + 工具集 + 执行循环 + 权限边界 + 结果验证`

核心本质：将模型推理，转化为受约束、可观测、可校验的标准化行动

（3）AI Agent 能力快速迭代的四大底层原因

1.  **模型更强**：推理、代码理解、多步骤规划能力持续升级
    
2.  **上下文更长**：可一次性读取海量代码、文档、历史任务、中间输出结果
    
3.  **工具更丰富**：支持 CLI 终端、浏览器、代码沙箱执行、MCP 协议、第三方外部 API
    
4.  **闭环更完整**：持久化指令、权限管控、自动化测试、结果评估体系完善，大幅提升稳定性
    
    总结：能力增长是「更强模型」+「完整行动系统」双向协同演进的结果

### 工具定位：区块链实时监控大屏

MonadVision 是 Monad 链的数据可视化浏览器，实时展示整条公链运行数据，分为两大核心板块：

1.  **Latest Blocks 最新区块**
    
    -   展示新出区块、区块提案验证者、区块打包交易笔数、区块产出 MON 奖励
        
    -   核心指标看板：总交易数、当前区块高度、合约总量、全网验证者数量、TPS、平均出块时间、Gas 均价、代币总量与销毁量等全局链数据
        
2.  **Latest Transactions 最新链上交易**
    
    -   实时刷新全网所有链上操作：转账双方、交易金额、发起时间
        
    -   交易不止转账：包含合约授权、函数调用、合约交互、质押等操作；金额为 0 也会消耗 Gas，属于有效链上交互
        

## 二、核心功能：钱包地址查询（步骤 9）

### 功能：查询账户资产与完整交易记录

粘贴钱包地址搜索后，进入 Account 账户详情页，核心分区：

1.  **Token Portfolio 代币资产面板**
    
    展示账户持有所有代币余额，示例：119.9961 MON
    
2.  **交易面板 Transaction**
    
    -   区分两类流向：IN（转入）、OUT（转出）
        
    -   每条交易包含：TX Hash（交易哈希）、交互方法（如 MonTransfer 转账）、发送方 / 接收方、时间、交易状态、转账金额、Gas 手续费
        
    -   交易状态 Status：Success 代表交易执行完成
        
3.  **Transaction Details 交易详情弹窗**
    
    单条交易完整核验信息：
    
    -   Transaction Hash：交易唯一哈希标识
        
    -   Block：该交易打包进的区块高度 + 区块确认数
        
    -   Status：交易执行结果 + 交互类型
        
    -   Timestamp：交易上链时间
        
    -   Internal Txns、Logs、Raw JSON：底层链上原始数据，供开发 / 深度核验
        

### 拓展知识点：链上产品 vs 传统互联网产品

1.  传统互联网：用户数据存储于平台私有服务器，数据库不对外公开，第三方无法核验
    
2.  区块链产品：所有关键行为（转账、领测试币、DApp 交互、质押等）永久记录在公开区块链，任何人可通过浏览器查询、溯源、核验，数据透明不可篡改
    

## 三、补充术语释义

1.  **Validator 验证者**：负责打包区块、维护链运行的节点，出块可获得 MON 奖励
    
2.  **Gas**：链上操作手续费，任何交互（转账、合约调用）均需消耗，用于激励节点
    
3.  **TPS**：每秒交易处理量，衡量公链运行速度
    
4.  **Contract 智能合约**：部署在链上自动执行的程序，所有 DApp、代币功能依托合约实现
    
5.  **TX Hash**：每笔交易专属唯一编码，用于精准定位、查证单条操作
    
6.  **Mainnet / Testnet**
    
    -   主网：正式环境，代币具备真实价值
        
    -   测试网：开发测试环境，代币仅用于调试，无实际价值

## 二、区块链网络运行

### 1\. 为什么不可篡改需要时间？为什么要等确认？

1.  区块像账本新一页，会引用上一页指纹 (hash)
    
2.  区块越往后叠，历史被推翻的成本越高
    
3.  PoS 下存在「最终确认 (finality)」，通常需要 12–13 分钟达到更稳定的最终状态
    
    -   12–13 分钟并非简单等待，由精心设计的流程决定，核心单位是**时段**
        
    -   以太坊 PoS：1 个时段≈6.4 分钟，时间被划分为连续时段，单时段 6.4 分钟（384 秒）
        
    -   最终确认需要**两个连续时段**：区块必须经过两个时段验证者集体验证同意，才获得最终确认
        
        -   第一个时段：提议并初步确定一个区块
            
        -   第二个时段：下一时段验证者集会检查并确认上一时段区块有效
            
        -   理论总耗时：6.4 分钟 / 时段 × 2 时段 = 12.8 分钟，即 12–13 分钟的由来
            
    -   「双重确认」机制作用：预留充足时间检测、应对恶意行为（双重投票），保障最终确认绝对安全
        

### 2\. PoS 机制中心化疑问：如果我的钱足够多，岂不是大部分都被我一个节点控制，失去去中心化？

配套缓解机制：对抗中心化趋势，多数 PoS 项目设计制衡规则

1.  设置验证者上限：以太坊要求质押 32 ETH 才可成为独立验证者，鼓励多人单独质押，避免集中大池
    
2.  惩罚机制（Slashing）：对作恶验证者惩罚，没收部分质押代币
    
3.  鼓励分布式质押服务：如 Lido 这类流动性质押协议，小额持有者可参与质押；但质押协议自身存在中心化风险
    
4.  底层挑战：代币分配的初始中心化问题，仍是 PoS 机制长期需要应对的难点
    

## 三、智能合约的本质

1.  智能合约 = 区块链账本里的「可执行代码」
    
2.  在 EVM (虚拟机) 里运行
    
3.  写进区块链：具备难以篡改、可追溯的特性

# Web2 / Web3.0（语义网）/ Web3 去中心化互联网 完整学习笔记

## 一、三代互联网范式区分

### 1\. Web2（当前主流互联网）

核心特征

1.  存储中心化：用户数据存于平台服务商服务器（谷歌、Meta、抖音、亚马逊等）
    
2.  用户身份：用户是内容生产者，但不拥有自身数据所有权
    
3.  商业模式：广告驱动流量，平台抽取佣金、垄断用户收益
    
4.  典型产品：微信、抖音、淘宝、亚马逊、各类中心化 App
    

### 2\. Web3.0（语义网，和区块链 Web3 完全区分）

核心特征

1.  技术底层：**不是区块链**，只是传统互联网数据集标准化升级
    
2.  核心技术：语义标记（RDF、OWL）、结构化数据、知识图谱、SPARQL 语义查询、链接数据
    
3.  设计逻辑：给全网数据统一标准化标签，让机器自动读懂数据关联（如同给图书馆每本书标注作者、题材、关联书目）
    

关键区别（与区块链 Web3）

1.  不依靠 AI，靠统一数据格式实现智能检索
    
2.  可与区块链 Web3 结合（语义标记 + 分布式存储）
    
3.  落地局限：纯语义网仅优化底层数据标准，没有面向 C 端出圈消费级产品，市面无大众熟知应用
    

### 3\. Web3（去中心化区块链互联网，行业通用 Web3）

核心特征

1.  数据主权：用户通过加密钱包自主持有身份、数字资产，平台无权删除、回收资产（NFT、链游道具永久归用户）
    
2.  信任无中介：智能合约自动执行规则，不需要银行、平台第三方担保
    
3.  原生金融属性：钱包内置借贷、理财、交易能力，无需实体银行卡
    
4.  可组合性：DeFi、链上应用如同积木，不同协议流动性可互相调用复用
    
5.  存储永久留存：内容、交易记录分布式存在全网节点，不会因平台关停丢失
    

核心创新与行业现实思考

1.  多链共存现状：公链赛道存在性能、安全、成本差异化竞争，类似互联网早期浏览器大战，是技术与市场自然选择；长期理想是多链互通跨桥，形成分层 L1+L2 健康去中心化生态，而非单一条链垄断。
    
2.  去中心化长期隐患：
    
    -   单点风险：若一条公链大量资产集中少数巨鲸节点，会弱化去中心化属性；
        
    -   网络脆弱性：节点数量不足时易遭遇 51% 攻击，资产、内容彻底冻结无法转移；
        
3.  安全兜底逻辑：分布式节点冗余备份、区块层层叠加，历史篡改成本极高；PoS 共识通过**双时段最终确认**机制（单时段 6.4 分钟，完整确认约 12-13 分钟）抵御双重投票作恶。
    
4.  PoS 质押中心化缓解方案
    
    -   门槛限制：设置验证者最低质押门槛（以太坊 32ETH），鼓励散户独立质押、不集中托管；
        
    -   惩罚机制 Slashing：恶意验证节点会被没收质押代币；
        
    -   流动性质押协议（Lido 等）降低参与门槛，但协议本身存在中心化运营风险；
        
    -   长期难题：代币初始分发不均带来的中心化问题，是 PoS 长期需要解决的痛点。
        

Web3 优劣势总结

优势（金融 / 产权 / 隐私）

1.  信任最小化：交易依靠密码学、代码自动证明，去除第三方中介信任成本；
    
2.  抗审查高弹性：数据多节点备份，单点故障、审查无法阻断全网；
    
3.  用户完全自主管理资产、身份、数据；
    

短板与落地局限

1.  性能瓶颈：公链 TPS 极低，无法承载微博类高频社交交互；
    
2.  成本门槛：每笔链上交互都消耗 Gas 费，高频使用用户成本难以接受；
    
3.  隐私缺陷：链上交易全部公开可查，不适合敏感个人交易场景；
    
4.  GameFi 落地矛盾：链上交互 Gas 消耗高、设备交互频繁，原生公链体验差，行业通用解决方案是**L2 低费扩容链**（Polygon、Immutable X）承载游戏业务，把高频交易下沉二层。
    

## 二、Web3 底层运行原理（加密钱包、交易、区块链网络、智能合约）

### 1\. 加密密钥体系（助记词 / 私钥 / 公钥 / 地址）

1.  助记词（种子根密码）
    
    -   定义：一个钱包对应唯一助记词，相当于一串能衍生无数钥匙的钥匙环；1 组助记词可派生多组独立私钥，对应多个不同链地址；
        
    -   安全优先级：助记词安全优先级 > 单个私钥，一旦泄露所有资产全部失控；
        
    -   安全准则：**Not your key, not your coin（私钥不在你手里，资产不属于你）**；仅自身泄露会丢失资产，不会因区块链平台被攻击被盗。
        
2.  私钥
    
    -   作用：交易签名唯一凭证，可推导公钥，不可逆；
        
    -   风险点：截图、明文存储、传输泄露 = 资产全部被盗；不可截图、分享、云端保存。
        
3.  公钥 & 转账地址
    
    -   公钥由私钥单向计算生成，公钥再压缩哈希得到钱包收款地址；
        
    -   地址可公开分享用于收款，无资产泄露风险。
        

### 2\. 交易与签名完整流程

1.  交易定义（Tx）：转账、mint、合约调用、投票、质押等所有链上操作；
    
2.  Gas Fee（手续费）三层作用
    
    -   资源付费：支付节点算力，激励打包者维护网络；
        
    -   垃圾交易防护：高额 Gas 成本阻挡垃圾刷量攻击；
        
    -   经济循环：供需付费维持去中心化网络长期稳定运转；
        
3.  Nonce 防重放序号：每个钱包独立计数，避免同一笔交易重复执行；
    
4.  数字签名核心逻辑
    
    用户用私钥对交易内容签名，全网任何人可用公开地址校验签名真伪，但无法反向推导私钥，实现单向可信验证；
    
5.  完整交易生命周期
    
    钱包签名 → P2P 广播至节点 → 交易池 Mempool 排队 → 验证者筛选打包上区块 → 全网同步账本 → 浏览器可永久查询记录。
    

### 3\. 区块链网络运行机制

1.  区块账本结构：每个区块头部包含上一区块哈希指纹，层层串联；区块高度越靠后，篡改整条历史记录成本指数级上升；
    
2.  不可篡改的时间逻辑（以太坊 PoS）
    
    -   基础单位：1 时段 = 6.4 分钟（384 秒）；
        
    -   最终确认规则：区块需要连续经过 2 个时段验证者集体复核，合计约 12.8 分钟（日常统称 12-13 分钟），完成双重校验，彻底杜绝双花作恶；
        
3.  区块浏览器：Etherscan 等工具，可输入地址 / 区块哈希，永久查询全部历史交易、资产、合约记录。
    

### 4\. 智能合约本质

1.  定义：部署在区块链账本上的**可执行代码**；
    
2.  运行载体：EVM 以太坊虚拟机；
    
3.  核心特性：写入链上后不可篡改、全程可追溯，规则自动执行无人工干预。
    

## 三、关键补充思考问答

1.  为什么 Web3 不适合微博类高频社交？
    
    公链 TPS 性能不足、每发一条动态都需支付 Gas 手续费、所有内容公开无隐私保护，经济与性能均无法支撑 C 端高频社交场景。
    
2.  如果绝大多数巨鲸持有一条公链大部分质押资产，还算去中心化吗？
    
    会大幅削弱去中心化公平性，行业通过质押门槛、Slashing 惩罚、流动性质押协议分散筹码缓解，但代币初始分配不均仍是 PoS 机制长期核心难题。
    
3.  Web3.0 语义网和区块链 Web3 怎么区分？
    
    语义网仅标准化互联网数据格式，无代币、无分布式资产；行业所说 Web3 = 区块链去中心化网络，带用户数字资产、加密钱包、智能合约金融体系，二者技术、商业模式完全无关。
    
4.  GameFi 行业主流落地方案是什么？
    
    不在以太坊 L1 主网运行，迁移至 Polygon、Immutable X 等低成本 L2 扩容链，把高频游戏交互下沉二层，降低 Gas 消耗、优化玩家操作体验。

五、任务4：链上产品基础认知

（一）链上产品 vs 普通互联网产品核心区别

1\. **数据存储底层不同** - 传统互联网：数据、资产记录存企业私有服务器，平台可修改、冻结、删除数据 - 链上产品：资产、交易、合约关键操作上链，分布式账本存储，全网可查，难以篡改

2\. **透明度与可追溯性** 链上转账、合约交互、领测试币等全部生成可查交易记录，包含收发地址、Gas、时间、交易状态；传统平台后台数据不对外公开

3\. **数据区分：并非全部内容都上链** - 上链内容：资产、交易行为、智能合约规则（核心价值数据） - 中心化存储：前端页面、图片、普通用户信息，仍走传统服务器

4\. **资产控制权** 传统平台账号资产归平台管控；链上资产由用户私钥全权掌控，平台无法封禁、没收

（二）智能合约商业模式（合约盈利方式） 智能合约类比自动售货机：预设固定规则，自动执行资金分发、扣费逻辑，盈利途径共6类：

1\. **交易手续费** 开发交易、借贷类工具，用户每笔交互抽取手续费

2\. **发行原生代币** 依靠生态流动性、治理权赋予代币价值，通过代币流通获利

3\. **搭建DeFi协议** 开发借贷、质押、币币兑换等去中心化金融基础设施

4\. **NFT/链游道具业务** 主导NFT铸造、发售、版税分成，游戏链上道具发行售卖

5\. **合约开发外包** 承接项目定制合约（ERC20、质押、空投合约等），按项目收取技术服务费，类似网页开发外包

6\. **合约安全审计服务** 为项目排查合约漏洞，属于行业“卖铲子”刚需服务

核心总结 合约只是技术工具，不会自动赚钱；盈利前提：拥有用户使用合约，或客户付费委托开发/审计；项目长期存活关键：商业模式、流量、用户信任、合规经营。
