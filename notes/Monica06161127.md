# 2026-07-07 — Day 2

## 今日学到什么

### 1\. 智能合约部署的完整流程

今天终于动手了——把一个 Solidity 合约真正部署到了 Monad Testnet 上。整个流程：用 Remix IDE 编译 SimpleStorage.sol → 选择 Injected Provider (MetaMask) → 切换到 Monad Testnet → 点击 Deploy → MetaMask 弹窗确认 → 等待交易上链。

**关键洞察**：部署合约没有想象中那么难，难的是理解每一步在做什么。看到合约地址出现在 Deployed Contracts 那一刻，有种"原来我也可以"的感觉。

### 2\. 合约地址 ≠ 交易地址

部署合约会产生一个交易 hash（66 字符），同时生成一个合约地址（42 字符）。之前一直搞混这两个，今天终于分清了：

-   **交易 hash**：记录"我做了一件事"（部署合约）
    
-   **合约地址**：记录"这个合约在哪里"（链上的位置）
    

### 3\. Write 函数要花钱，Read 函数免费

`set(42)` 需要签名 + Gas，`get()` 直接调用就行。这就像——写入是寄快递（要邮费），读取是查快递单号（免费）。这种设计约束其实是在保护网络资源。

### 4\. MetaMask 是"签名器"，不是"执行器"

MetaMask 只负责帮你签名确认，真正的执行是矿工/验证者做的。签名完了还要等区块确认。这让我理解了区块链的去中心化本质——没有单一的服务器在执行，而是全网共识。

### 5\. RPC 直接查询验证

区块浏览器有时候显示不全（比如合约地址被截断），所以用 Monad Testnet RPC 直接查询了交易详情。这种方式拿到的数据比浏览器更完整，也让我理解了"链上数据是公开可查的"这句话的真正含义。

**技术细节**：

```bash
curl -s -X POST https://testnet-rpc.monad.xyz \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"eth_getTransactionReceipt","params":["0xf00f..."],"id":1}'
```

返回的数据是十六进制的，比如 `0x2a` 就是 42。把十六进制转换成十进制的过程，让我第一次直观感受到"链上数据是裸的"——没有中间层帮你翻译，一切都需要自己解读。

### 6\. 区块链上的每一次写入都是有成本的

写入 `set(42)` 的时候，MetaMask 弹窗让我确认 Gas 费。这个体验让我想到——区块链上的每一次写入都是有成本的，这和互联网时代"免费发布"完全不同。这种设计约束其实是在保护网络资源。

### 7\. 合约地址 + ABI：房子的门牌号和钥匙

今天在 Remix 里又学了一个重要概念：**部署合约之后，会同时得到两样东西**——合约地址和 ABI。

**合约地址**就像房子的**门牌号**（比如 `0x321...79c1F`），**ABI** 就像这栋房子的**钥匙**（Application Binary Interface，应用二进制接口）。

ABI 本质上是一份「接口说明书」，告诉外部程序：这个合约有哪些函数、每个函数需要什么参数、返回什么值。没有 ABI，你虽然知道房子在哪（合约地址），但不知道门在哪里、怎么开。

**但为什么平时感觉不到 ABI 的存在？** 因为 Remix 会自动读取 ABI。当你在 Deployed Contracts 里看到已经部署的合约时，Remix 已经帮你把 ABI 加载好了，所以你可以直接点按钮调用函数。

**什么时候需要手动用 ABI？** 当你想在另一个工具（比如 Etherscan、另一个 IDE、或者前端代码）里调用已部署的合约时，就需要手动提供 ABI。这时候你需要：

1.  在 Remix 编译器面板点 **ABI 按钮**，复制 ABI 内容
    
2.  点 **\+ Add Contract**，粘贴合约地址和 ABI
    
3.  就可以重新和这个合约交互了
    

**关键洞察**：你不需要重新部署合约，只要知道门牌号（合约地址）+ 有钥匙（ABI），就可以随时回到原来的地方。这就像你搬了家，但只要还记得老房子的地址，又拿着钥匙，随时可以回去看看。

**💡 核心收获**  
  
**1\. 链上产品 vs 普通互联网产品**  
  

> **普通互联网产品：** 数据存公司服务器，平台能改能删，用手机号注册。  
> **链上产品：** 数据存全球节点网络，不可篡改，用钱包地址掌控身份和资产。

  
**核心区别：** 控制权——互联网是"平台说了算"，区块链是"代码和共识说了算"。  
  
**2\. 交易的生命周期**  
  

```
发起交易 → 网络验证 → 打包上链 → 永久记录
```

  
  
每一步都有对应的 Gas 费用，即使失败也不退还。  
  
**3\. AI 辅助开发的正确姿势**  
  
\- AI 生成初稿，人来做最终判断  
\- 需要检查：权限、逻辑、安全、命名  
\- 保留好的设计，拒绝不合理的地方  
\- **重点不是让 AI 替你做，而是用 AI 辅助你理解**  
  
**4\. DevRel 与 Ops 的关系**
