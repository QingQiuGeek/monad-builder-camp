# DAY 5｜Monad 理解、Build Log 与 Mini Demo 0：我如何总结第一周并选择方向？

今天主要做 Week 1 的收尾整理，也开始思考 Week 2 的方向。今天的学习内容包括 Monad 的高性能应用资料、Week 1 Build Log 整理、Mini Demo 0 准备，以及 Tech / Ops / Research 方向选择。

Monad 这部分让我重新理解了“更快”这件事。它不只是 TPS 数字更高，也会影响产品能不能做更高频的交互。比如游戏、小任务系统、实时状态更新、链上社交、AI Agent 频繁调用工具或发起小额交易，这些场景如果每次交互都很慢、gas 成本很高，用户体验会很差。Monad 想解决的是链上产品的交互密度问题，让更多接近普通 App 的体验有机会搬到链上。

今天也看了 Monad 和 Ethereum 的一些差异。Monad 仍然兼容 EVM，但在交易处理、gas 机制、高吞吐场景和开发实践上有自己的特点。官方文档里也提到，高性能应用不能只依赖链本身快，前端、RPC 调用、索引器、并发请求、gas 估算这些细节都会影响体验。这让我意识到，链上产品的性能不是只看公链参数，开发者自己的设计也很重要。

Week 1 的 Build Log 主要整理这几类内容：课程专用钱包、Monad Testnet、第一笔测试网交易、区块浏览器查询、AI 辅助生成 Solidity 合约、Remix 编译部署、AI Agent 支付和安全分享、Co-learning 记录。回头看这一周，很多概念一开始都很陌生，但通过截图、链接、hash 和笔记保存下来以后，整个学习路径会清楚很多。

今天例会里有一些同学上来分享自己的笔记和项目。我看了一下他们做的东西，感觉都挺厉害的，有链上小宠物，也有链上防诈骗相关的项目，还有一些方向我之前完全没想过。看完之后算是大开眼界，也更直观地看到同样是 Week 1，不同人可以把学习内容延展到很多不同的小项目里。

现在重要的是把 Week 1 的钱包、交易、合约、安全和活动记录整理好，先把 Build Log 做完整。

Week 2 方向我现在更倾向 Ops / 运营。学完这一周之后，我发现 Tech 对代码、合约、部署和调试的要求还是挺高的，暂时不太适合作为主方向。我的网络能力和信息整理能力相对更强，也更适合先从运营、社区、内容整理和活动参与这些事情开始学起。后面可以继续了解技术内容，但 Week 2 主方向先放在 Ops 上。

# **DAY 4｜AI + Solidity + 合约部署：我如何从交易进入最小合约实践？**

今天开始接触最小 Solidity 合约和 Remix 部署流程。我看了智能合约开发、Solidity、Remix、OpenZeppelin 以及 Monad Remix Deployment Guide 相关资料，也梳理了合约源码、ABI、合约地址、read / write function 和 transaction hash 之间的关系。

今天先用 AI 辅助生成一个最小 Solidity 合约，再通过 Remix 编译、检查、部署和交互。AI 可以帮忙起草合约，但代码还是要自己看一遍，至少要知道合约大概做什么、有哪些函数、哪些操作会发起链上交易。

在 Remix 里，合约源码是写在编辑器里的代码，编译之后会生成 ABI 等信息；部署到测试网以后，会得到一个合约地址；read function 更像是读取链上状态，write function 通常会触发交易，需要钱包确认，也会产生 transaction hash。这样一串下来，才算把“写合约”和前几天学到的钱包、交易、区块浏览器联系起来。

今天还参加 / 观看了 7.09「从‘裸奔’到‘受控’：AI Agent 高危案例解析与安全防线构建」分享，以及 7.09 Co-learning。

今天的分享会让我对 AI Agent 的安全风险有了更具体的认识。Alu 老师讲到，Agent 拥有 API、工具、钱包和链上交易权限以后，风险会变得更真实。一次 Prompt Injection、一个恶意插件、一个过大的权限配置，都可能让 Agent 执行泄露密钥、删除数据、发起转账这类高危操作。

这场分享主要讲的是 Agent 的权限和执行边界。开发或使用 AI Agent 时，需要考虑最小权限、敏感操作二次确认、插件和依赖扫描、运行时防护。Web3 场景里还要特别注意钱包、私钥、API Key 和链上交易权限。

# DAY 3｜钱包、安全与第一笔测试网交易：我如何留下第一条链上记录？

今天主要学习了钱包安全和测试网交易相关内容，也跟着任务把课程专用钱包、Monad Testnet 和区块浏览器这些基础流程过了一遍。相比只看概念，实际操作之后会更容易理解钱包、地址、交易和 gas 这些词到底出现在什么地方。

今天最重要的收获是安全意识：课程实验要用专门的钱包，不要拿主力钱包去试。钱包地址可以公开查询，但助记词和私钥不能泄露，也不能截图、上传或放进任何提交材料里。这个和普通互联网账号不太一样，钱包更像是进入链上世界的入口，谁掌握了助记词或私钥，谁就可能控制这个钱包。

我也记录了课程专用钱包和 Monad Testnet 的配置过程，包括添加网络、确认 RPC、Chain ID、MON 和区块浏览器信息。通过区块浏览器查询地址和交易后，我对 transaction hash 有了更直观的理解：它就像一笔链上操作的查询编号，可以用来查看交易是否成功、from / to 分别是谁、value 是多少、用了多少 gas，以及在哪个区块里被确认。

![image.png](https://raw.githubusercontent.com/IntensiveCoLearning/monad-builder-camp/main/assets/HengYuLexus/images/2026-07-08-1783523457811-image.png)

7.08 的「AI Agent 如何拥有支付能力？」分享也刚好和今天的钱包学习有联系。Jack Xiao 老师主要介绍了 AI Agent 在支付场景中的应用，以及目前行业里正在探索的方向。在这之前，我对 Agent 支付了解不多，也不清楚相关产品和协议已经推进到什么程度。听完后我了解到，现在行业里已经在探索 Agent 钱包、Agent Card、payment link、稳定币结算、小额高频支付、预算限制和风控等方案。

晚上的 Co-learning 主要听 Yoona 老师分享自己的学习经历、实习经历、远程工作和 Web3 入门从业经验。整场氛围比较轻松，尤其是听到 Yoona 老师和 Huahua 老师的绝美爱情故事，真的非常欢乐。相比前几场偏技术和概念的分享，这一场更像是在听一位 Web3 从业者讲自己的真实经历，也终于是我能完全听懂的一集了。

这次分享让我对 AI Agent 支付方向的前沿进展有了更具体的认识。Agent 支付不只是让 AI 完成付款动作，还涉及身份、授权、预算、风控和结算等问题。Web3 的钱包、链上身份、稳定币和可验证交易记录，可能会成为这个方向的重要基础设施。接下来我会继续用课程专用钱包完成 Monad Testnet 的测试网操作，保存 transaction hash 和 explorer 截图，并把这些内容整理进 Week 1 Build Log。

# DAY 2｜工具准备与 Builder 身份：我如何开始远程协作？

今天是 Web3 实习计划 Day 2，我主要做了三部分学习：先了解 Web3 远程协作需要准备哪些工具，再听 JackCC 老师关于「普通开发者如何进入以太坊协议层」的分享，晚上又参加了 coooder 老师主持的 Co-learning，重点看了如何用 Codex / AI Coding 部署智能合约。

今天的学习让我感觉，Web3 的工作方式和普通学习不太一样。它不是只在一个平台里听课，而是会同时用到很多工具，比如 X / Twitter 获取信息和参与公开讨论，Telegram / Discord 进入社区和提问，GitHub 记录代码和协作，Notion 或文档工具整理学习过程，再加上 Codex、ChatGPT、Cursor 这类 AI 工具辅助学习和开发。

我现在对 Builder 的理解也更具体了一点。Builder 不一定一开始就能做很复杂的项目，但至少要开始动手，把学习变成可以被看到的 proof of work。比如记录 Build Log、提交学习笔记、保存截图和链接、在 GitHub 上留下代码或文档、把自己遇到的问题和解决过程写下来。这些东西以后可能比单纯说“我学过”更有说服力。

JackCC 老师的分享让我知道，Web3 里还有更底层的方向，比如以太坊协议层、EPF、执行层、共识层、客户端、EIP、specs 和开源贡献。这个内容对我来说比较难，但也让我知道 Web3 不只有 DApp 和智能合约开发，还有协议研究和长期贡献的路线。对普通开发者来说，可以先从理解以太坊这台机器怎么运行开始，再慢慢读资料、复现问题、写笔记，甚至从小 issue 或小 PR 开始贡献。

# DAY 1｜开营与 Web3 入门：我为什么要进入链上世界？

今天是 Web3 实习计划的第 1 天，主要是先了解 Web3 和链上世界的一些基础概念，同时也参加了两场线上活动：19:00 的「DevRel 的成长之路 —— 从 Builder 到生态连接者」和 20:00 的 Co-Learning。说实话，我现在还是 Web3 小白，很多词只是第一次真正系统地看到，比如区块链、以太坊、钱包、地址、交易、gas、智能合约这些。现在还谈不上完全理解，但至少开始知道它们分别大概在链上世界里扮演什么角色。

今天看了 Web3 实习手册里的入门导读、区块链基础概念和以太坊概览，也看了 BuildAnything Freshman 里关于 Vibecoding 和为什么要构建去中心化应用的内容。我的感觉是，Web3 和普通互联网最大的不同在于：很多东西不是只存在某个平台自己的服务器里，而是通过钱包、地址、交易和智能合约记录在链上。用户不再只是用账号登录一个平台，而是用钱包作为自己的链上身份去和不同应用交互。

目前我对几个概念的粗浅理解是：

-   钱包：更像是进入链上世界的身份入口，不只是“放币”的工具。
    
-   地址：可以理解成钱包在链上的公开身份，别人可以通过地址看到相关链上记录。
    
-   交易：不只是转账，也可能是和某个链上应用或智能合约发生交互。
    
-   gas：发起链上操作时需要付出的手续费，类似让网络帮你执行操作的成本。
    
-   智能合约：部署在链上的程序，规则写好以后，用户可以通过交易去调用它。
    

我现在还不太清楚的是：钱包、地址、交易、gas、智能合约这些东西在真实的 Web3 产品里分别具体起什么作用。比如我知道钱包很重要，也知道交易会被记录在链上，但还不太能把这些概念和一个真正的区块链应用联系起来。今天的资料让我对 Web3 和区块链方向更感兴趣了，只是很多东西还需要通过后面的实际任务，比如配置测试网、发交易、查区块浏览器、部署简单合约，才能慢慢理解它们到底怎么用。

晚上 19:00 参加了「DevRel 的成长之路 —— 从 Builder 到生态连接者」。这场分享让我第一次比较具体地听到 DevRel 这个方向。之前我以为 DevRel 可能更像社区运营，但听完之后感觉它其实更像是连接开发者、社区和项目生态的人：既要理解一些技术，也要会沟通、写内容、组织活动、帮助开发者理解项目。对我这种刚入门的人来说，这个方向还比较远，但它让我知道 Web3 里不只有写代码一种角色，也有很多和社区、生态、内容、开发者支持相关的工作。听海哥聊天知道了外语的重要性，也大概知道可以从运营入手再逐步学习技术。插一句，运营最重要的真的是人脉吗哈哈哈哈哈（?）
