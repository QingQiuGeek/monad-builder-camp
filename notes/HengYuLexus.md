# 2026-07-19

今天主要继续学习了区块链运行机制，重点理解了**共识机制、Proof of Work（PoW）、Proof of Stake（PoS）、Validator、Gas 和 Finality** 等概念。

通过学习，我进一步理解了区块链为什么能够在没有中心机构的情况下维护可信账本。PoW 通过算力竞争让矿工获得出块权，而 PoS 则通过质押资产让 Validator 参与区块验证和生产，并利用 Slashing（罚没）机制约束恶意行为。Validator 并不是人工审核交易，而是运行节点软件，根据协议规则自动验证区块，通过共识机制达成一致。

同时也梳理了链上交易的完整流程：

用户发起交易 → 交易进入交易池 → Validator 提议区块 → 其他 Validator 验证 → 共识确认 → 区块上链 → 达到 Finality。

此外，我也理解了 Gas、Chain ID、RPC、Faucet、Explorer 等基础工具在实际使用中的作用。Gas 不仅用于奖励网络维护者，也用于防止垃圾交易；Explorer 可以帮助用户查看链上状态；RPC 则是应用与区块链交互的接口。

通过今天的学习，我对区块链从“概念理解”进一步进入了“系统运行逻辑”的理解。

# DAY 3｜钱包、安全与第一笔测试网交易：我如何留下第一条链上记录？

今天主要学习了钱包安全和测试网交易相关内容，也跟着任务把课程专用钱包、Monad Testnet 和区块浏览器这些基础流程过了一遍。相比只看概念，实际操作之后会更容易理解钱包、地址、交易和 gas 这些词到底出现在什么地方。

今天最重要的收获是安全意识：课程实验要用专门的钱包，不要拿主力钱包去试。钱包地址可以公开查询，但助记词和私钥不能泄露，也不能截图、上传或放进任何提交材料里。这个和普通互联网账号不太一样，钱包更像是进入链上世界的入口，谁掌握了助记词或私钥，谁就可能控制这个钱包。

# DAY 1｜开营与 Web3 入门：我为什么要进入链上世界？

今天是 Web3 实习计划的第 1 天，主要是先了解 Web3 和链上世界的一些基础概念，同时也参加了两场线上活动：19:00 的「DevRel 的成长之路 —— 从 Builder 到生态连接者」和 20:00 的 Co-Learning。说实话，我现在还是 Web3 小白，很多词只是第一次真正系统地看到，比如区块链、以太坊、钱包、地址、交易、gas、智能合约这些。现在还谈不上完全理解，但至少开始知道它们分别大概在链上世界里扮演什么角色。

今天看了 Web3 实习手册里的入门导读、区块链基础概念和以太坊概览，也看了 BuildAnything Freshman 里关于 Vibecoding 和为什么要构建去中心化应用的内容。我的感觉是，Web3 和普通互联网最大的不同在于：很多东西不是只存在某个平台自己的服务器里，而是通过钱包、地址、交易和智能合约记录在链上。用户不再只是用账号登录一个平台，而是用钱包作为自己的链上身份去和不同应用交互。

目前我对几个概念的粗浅理解是：

-   钱包：更像是进入链上世界的身份入口，不只是“放币”的工具。
    
-   地址：可以理解成钱包在链上的公开身份，别人可以通过地址看到相关链上记录。
    
-   交易：不只是转账，也可能是和某个链上应用或智能合约发生交互。
    
-   gas：发起链上操作时需要付出的手续费，类似让网络帮你执行操作的成本。
    
-   智能合约：部署在链上的程序，规则写好以后，用户可以通过交易去调用它。
    

我现在还不太清楚的是：钱包、地址、交易、gas、智能合约这些东西在真实的 Web3 产品里分别具体起什么作用。比如我知道钱包很重要，也知道交易会被记录在链上，但还不太能把这些概念和一个真正的区块链应用联系起来。今天的资料让我对 Web3 和区块链方向更感兴趣了，只是很多东西还需要通过后面的实际任务，比如配置测试网、发交易、查区块浏览器、部署简单合约，才能慢慢理解它们到底怎么用。

晚上 19:00 参加了「DevRel 的成长之路 —— 从 Builder 到生态连接者」。这场分享让我第一次比较具体地听到 DevRel 这个方向。之前我以为 DevRel 可能更像社区运营，但听完之后感觉它其实更像是连接开发者、社区和项目生态的人：既要理解一些技术，也要会沟通、写内容、组织活动、帮助开发者理解项目。对我这种刚入门的人来说，这个方向还比较远，但它让我知道 Web3 里不只有写代码一种角色，也有很多和社区、生态、内容、开发者支持相关的工作。听海哥聊天知道了外语的重要性，也大概知道可以从运营入手再逐步学习技术。插一句，运营最重要的真的是人脉吗哈哈哈哈哈（?）
