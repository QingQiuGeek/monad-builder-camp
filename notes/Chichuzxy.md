# Web3 简历撰写学习笔记

基于 [web3intern.xyz](http://web3intern.xyz) 及行业资料整理 | 2026 年 7 月 18 日

## 一、核心理念：Web3 简历的 "去中心化" 思维

Web3 招聘看重的是可验证的贡献与持续参与度，而非静态的头衔、学历或流水账式的职责。你的链上数据、社区足迹、协作提案、黑客松奖项、研究报告等，都是最好的 "工作证明"。

三个核心要点：

表格

| 要点 | 说明 |
| --- | --- |
| 你的价值 | 取决于持续产出的链上痕迹（运营 / 开发）和在社区中建立的信任度 |
| 稀缺性 | 源于独一无二的组合技能（如 DeFi 高手 + meme 图高手 + 链上治理高手） |
| 贴合度 | 你的价值与技能与该职位需求的耦合程度 |

一份优秀的 Web3 简历应像区块链一样 —— 透明、可追溯、可量化。

## 二、简历结构（推荐）

表格

| 模块 | 核心内容 |
| --- | --- |
| 个人信息 | Telegram、Twitter/X、GitHub、Discord Tag、邮箱、电话、微信 |
| 学历 | 本科 / 研究生 / 博士经历，与岗位相关则重点突出，不相关则弱化 |
| 详细经历 | 选取 3 段以内最相关的经历，采用 STAR 模型（情境 - 任务 - 行动 - 结果）描述 |
| 技能栈 & 工具 | Solidity、Galxe、Dune 等，每项技能标注具体产出示例或项目链接 |
| 社区与 DAO 角色 | Delegate、Core Contributor、Bounty Hunter 等，附 Snapshot 投票链接或区块高度 |
| 奖项 | 黑客松奖项等与 Web3 高度相关且有官方背书的荣誉 |

**隐私提示**：涉及链上证明时，使用专门的展示地址，不公开主钱包及大额余额；优先提供可验证的交易链接、PR 链接或作品页。

## 三、核心写作原则："三板斧"

### 1️⃣ 链上战绩 > 宏大叙事

❌ 避免空泛描述：不要只写 "负责社区增长"

✅ 量化并验证：应改为 "设计 3 条 Galxe 任务，使测试网跨链交互数 30 天内从 4,000 → 17,500（+337%），相关交易哈希：0xabc…123"

✅ 提供证据：将 PR、提案、黑客松提交等附上链接

Web3 简历的底层逻辑是可验证 —— 链上行为能追溯，社区贡献能查证，数据结果能截图。

### 2️⃣ 参与度曲线 > 从属关系

Web2 简历强调 "在某某公司 / 部门任职"

Web3 看重的是持续贡献，而非你曾在哪家公司待过

### 3️⃣ 技能证明 > 技能列表

不要罗列一堆不熟悉的技能。针对每项技能，最好能标注具体的产出示例或项目链接，证明你的实战能力。

## 四、实用 Tips

✅ **第一屏要精炼**

只写三样东西：①投什么岗位 ②初筛信息（时区、能否远程等） ③核心亮点（2-3 行）

✅ **删掉 "大厂黑话"**

❌ "负责用户拉新留存"

✅ "从 0-1 搭建 Discord 社群，通过 Zealy 任务体系提升活跃度，策划白名单活动实现 Twitter 涨粉 10k+"

Web3 不看你懂不懂 "拉新"，看你懂不懂运营工具。

✅ **简历控制在一页**

格式简洁美观，与岗位无关的经历可以不写。

✅ **无经验也能写**

-   自学经历 → "系统学习区块链基础知识，完成 Solidity 入门课程，能独立阅读智能合约代码"
    
-   研究资料 → "长期关注 Web3 行业媒体，并在个人账号上总结学习笔记 20+ 篇"
    
-   链上体验 → 体验过哪些钱包 / 产品、做过哪些操作（充值、swap、跨链）、是否参与过测试版体验
    

✅ **工具推荐**

可使用在线简历生成器（如 Canva 模板）制作简历，方便填写且简洁美观。

## 五、一句话总结

把简历写成一份能被验证的能力证明，比堆砌形容词管用得多。

# Web3 求职学习笔记

## 一、选招聘平台的核心标准

判断一个招聘平台靠不靠谱，记住三个关键词：及时、真实、准确。

-   **及时**：岗位的 HC（招聘名额）还在开放中，不是二手三手传过来的过时信息
    
-   **真实**：岗位确实存在，不是骗流量的假岗位
    
-   **准确**：岗位描述、薪资范围、工作地点、面试流程、KPI 预期都跟实际情况匹配 —— 很多求职者的 "落差感" 就来自信息不准
    

## 二、推荐招聘平台

以下平台在 "及时、真实、准确" 三方面做得比较好：

表格

| 平台 | 特点 |
| --- | --- |
| web3career.build | Web3 实习计划官方平台，适合想系统参与实习的同学 |
| Web3.career | 全球性平台，职位更新快、覆盖面广，涵盖 DeFi、NFT、元宇宙等 |
| DeJob（现 dejob.ai） | 专注 Web3 和远程招聘，支持免费发布职位，适合国内项目和求职者 |
| SmartDeer | 新兴平台，注重人才质量和项目可靠性，覆盖技术、运营、市场等多职能 |

## 三、其他信息渠道

除了专业招聘平台，还可以通过以下渠道获取机会：

-   **DAO 社区**：LXDAO、ETHPanda 等会整理社区招聘信息和 Bounty 任务
    
-   **项目方官方社群**：Discord / Telegram 会发布招聘信息，尤其早期项目
    
-   **Twitter(X)**：关注 KOL、创始人、投资机构，他们会分享招聘信息
    
-   **LinkedIn**：成熟区块链公司和传统企业转型部门会在上面发布职位
    
-   **行业媒体与研究机构**：CoinDesk、Messari 等，从中发现新兴项目
    
-   **线下活动 / 线上 AMA**：直接接触项目方，了解招聘需求
    

## 四、项目可靠性打分表（满分 120 分）

用这 12 项给项目方打分，每项 0-10 分：

1.  项目方愿景（要做什么、怎么实现）
    
2.  项目方声誉（负面舆情多不多）
    
3.  创始人背景与权力结构（可在 [rootdata.com](http://rootdata.com) 查）
    
4.  是否上市及交易所层次
    
5.  办公地点（美国 / 新加坡 / 日本 / 香港 / 不明？）
    
6.  能否在 LinkedIn 上搜到
    
7.  总市值排名
    
8.  创始年份
    
9.  合规性（是否在 Coinbase 上市 / 有无牌照 / 是不是庞氏变种）
    
10.  发言人立场 / 观点 / 价值观是否与你匹配
     
11.  获得了哪些投资机构支持
     
12.  社区运营情况（官方推特 / Discord 更新频率、互动情况）
     

**评估建议：**

-   低于 72 分（及格线）要格外小心
    
-   如果项目只是套概念（如 AI）、靠拉人头、没有实际业务、查无此号，要警惕可能是诈骗
    
-   拿不定主意可以在 LXDAO 或 ETHPanda 社区问询
    
-   0 经验新人可考虑从分数稍低但有潜力的项目开始积累经验
    

## 五、识别招聘骗局

**常见骗局特征：**

-   要求先付费（培训费 / 工具费 / 押金 / 买代币）
    
-   要求安装不明软件或插件并开启高权限
    
-   公司无法核验（官网、LinkedIn、GitHub 缺失或对不上）
    
-   岗位描述宽泛且承诺高回报（"零门槛高收益"）
    
-   用 Telegram/WhatsApp/ 私人邮箱沟通，不用公司域名邮箱
    

**自我保护清单：**

-   核验主体：通过官网、LinkedIn、GitHub、工商信息验证身份
    
-   拒绝付费招聘：正规招聘不应要求预付任何费用
    
-   只用官方渠道沟通
    
-   不运行未知程序
    
-   存疑时通过官网公开联系方式再次确认
    

## 六、职业发展路径

### 运营向

# 学习笔记：用人方视角 —— 如何成为靠谱的 Web3 实习生

> 来源：Bruce（LXDAO & ETHPanda 联合发起人，十余年全栈开发与团队管理经验）访谈

## 一、什么是 “靠谱”？三个关键词

Bruce 对 “靠谱” 的定义可以浓缩为三个词：

-   **可预期**：你说 3 天做完，第 3 天就要能看到结果或至少知道进度。最怕最后一刻才说 “做不完”。
    
-   **可沟通**：遇到问题别闷头自己扛，主动同步进度和卡点，别等别人来问你。
    
-   **可复盘**：做完任务主动思考哪里好、哪里能改进，同样的问题别反复出现。
    

> 具备这三点的人，团队会主动给更重要的任务和更多机会。

## 二、“不靠谱” 的三种典型表现

Bruce 提到的 “血泪教训”：

1.  **人间蒸发**：约好开会找不到人，无解释 —— 一次就足以失去信任。
    
2.  **明日复明日**：今天说明天能做完，明天又说还要一天，且不主动报告，全靠别人追问。
    
3.  **闷头工作**：一周不沟通，以为在专心做事，结果 deadline 到了才说做不完，此时已来不及调整。
    

> 核心问题都是没做到 “可预期、可沟通、可复盘”。

## 三、任务时间评估：“经验倍数法”

对新人特别实用的方法：

表格

| 熟悉程度 | 时间系数 |
| --- | --- |
| 技术栈非常熟练 | × 1.25 |
| 做过类似的东西 | × 1.5 |
| 完全没接触过 | × 2 |

**示例**：你觉得需要 3 天，完全不熟悉就估 6 天。

此外还要做到：

-   **拆分任务**：别说 “做个网站”，要说 “设计数据库→搭前端框架→实现登录” 这样具体的步骤。
    
-   **加缓冲**：总时间外再加 10%-15%。
    
-   **提前同步**：开始前先跟 Mentor 确认计划是否合理。
    

## 四、沟通频率与方式

分阶段来看：

-   **初期**：每天发简短日报 —— 今天做了什么、遇到什么问题、明天计划做什么。三段就够了。
    
-   **磨合后**：可减少频次，但卡住**超过半小时**没思路就要及时求救。
    

> **原则**：先自己试着解决（AI、Google 查一下），实在不行再问人。既体现主动性，也不过度依赖别人。

## 五、技术深度 vs 交付节奏

核心思路：

1.  **先明确硬性 deadline**—— 不可妥协。
    
2.  **写方案文档或画流程图**（半小时即可）—— 让 Mentor 提前发现方向问题，避免做无用功。
    
3.  **分层迭代**—— 先把核心功能跑通，再考虑性能优化、错误处理。时间不够也有可交付的版本。
    

## 六、最看重的软技能

技术可以培训，但软技能反映潜力：

1.  **清晰沟通**：能用结构化语言清晰表达。
    
2.  **深度思考**：能从技术、团队、市场等多个角度分析问题本质。
    
3.  **反思能力**：遇到挑战后能否主动复盘总结（面试难验证，试用期可观察）。
    

## 七、进度拖延或质量出问题怎么办

Bruce 的辅导思路：

1.  **先了解情况**—— 让实习生说明遇到的问题和已尝试的解决方案。
    
2.  **一起分析根因**，制定下一步具体计划。
    
3.  **设定观察窗口**（如一周），看是否有明显提升。
    

**区分能力和态度**：同样错误反复出现且不反思→态度问题；错误类型各不相同且能快速吸取教训→能力短板，可通过额外指导解决。

## 总结

> 知识和技能需要时间积累，但**靠谱的特质可以加速这个过程**。

Web3 行业开放包容，愿意学习和付出就有机会。核心就一句话：**可预期、可沟通、可复盘**—— 做到了，团队自然会把更重要的任务交给你。

# Jason：从 Web2 到 Web3 的转型之路 —— 学习笔记

## 一、人物背景

Jason，2023 年计算机硕士毕业（深大），现为国内头部互联网公司 C/C++ 开发工程师，曾获阿里、华为全职 Offer。研二时跑过自己的以太坊节点，2016-2017 年上大学期间就对虚拟货币有过接触。

## 二、进入 Web3 的契机

毕业前后，与在新加坡读研的同学交流发现那边 Web3 氛围很好，加上研友一起系统做链上研究。公司曾做过区块链板块的 FT 项目（做到国内第二），让他确信方向有前景，于是决定 "跨过去"。

## 三、求职渠道：内推 > 海投

核心渠道是**内推 + 线下社群**。Web3 组织扁平，通过参加线下活动结识朋友后互相内推，比在招聘平台 "海投" 更高效，人也更真诚。

## 四、给在校生的实习建议

1.  **多实习**：本科 1 段 + 硕士 2 段实习经历，在萧条期仍能拿到大厂 Offer，实习是最大加分项。
    
2.  **打牢基础**：面试几乎都考最基础的计算机知识（组成原理、IO、CPU / 内存分片等）。实践中串起这些知识，胜过死背 "八股"。
    

## 五、面试重点：底层原理

面试国内 Top2 交易所专家岗时，整场一小时几乎都在问底层基础：**CPU 分片、内存布局、缓存机制、文件 IO 流程**等。Web2 和 Web3 都运行在 CPU / 服务器上，理解这些原理对系统优化至关重要。

## 六、不同岗位的转型难度

表格

| 岗位类型 | 转型难度 | 关键点 |
| --- | --- | --- |
| 前端开发者 | 门槛最低 | 换包调接口即可 |
| 底层开发者（C/C++ 等） | 语言迁移快 | 但更需要产品思维 |

Web3 还缺一款真正的 "超级应用"，**理解业务和生态比单纯写代码更重要**。

## 七、AI 时代的核心竞争力

大模型可以帮忙改写或生成代码，但**无法替你理解业务场景、做产品决策**。领悟产品逻辑、主导系统落地，才是不可替代的核心价值。

## 八、不同背景小白的入行路径

-   **商科背景**：利用调研与金融知识，学会用 SQL/Superset 等工具做链上数据归因与洞察。
    
-   **理工背景**：已学 C/C++，转学 Solidity、Rust 相对轻松；跟着 GitHub/Ethereum 官方教程做项目。
    
-   **文科背景**：不必硬学代码，可发挥 BD、社区运营、内容策划优势 ——Web3 同样需要市场、媒体、展会人才。
    

核心原则：**定位自身强项，与行业痛点对接。**

## 九、"实习 ↔ 基础" 正向循环

先找一份相关实习 → 真实项目中发现基础短板 → 回校夯实基础 → 再申请更有挑战的实习。每轮都能看见成长与反馈，形成闭环。

## 十、一句话总结

> 先认清自己的优势，再把它映射到行业真正缺口上；学习语言只是起点，**理解业务才能立足**。
> 
> Web3 仍在早期，机会很多。**从小处切入、快速试错**，比盲目追风更重要。

## 核心 Takeaways

1.  **基础为王**：计算机底层原理（CPU / 内存 / IO）是面试和工作的核心，Web2 和 Web3 通用。
    
2.  **业务思维**：在 Web3，理解业务和生态比写代码更重要 —— 这也是 AI 难以替代的价值。
    
3.  **实践闭环**：实习中发现短板 → 回校补基础 → 再实习，形成正向循环。
    
4.  **扬长避短**：不必硬磕代码，找准自己的专业优势与行业需求的交汇点。

AI 做不到：

-   拆解复杂业务需求并校验正确性
    
-   处理公共样式级联、跨页面依赖
    
-   替代编写技术文档、覆盖场景、拉齐多部门
    

Logic 实际怎么用 AI：

-   生成难记的渐变或阴影 CSS
    
-   接口 JSON 自动转 TypeScript 类型
    
-   画时序图，节省白板讨论时间
    
-   贴 Sentry 错误栈，让 AI 先给诊断思路
    

复杂组件与跨文件依赖仍需自己把关。AI 是 “好帮手”，但远没到能取代人的程度。

# Web3 安全与合规・学习笔记

## 一、中国监管底线：什么不能碰

中国对虚拟货币的监管基调是 “全面封堵金融属性，有限容忍技术创新” 。Web3 作为技术本身不违法，但一旦涉及代币发行、融资、交易、矿池运营，就极易触碰红线。

明确禁止的行为：

-   ICO / IEO / IDO 等任何形式的代币融资
    
-   虚拟货币交易所运营及撮合服务
    
-   境外交易平台向境内居民提供服务
    
-   虚拟货币作为支付工具
    
-   虚拟货币 “挖矿” 活动（已被列为淘汰类产业）
    

代币不管叫 “积分” 还是 “治理 Token”，只要具备融资功能或可流通性，都可能构成非法金融行为。技术人员参与代币模型设计、合约部署，也可能被视为 “共同行为人”——“只是写代码” 不能免责。

## 二、四大刑事风险（重点）

### 1\. 开设赌场罪

链游中常见的 “充值 — 抽奖 — 提现” 闭环，容易被认定为开设赌场。有提现功能的应用尤其危险 —— 最高检专门强调过这一点。

真实案例： BigGame 赌博平台（EOS 链），开发了 “掷骰子”“红黑大战” 等游戏，2 万用户投注，赌资 3.3 亿 EOS。从运营到开发、产品设计、客服共 10 人全部被判刑，主犯获刑五年九个月。

### 2\. 非法经营罪

用虚拟货币做 “地下换汇” 媒介 —— 人民币→USDT→外币 —— 绕过外汇监管，可能构成非法经营。

真实案例： 郭某钊搭建平台，以 USDT 为媒介提供外币与人民币汇兑服务，非法兑换 2.2 亿元，被判非法经营罪，获刑五年。

### 3\. 非法吸收公众存款罪

以 “挖矿”“投资返利” 等名义吸收公众资金（包括虚拟货币），都可能构成此罪。

真实案例： AIP 平台卖 “矿机”、释放积分、兑换代币，非法吸收公众存款 1.16 亿元。一位商学院副院长（受雇佣）因参与教学宣传，也被判非法吸收公众存款罪。

### 4\. 组织、领导传销活动罪

“邀请返利”“多级推广”“层级返佣”—— 只要以发展人员数量作为计酬或返利依据，就可能触犯传销罪。

真实案例： PlusToken 平台，以区块链概念包装，要求交数字货币作为门槛费，鼓动发展下线，最终被认定为传销。关键点： 技术开发人员如果明知软件用于传销仍提供开发，会被认定为共犯，而非仅仅 “帮信罪”。

### 5\. 洗钱罪

虚拟货币的匿名性使其成为洗钱工具。如果交易对手的资金涉及诈骗、赌博等犯罪，你可能在不知情的情况下卷入 “协助洗钱”。

真实案例： 陈某某明知是金融诈骗所得，仍将资金转换为虚拟货币并转往境外，被认定洗钱罪。

## 三、入职风险：找工作也得防坑

Web3 项目多采用境外注册（新加坡、开曼、BVI 等），境内无实体公司。这意味着：

-   无劳动合同保障：无法签有效《劳动合同》，无法缴纳五险一金
    
-   社保缺失影响生活：落户、购房、贷款、子女教育都可能受影响
    
-   薪酬结构风险：“人民币 + Token” 或 “全 USDT” 模式中，Token 部分若被认定为工资支付可能无效；若不被认定，则拉低社保基数
    
-   出金风险：C2C 卖币换人民币，若对方资金涉赌涉诈，你的银行卡可能被冻结，甚至涉嫌 “帮信罪” 或 “掩饰、隐瞒犯罪所得罪”
    

入职前建议：审查项目白皮书、代币分发机制、收益模型、是否面向中国大陆用户、是否含返利结构。拿不准时找专业律师。

## 四、网络安全：别被 “钓鱼”

常见攻击方式

### 1\. 钓鱼攻击（最常见）

伪造官网、邮件、社交账号，诱导你点击链接、输入私钥 / 助记词。

典型案例：冒充 HR 发 “[面试专用软件.zip](http://面试专用软件.zip)”，安装后木马上传浏览器 Cookie 和桌面文件；冒充学校发 “奖学金空投”，要求连接钱包签名，结果钱包被掏空。

### 2\. 恶意软件 / 木马

伪装成面试软件、学习资料、破解工具。典型功能：剪贴板劫持 —— 你复制钱包地址，粘贴时被悄悄换成黑客的地址。

### 3\. 社交工程攻击

冒充 HR、导师、学长学姐，获取信任后诱导操作。

### 4\. 供应链攻击

恶意浏览器插件、开源库后门 —— 即使是 “正规渠道” 下载的也可能中招。

### 5\. 地址污染

剪贴板劫持 + 输入框监听，转账时地址被替换。

## 五、防护清单（精简版）

**面试 / 实习：**

-   只用 Zoom/Teams/ 腾讯会议等公开工具，拒绝任何 “专用面试软件”
    
-   核实邮箱域名、多渠道确认面试邀请
    

**钱包 / 资产：**

-   助记词 / 私钥只离线保存，绝不截图、不上传网盘
    
-   任何要求 “连接钱包”“签名验证” 的网页，99% 是骗局
    
-   转账前核对完整地址，不只检查后几位
    

**账号安全：**

-   开启 2FA（优选谷歌验证器或硬件密钥，短信易被劫持）
    
-   使用密码管理器，每个平台独立密码
    
-   邮箱安全重于一切 —— 邮箱被盗等于所有账号沦陷
    

**软件安装：**

-   只从官网或官方应用商店下载
    
-   浏览器插件控制在 3 个以内
    
-   安装前查口碑、查 MD5、核对数字签名
    

**应急：** 怀疑中招→立即断网、截屏、记录时间点。

* * *

### 一句话总结

在中国做 Web3，不碰融资、不碰交易、不碰挖矿、不碰返利；找工作先查项目合法性；日常上网不装陌生软件、不连陌生钱包、不信陌生人。技术无罪，但参与有罪 —— 别以为 “只是写代码” 就能免责。

# Web3 社区运营学习笔记

> 来源：[https://web3intern.xyz/zh/community-intern/](https://web3intern.xyz/zh/community-intern/)

## 一、社区运营到底是干啥的？

说白了，Web3 社区运营就是**在项目和用户之间搭桥**—— 让用户知道项目在干嘛，让项目知道用户在想啥。

### 1\. 日常内容与社群维护

-   定期发推特、公众号，别让账号长草
    
-   群里有人问问题就答，有人发广告就踢，没事抛点话题让大伙聊起来
    

### 2\. 内容发布与互动引导

-   活动预热、AMA 提醒、Space 开播通知要提前发
    
-   发的内容得跟项目调性搭边，发完还要引导大家讨论、转发
    

### 3\. 活动策划与组织

-   **线上**：Twitter Space、课程分享、线上黑客松
    
-   **线下**：Meetup、Workshop、黑客松
    
-   **流程**：策划主题 → 邀嘉宾 → 设计海报推文 → 活动中主持控场 → 结束后总结发稿
    

### 4\. 对外合作与社区联动

-   跟其他项目社区搞联合活动（联名 AMA、联合办活动）
    
-   跟 KOL（关键意见领袖）处好关系，求转发求曝光
    
-   跟媒体保持联系，重要活动争取快讯报道

## 三、实战模板（直接抄作业）

### 线上活动（Twitter Space）执行模板

**背景**：以 ETHPanda 在 Pectra 升级时办的 Space 为例

**活动目标**：宣传以太坊升级 + 拉新（目标：Twitter 涨粉 200+，社群加 100+）

**时间线**：

-   **T-5 ~ T-4 天**：定主题、邀嘉宾（至少 4 位）、收集嘉宾信息（推特号、头像、Title）、设计海报和文案
    
-   **T-4 ~ T-2 天**：推特发第一波宣发、请嘉宾帮忙转发、同步到 Telegram 和微信群
    
-   **T-1 天～活动当天**：发第二波预热帖
    
-   **活动当天**：提前 30 分钟提醒主持人和嘉宾，提前 10 分钟在群里提醒社区
    
-   **Space 流程**：主持人开场 → 嘉宾自我介绍 → 嘉宾问答（每人 15 分钟轮流）→ 观众提问（20 分钟，提问者奖 10U）
    
-   **活动后 7 天内**：复盘数据（浏览数、在线人数、收听量），总结可优化项
    

### 线下活动执行模板

**策划阶段（提前 4-5 周）**：

-   定主题和形式（ keynote /panel/workshop / 黑客松 等）
    
-   定目标（为什么办？给谁办？预期多少人？）
    
-   做预算（场地、嘉宾差旅、物料、茶歇……）
    
-   拉分工表（用 Notion 或 Google Sheet 管起来）
    
-   查合规（大型活动要不要备案？）
    

**筹备阶段（提前 2-4 周）**：

-   定议程、邀嘉宾、收资料
    
-   设计视觉物料（嘉宾欢迎图、倒计时、场地导引图、易拉宝、胸卡……）
    
-   排宣发日历（哪天发什么、谁负责）
    
-   找媒体和合作社区联动，入驻活动日历（Luma、ETHGlobal 等）
    
-   采购周边（T 恤、帽子、贴纸、徽章……），留够物流时间
    

**执行阶段（活动当天）**：

-   提前 2 小时以上布置场地（背板、签到台、摄影、设备）
    
-   嘉宾接待、观众签到
    
-   每个模块安排专人负责（签到、灯光、摄影、控场），设应急联系人
    

**复盘阶段（活动后 1 周内）**：

-   2 天内出 Recap 内容（推特 Thread、公众号文章、图片合集、短视频）
    
-   分析数据（到场人数 vs 报名人数、曝光量、互动量）
    
-   写总结报告（哪些做得好、哪些要改、嘉宾和观众的建议）
    

## 一句话总结

> Web3 社区运营 = **会发内容 + 会聊社群 + 会办活动 + 会用工具**，核心是让社区里的人觉得这个项目值得跟。

# 智能合约开发

## 一、DApp 架构（四个核心部分）

表格

| 组件 | 通俗理解 | 作用 |
| --- | --- | --- |
| 前端 | 用户看到的网页 / App 界面 | 用 React/Vue 等搭建，通过钱包连接区块链 |
| 智能合约 | 跑在区块链上的 “自动执行程序” | 定义业务逻辑，用 Solidity 编写，部署后不可篡改 |
| 数据检索器（Indexer） | 链上数据的 “搬运工” | 监听合约事件（Event），把数据同步到传统数据库，方便前端快速查询 |
| 区块链 + 去中心化存储 | 数据和文件的 “公共账本 + 硬盘” | 存交易记录和状态；大文件（图片等）存 IPFS/Arweave |

## 二、DApp 开发流程

1.  **需求分析**：定功能、选链（以太坊 / Solana/Polygon 等）、设计交互
    
2.  **智能合约开发**：写合约 → 写测试 → 安全审计
    
3.  **检索器开发**：用 Ponder/Subgraph（TypeScript）捕获链上事件入数据库
    
4.  **前端开发**：连接钱包（MetaMask）、展示数据、处理交易签名
    
5.  **与区块链交互**：用 Viem / Ethers.js 等库读取数据、发送交易
    
6.  **部署上线**：合约用 Hardhat/Foundry 部署；前端可放 IPFS 或 Vercel
    

## 三、开发环境搭建

**基础工具**：Node.js + npm/yarn + Git

**两种主流框架**：

表格

| 框架 | 特点 | 核心工具 |
| --- | --- | --- |
| Foundry | Rust 实现，速度极快 | forge（构建 / 测试）、anvil（本地节点）、cast（命令行交互） |
| Hardhat | 老牌框架，生态成熟 | 一键初始化项目，内置本地节点 |

**辅助工具**：Remix（网页 IDE，快速测试）、OpenZeppelin（安全合约库）

## 四、RPC 节点（通俗理解）

**RPC 节点就像银行的 ATM 机**—— 你通过它查余额（读链上数据）、转账（发交易），不同的 ATM 机对应不同的服务商。

**主流 RPC 服务商**：

表格

| 服务商 | 适用场景 |
| --- | --- |
| Alchemy | 生产环境、企业级（稳定性高） |
| Infura | 开发测试、中小项目 |
| Public Node | 学习测试（完全免费） |

**最佳实践**：API Key 用环境变量存，别提交到 Git；实现重试和故障转移机制

## 五、Solidity 基础速览

### 数据类型

-   **基础**：`bool`、`uint256`（无符号整数）、`address`（地址）、`string`（字符串）
    
-   **复合**：数组（`uint[]`）、映射（`mapping(address => uint256)`）、结构体（`struct`）
    

### 函数可见性（由松到严）

表格

| 修饰符 | 谁可以调用 |
| --- | --- |
| public | 任何人（内部 + 外部） |
| external | 仅外部调用（gas 更省） |
| internal | 当前合约 + 子合约 |
| private | 仅当前合约 |

### 函数状态修饰符

-   `view`：只读不改状态
    
-   `pure`：既不读也不改（纯计算）
    
-   `payable`：可以接收 ETH
    

### 事件（Event）

相当于在链上 “发广播”，记录重要状态变化，前端和检索器可以监听

## 六、常见安全问题（⚠️重点）

### 1\. 重入攻击（Reentrancy）

**通俗理解**：你在银行取钱，柜员还没记账就把钱给你了，你拿到钱后立刻再取一次，反复薅羊毛。

**防护**：

-   **CEI 模式**：先检查（Check）→ 先更新状态（Effects）→ 最后外部交互（Interactions）
    
-   **重入锁**：用 `noReentrant` 修饰符加锁
    

### 2\. 访问控制缺失

**问题**：关键函数没加权限，任何人都能调用（比如随便取走合约里的钱）

**防护**：用 `onlyOwner` 修饰符或 OpenZeppelin 的 `AccessControl` 做基于角色的权限控制

### 3\. 整数溢出

**问题**：旧版 Solidity（<0.8.0）不会检查溢出，最大值 + 1 会变成 0

**防护**：用 Solidity 0.8.0+（自带溢出检查），或加逻辑上限

## 七、实战：链上留言板

用 Remix 部署一个简单合约，核心功能：

-   `leaveMessage(string)`：发留言
    
-   `getMessage(address, index)`：查某人的第 N 条留言
    
-   `getMessageCount(address)`：查某人留言总数
    

**部署步骤**：写合约 → 编译 → 选环境（本地 VM 或测试网）→ 点 Deploy

## 八、测试网部署

**为什么用测试网**：和主网一样但代币无真实价值，免费练手

**Sepolia 测试网**：目前最常用的测试网，与主网最相似

**流程**：MetaMask 切到 Sepolia → 从水龙头领测试 ETH→ Remix 选 "Injected Provider - MetaMask" → 部署→ 在 Etherscan 上查看

## 九、前端整合

**交互流程**：前端检测钱包 → 请求授权 → 用 ABI 创建合约实例 → 调用函数 → 钱包签名 → 广播交易 → 更新界面

**技术栈**：

-   交互库：Viem / Wagmi（新项目推荐）、Ethers.js/ Web3.js（了解即可）
    
-   钱包连接：RainbowKit、ConnectKit 等

# Web3 行业核心领域学习笔记

**整理日期：2026 年 7 月 8 日**

## 概述

区块链技术已从单一的加密货币支付系统，发展为涵盖金融、艺术、组织治理和社区文化的多元生态系统。本篇笔记围绕四大核心领域展开：**DeFi（去中心化金融）、NFT（非同质化代币）、DAO（去中心化自治组织）与 Meme（模因币）**。

* * *

## 一、DeFi：去中心化金融

**全称**：Decentralized Finance

**定义**：基于区块链技术搭建的金融体系，目标是提供不依赖传统银行或金融中介的服务（借贷、交易、支付等）。人人可自由参与，无需审批、无需信任中介机构。

### 典型案例：Uniswap

Uniswap 是去中心化交易所（DEX）的代表项目，由 Hayden Adams 于 2018 年创立，核心创新是引入**自动做市商（AMM）** 模式。

核心运行机制

1.  **恒定乘积公式：x × y = k**
    
    -   x、y 分别代表流动性池中的两种加密货币（如 ETH 和 USDC）
        
    -   k 为常数，交易前后数值保持不变
        
    -   当池中一种资产数量变化时，另一种资产数量自动调整，维持 k 值恒定
        
2.  **价格与滑点**
    
    -   池内价格与外部市场偏离时，会被套利交易逐步拉回均衡价格
        
    -   单笔交易规模相对池子过大时，会出现明显的价格滑点
        
3.  **流动性提供者（LP）**
    
    用户向流动性池存入资金即可成为 LP，赚取平台交易手续费收益。
    

* * *

## 二、NFT：非同质化代币

**全称**：Non-Fungible Token

**定义**：代表独一无二的数字资产，每个代币不可互换，与比特币、以太坊等同质化代币有本质区别，核心属性是唯一性和稀缺性。

**应用场景**：数字艺术、收藏品、游戏道具、虚拟土地等。

* * *

## 三、DAO：去中心化自治组织

**全称**：Decentralized Autonomous Organization

**定义**：通过智能合约运行的组织形式，成员通过持有治理代币参与决策投票，所有规则公开透明地写入链上。

**核心特点**：去中心化治理、社区驱动、规则不可篡改。

> 注：该部分内容需进一步展开阅读，补充完整信息。

* * *

## 四、Meme：模因币

**定义**：由网络文化和社区情绪驱动的加密货币，通常无复杂技术背景或落地应用场景，价值主要来源于社区共识和社交传播。

**典型代表**：Dogecoin、Shiba Inu

**核心意义**：体现了 Web3 世界中社区文化与注意力经济的影响力。

Web3 行业已形成四大核心赛道，基础信息汇总如下：

## **二、今日理论要点**

### **1\. 以太坊是什么**

-   开源去中心化平台，支持智能合约，被称为“全球计算机”。
    
-   原生代币ETH，市值第二。
    

### **2\. 以太坊 vs 比特币**

-   比特币：数字黄金，脚本有限，PoW，10分钟/块。
    
-   以太坊：可编程，图灵完备，PoS（The Merge后），12秒/块，支持DeFi/NFT等。

-   **The Merge（2022.9）**：从PoW切到PoS，能耗降99.95%，验证者需质押32 ETH。
    
-   **未来**：EIP-4844（Blob交易）降低L2费用；分片+Rollup扩容。
    

### **4\. 三大核心机制**

-   **账户**：EOA（私钥控制）和CA（代码控制）。
    
-   **Gas**：计算费用（Limit × Price），EIP-1559后Base Fee销毁+Tip。
    
-   **EVM**：全球同步执行的虚拟机，图灵完备，沙盒隔离。
    

### **5\. Layer2**

-   Rollup（Optimistic / ZK）将交易批量提交至主网，大幅降费提速。

## **三、个人理解（链上 vs 普通互联网）**

-   **普通产品**：数据归平台，可改可删，账户靠平台验证。

\--

\## 🧱 区块链是什么？

\- **本质**：去中心化的分布式账本，数据公开、透明、不可篡改。

\- **核心逻辑**：每个区块存着交易记录 + 上一个区块的“指纹”（哈希），环环相扣。改旧账 = 改后面所有账，几乎不可能。

\- **安全机制**：全球无数节点各有一份完整账本，想作弊得控制全网 **51%以上** 的算力，成本极高。

\---

\## ₿ 比特币（BTC）

\- **地位**：区块链的第一个成功应用。

\- **特点**：总量固定（不通胀）、匿名转账、跨境快。

\- **风险**：价格波动大，容易被炒作；实时性差（10分钟打包一次）。

\---

\## ⚙️ 运行流程（一笔交易怎么走的）

1\. 用户发起转账 → 2. 广播给全网节点 → 3. 节点验证合法性 → 4. 矿工打包成新区块（共识机制） → 5. 链接到主链 → 6. 矿工获得奖励（区块奖励 + 手续费）。

\---

\## 🏛️ 三种链的区别

\- **公链**（如比特币）：谁都能进，完全公开，去中心化最高（但慢）。

\- **联盟链**：需邀请加入，仅成员可见，半中心化（效率中等）。

\- **私链**：内部私有，老板说了算，完全中心化（效率最高）。

\---

\## 🌐 Web3 核心扫盲（关键区分）

\- **Web2**：平台掌控你的数据和账号（你是租客）。

\- **Web3**：私钥在手，资产和数据归你（你是房东）。基于区块链。

\- **⚠️ Web 3.0 ≠ Web3**：Web 3.0 是语义网（数据标签化），和区块链不是一码事。

\---

\## ✅ 优势 vs ❌ 挑战

\- **优势**：无需信任中介、抗审查、用户掌握主权、生态开放。

\- **挑战**：交易慢/贵（性能瓶颈）、智能合约有漏洞风险、私钥丢了救不回、法律监管灰色地带。

\---

**一句话总结**：区块链就是“全民参与记账”的不可篡改大账本，比特币是它的杀手级应用，Web3用它来实现“我的数据我做主”。
