# Web3 暑期实习第二天学习总结

今天的课程让我对 Web3 的学习方向有了更清晰的认识。一方面，我了解到 EPF 是 Ethereum Protocol Fellowship，也就是以太坊协议奖学金。它不是普通培训班，而是面向以太坊协议层开发的实战训练场。EPF 更适合已经具备一定编程基础、愿意长期参与以太坊底层建设的人，虽然有导师指导，但核心仍然依靠自学、实践和真实贡献。

通过第一节课，我初步区分了协议层和应用层的不同。应用层更多是开发智能合约和 DApp，常用 Solidity；协议层则更接近以太坊本身的底层代码，例如节点同步、交易打包、客户端运行机制等，通常需要 Go 或 Rust 等语言基础。这让我意识到，Web3 并不只是写合约或做前端应用，它还有更底层、更长期的协议研究和工程贡献方向。

第二节答疑让我对 Web3 应用开发流程有了更直观的理解。主持人通过 AI Coding 示范了一个最小 Demo 的开发闭环，包括生成智能合约、安装依赖、编译合约、部署到测试网、获取合约地址和 ABI，再通过前端连接钱包并调用合约。此前我只是知道“智能合约”这个概念，但今天才明白，合约写完并不等于能用，还需要部署、签名、支付 Gas，并通过区块浏览器验证链上结果。

今天让我印象最深的是钱包安全问题。开发钱包和真实资产钱包必须分开使用，私钥和助记词不能泄露，.env 文件不能上传到 GitHub。作为初学者，我意识到 Web3 学习不能只关注功能实现，也必须从一开始建立安全意识，否则很容易因为操作不当造成资产风险。

总体来看，今天最大的收获是：我对 Web3 的两个方向都有了初步认识。协议层开发门槛更高，需要编程能力、源码阅读和长期贡献；应用层开发则可以从智能合约、钱包交互、测试网部署和前端调用开始逐步入门。虽然目前很多技术细节我还没有完全掌握，但我已经理解了 Web3 学习不是简单听课，而是要通过持续阅读、实践、记录和贡献来积累能力。

# DAY 1｜Web3 入门学习记录

今天是参与 Web3 实习计划的第一天，我主要做了三件事：先阅读入门资料，了解 Web3 的基础概念；再梳理 Week 1 需要完成的任务；最后开始准备自己的 Build Log，方便后面记录学习过程。

今天接触到的内容比较多，包括 Web3、区块链、以太坊、钱包、地址、交易、gas、智能合约等。因为之前对这些内容了解不多，所以一开始会觉得有点乱。后来我尝试把它们理解成一条链上操作流程：先有钱包和地址，才能进入链上世界；发起操作时会产生交易；交易需要消耗 gas；如果交互对象是链上的程序，那它可能就是智能合约。

我今天比较有印象的一点是，Web3 里的“身份”和普通互联网不太一样。以前使用网站或 App，大多数时候是注册账号，然后数据和权限都在平台那里；但在 Web3 里，钱包地址会变成一个很重要的入口，很多操作都和这个地址有关。这让我第一次感觉到，Web3 的学习不能只看概念，还要真正去操作钱包、测试网和区块浏览器。

今天也大概了解了 Week 1 的任务安排。Week 1 不是要求我们马上做出完整项目，而是先把最基础的链上动作跑通，比如准备课程专用钱包、配置 Monad Testnet、领取测试币、发起测试网交易、保存 transaction hash，并用区块浏览器查看记录。对新手来说，这些步骤可能看起来简单，但应该是后面学习合约和链上应用的基础。

我目前还不太熟悉这些工具具体怎么配，也不太清楚每一步背后的原理，比如 gas 为什么会变化、transaction hash 能查到哪些信息、智能合约和普通代码到底有什么区别。这些问题我准备在后面的实践里慢慢记录下来。

接下来我的计划是：

\- 先创建或准备一个课程专用钱包，不使用自己的主力钱包。

\- 配置 Monad Testnet，并截图保存配置结果。

\- 尝试完成第一笔测试网交易。

\- 记录交易 hash 和区块浏览器链接。

\- 建立 Week 1 Build Log，把每天的学习内容、截图、问题和修复过程都写下来。

今天的收获是，我对 Web3 还只是刚刚入门，但已经知道接下来不能只停留在看资料上，而是要通过实际操作把钱包、交易、区块浏览器和智能合约这些概念串起来。
