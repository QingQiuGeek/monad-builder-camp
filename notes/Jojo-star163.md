## 一、DApp 核心架构

DApp（去中心化应用）与传统 Web 应用的核心差异在于**去中心化**，由四类组件构成：

| 组件 | 核心作用 | 关键说明 |
| --- | --- | --- |
| 前端（UI） | 用户交互入口 | 不直接连链，通过钱包 Provider/RPC 节点与链交互；需集成 MetaMask 等钱包 |
| 智能合约 | 业务逻辑核心 | 部署在链上，Solidity 编写，EVM 执行，自动运行且不可篡改 |
| 数据检索器（Indexer） | 链上数据索引 | 监听合约 Event，将链上日志同步到传统数据库（如 PostgreSQL），解决合约无复杂查询能力的问题 |
| 区块链 & 去中心化存储 | 数据与状态存储 | 链存核心状态与交易；IPFS/Arweave 存大文件（图片、文档） |

## 二、DApp 开发全流程

1.  **需求分析与规划**
    
    -   明确功能（转账、查询等）、选择链（以太坊/Polygon/Solana，权衡成本与性能）、设计 UX
        
2.  **智能合约开发**
    
    -   编写 → 单元测试 → 安全审计（防重入、溢出等漏洞）
        
3.  **检索器开发**
    
    -   确定前端所需数据 → 用 TypeScript（Ponder/Subgraph 框架）编写事件处理逻辑 → Docker 部署/使用 SaaS
        
4.  **前端开发**
    
    -   框架选 React/Vue → 集成钱包 → 对接链数据（合约 + 检索器）→ 处理交易签名
        
5.  **部署上线**
    
    -   合约：Hardhat/Foundry 部署到测试网（Sepolia）→ 主网
        
    -   前端：IPFS 或 Vercel 部署

## 五、Solidity 智能合约基础

### 基础语法

-   **版本声明**：`pragma solidity ^0.8.0;`
    
-   **数据类型**：
    
    -   基础：`bool`、`uint`/`int`（无符号/有符号整数）、`address`、`string`
        
    -   复合：数组（`uint[]`）、映射（`mapping(address => uint)`）、结构体（`struct`）、枚举（`enum`）
        
-   **函数修饰符**：
    
    -   可见性：`public`（内外可调）、`external`（仅外部，Gas 更低）、`internal`（合约+子类）、`private`（仅当前合约）
        
    -   状态：`view`（只读）、`pure`（不读不改）、`payable`（可收 ETH）
        

### 安全实践

| 风险 | 防护方案 |
| --- | --- |
| 重入攻击 | 遵循「Checks-Effects-Interactions」模式（先改状态再外部调用）；用 OpenZeppelin 的 ReentrancyGuard |
| 访问控制缺失 | 用 onlyOwner修饰符（OpenZeppelin 的 Ownable）限制敏感函数 |
| 整数溢出 | Solidity 0.8+ 内置溢出检查；老版本用 SafeMath库 |

# 一、Web3合规核心：中国监管基调与法律风险

## （一）国内监管总原则

-   明确禁止：ICO融资、虚拟货币交易所运营、虚拟货币作为支付工具、境外交易所向中国居民提供服务、虚拟货币相关非法集资/传销/洗钱/赌博等活动。
    
-   技术本身（如区块链研发）不违规，但触碰Token发行、融资、交易、挖矿等场景，将面临行政甚至刑事处罚。
    

## （二）高频法律风险梳理

| 风险类型 | 核心红线 |
| --- | --- |
| 代币发行与交易​ | 任何具备融资/流通属性的代币（含“积分”“治理Token”“空投”）均可能构成非法金融活动；技术人员参与模型设计、合约部署也会被认定为“共同行为人”，无法以“仅写代码”免责。 |
| 刑事犯罪​ | 1. 开设赌场罪：链游“充值-随机收益-提现”闭环、赌博网站代理/分成均属此类。2. 非法经营罪：以虚拟货币为媒介从事跨境汇兑、非法资金结算。3. 非法吸收公众存款罪：以“挖矿”“虚拟币投资”为名公开吸资。4. 组织、领导传销活动罪：多级返佣、拉人头计酬（如PlusToken案）。5. 洗钱罪：未落实KYC/AML，协助转移犯罪所得（含电信诈骗、赌博等资金）。 |
| 民商事争议​ | 虚拟货币交易合同大概率被认定无效，风险自担；委托代投亏损可能引发民事赔偿，甚至涉嫌非法经营/非法集资。 |
| 场外交易（OTC）​ | 接收涉诈/涉赌资金会被冻卡，甚至构成“帮信罪”“掩饰、隐瞒犯罪所得罪”；高价出U、协助他人洗钱均属违法。 |

### （三）全球监管趋势

-   **趋严化**：欧盟MiCA法规生效（禁止算法稳定币、要求稳定币100%储备）、美国SEC将多数代币视为证券监管、香港要求VASP持牌运营。
    
-   **核心共识**：落实KYC、反洗钱、稳定币储备审计。
    
-   **影响**：合规成本上升，中小项目生存空间压缩，行业向机构化、合规化发展。
    

### （四）入职避坑指南

1.  **雇佣关系**：多数Web3项目境外注册，境内无用工主体，可能无法缴纳社保公积金（影响落户/购房/子女教育）；优先选择EOR（名义雇主）模式，明确劳动关系认定条款。
    
2.  **薪酬风险**：Token/USDT支付不受《劳动法》保护，代币暴跌可能归零；建议协商保留人民币薪酬比例，避免全虚拟货币结算。
    
3.  **出金安全**：拒绝灰色渠道，留存资金来源证明，避免接收不明资金。
    
4.  **项目审查**：入职前核查白皮书，确认无中国大陆用户定向服务、无多级返利、无保本收益承诺。
    

## 二、网络安全

### （一）常见攻击手段与后果

| 攻击类型 | 典型场景 | 后果 |
| --- | --- | --- |
| 钓鱼攻击​ | 伪造面试通知、奖学金表单、官方客服，诱导点击恶意链接/连接钱包签名。 | 助记词/私钥泄露，资产被盗；账号被控制，二次受害。 |
| 恶意软件/木马​ | 伪装成“面试软件”“学习资料”，植入剪贴板劫持木马（自动替换转账地址）、远程控制后门。 | 钱包文件、账号密码被盗；电脑被监控/勒索。 |
| 社交工程​ | 冒充HR/学长/好友，以“内推”“紧急求助”为名诱导转账或泄露信息。 | 资产损失，个人信息泄露。 |
| 供应链攻击​ | 恶意浏览器插件、开源库后门，批量篡改转账地址。 | 大量用户资产被盗，溯源困难。 |
| 传统账号风险​ | 弱密码/密码复用、未开启2FA、邮箱/SIM卡劫持。 | 全账号被盗，隐私与资产双重损失。 |

## （二）防护清单

1.  **面试/实习**：只用Zoom/Teams/腾讯会议等官方工具，拒绝安装“专用面试软件”；核实邀请人邮箱/域名，安装前查SHA256摘要与数字签名。
    
2.  **空投/福利**：任何要求“连接钱包”“输入助记词”的网页99%是骗局；主钱包冷存储，空投用独立测试钱包。
    
3.  **社交平台**：不轻信群内“官方人员”私聊，涉及转账/链接需多渠道核实。
    
4.  **软件安装**：仅从官网/官方应用商店下载，浏览器插件控制在3个以内（官方钱包+密码管理器+广告屏蔽器）。
    
5.  **账号安全**：启用2FA（优先谷歌验证/硬件密钥，禁用短信验证）；用密码管理器生成独立强密码，定期更换邮箱/手机号密码。
    
6.  **钱包操作**：转账前核对完整地址（优先用地址簿/ENS/白名单，不信赖局部字符）；助记词离线保存（不截图/不上传网盘）；取消不必要钱包授权。
    
7.  **应急：遇攻击立即断网，截屏记录时间点，联系专业社群/校内信息中心求助。**

# 一、 行业赛道全览

Web3 已经从一个单一的支付系统，演变成一个涵盖金融、艺术、组织和文化的大生态。主要有这四个热门赛道：

## 1\. DeFi：去中心化金融

-   **它是什么？** 一个基于区块链，不需要银行等中介的金融体系。你可以自由地进行借贷、交易，无需审批。
    
-   **核心案例：**
    
    -   **Uniswap (去中心化交易所/DEX)**：通过“自动做市商”机制，让任何人都能为交易池提供流动性来赚取手续费。
        
    -   **Compound (借贷协议)**：你可以抵押自己的加密货币来借出其他资产，或者存钱进去赚利息。整个过程由智能合约自动执行，非常透明。
        
    -   **MakerDAO / Sky (稳定币系统)**：通过超额抵押生成与美元挂钩的稳定币（如 DAI / USDS），为用户提供了一个相对稳定的数字资产。
        

## 2\. NFT：数字所有权革命

-   **它是什么？** 非同质化代币，让数字作品（如图片、视频）拥有唯一的、不可篡改的所有权证明。
    
-   **核心技术**：利用**智能合约**自动化交易，比如当你的 NFT 被转卖时，合约会自动给原作者支付版税，无需中间人。
    
-   **典型案例**：CryptoPunks（最早的像素头像NFT）、OpenSea（最大的 NFT 交易市场）。
    

## 3\. DAO：去中心化自治组织

-   **它是什么？** 一个没有老板，由社区成员通过持有代币来投票决策的组织。所有决策公开透明。
    
-   **典型案例**：
    
    -   **Nouns DAO**：每天拍卖一个 NFT头像，收益归组织金库，持有者可以投票决定金库资金的用途。
        
    -   **ConstitutionDAO**：一群人在2021年众筹4700万美元，试图竞拍一份美国宪法原件。虽然没成功，但展示了DAO巨大的集体行动潜力。
        

## 4\. MEME：文化与投资热潮

-   **它是什么？** 基于网络流行梗、表情包产生的加密货币（如狗狗币DOGE、青蛙币PEPE），主要靠社区共识和文化认同获得价值，而非技术。
    
-   **风险提示**：MEME币没有实际应用支撑，价格波动极大，高风险极高。应**警惕炒作、拒绝盲目跟风、关注链上数据、严格控制仓位**。
    

## 5\. 新兴趋势与交叉创新

-   **Intent-Based 交易**：用户只需表达“想用 100 USDC 买最多的ETH”，系统会自动寻找最优交易路径，简化了操作。
    
-   **模块化区块链**：将区块链的“执行、共识、数据存储”等功能拆分，让架构更灵活、扩展性更好。
    
-   **AI + Web3**：AI 驱动的自动化交易、去中心化AI训练网络、AI生成的NFT等。
    
-   **Web3 + 乡建 (南塘DAO)**：一个将区块链技术落地到中国乡村的实践案例，发行社区货币“南塘豆”，探索利用区块链技术振兴乡村的新模式。

# 二、 区块链基础概念

## **1\. 区块链是什么？**

### **（1）定义**

-   一种去中心化的**分布式账本技术**，用于在网络节点之间安全、透明且不可篡改地记录事务数据。
    

### **（2）核心结构**

-   由一系列按时间顺序相连的 **“区块”** 组成。每个区块包含交易记录和上一个区块的哈希值。
    
-   **不可篡改**：修改一个区块的历史信息，必须同时修改之后的所有区块，成本极高。
    
-   **公开透明、匿名**：链上所有信息公开可查，但地址与现实身份不直接关联。
    

### **（3）关键机制 — 分布式网络**

-   网络中众多节点（矿机）共同维护和记录同一份完整的区块链数据。
    
-   **去中心化**：没有人能控制全部节点，数据不会丢失，也无法被轻易篡改。
    

## **2\. 为什么要使用区块链？**

### **（1）节点激励**

-   提供网络服务的节点（矿工/验证者）会因为维护网络安全和处理交易而获得**代币奖励**（如比特币）。
    

### **（2）货币属性**

-   有限的供应量和自由的转账功能，使代币具备了货币属性，成为加密货币。
    

## **3\. 区块链的核心组成部分**

### **（1）去中心化网络 + 区块链**

-   区块链记录所有信息，去中心化网络维持其运行。
    

### **（2）代币激励**

-   激励节点提供服务。
    

### **（3）运行流程**：

-   用户发起交易
    
-   交易广播至全网
    
-   节点验证交易合法性
    
-   通过共识机制将交易打包成新块
    
-   新块链接到主链，更新账本
    
-   成功出块的节点获得奖励
    

## **4\. 公链 vs 私链 vs 联盟链**

| 类型 | 节点加入方式 | 数据可见性 | 管理模式 | 适合场景 | 去中心化程度 |
| --- | --- | --- | --- | --- | --- |
| 公链 | 任何人自由加入 | 所有人可见 | 去中心化 | 加密货币、公共存证 | 高 |
| 联盟链 | 需成员邀请/审批 | 仅联盟成员可见 | 多中心化 | 供应链、金融协作 | 中 |
| 私链 | 由老板严格审批 | 仅内部成员可见 | 中心化 | 企业内部管理 | 低 |

## **5\. Web3 vs Web 3.0 vs Web2**

| 维度 | Web2 | Web 3.0 (语义网) | Web3 |
| --- | --- | --- | --- |
| 核心特征 | 中心化控制、用户不拥有数据 | 数据语义化、结构化、机器可读 | 数据主权归用户、无需信任中介 |
| 数据存储 | 中心服务器 | 混合存储 | 区块链 / IPFS |
| 典型技术 | JavaScript, React, Node.js, MySQL | RDF, OWL, SPARQL | 智能合约 (Solidity), IPFS |
| 典型应用 | 微信、抖音、亚马逊 | 知识图谱、语义搜索 | MetaMask, Uniswap, ENS |

# 三、 以太坊概览

## **1\. 以太坊定位**

-   被誉为“区块链 2.0” 或 **“全球共享计算机”** 。
    
-   核心创新是**智能合约**：存储在区块链上的可执行代码，在满足条件时自动执行，无需第三方。
    
-   其原生代币**以太币 (ETH)** 是市值排名第二的加密货币，用于支付网络操作费用（Gas费）。
    

## **2\. 以太坊 vs 比特币**

| 维度 | 比特币 (Bitcoin) | 以太坊 (Ethereum) |
| --- | --- | --- |
| 目标与定位 | 去中心化数字货币（数字黄金） | 去中心化平台，支持智能合约和 Dapps |
| 编程能力 | 有限，仅支持简单交易验证 | 图灵完备，可开发复杂智能合约 |
| 共识机制 | 工作量证明 (PoW) | 权益证明 (PoS) |
| 交易速度 | 约 10 分钟一个区块 | 约 12 秒一个区块 |

## **3\. 以太坊 2.0 与 The Merge (合并)**

### **（1）新架构**

-   分为**执行层**（处理交易和智能合约）和**共识层**（管理验证者）。
    

### **（2）PoS 机制**：

-   **验证者**：需质押 32 ETH，负责提议和验证区块。
    
-   **奖励**：获得新发行的 ETH 和交易费用。
    
-   **惩罚**：作恶者质押的 ETH 会被销毁 (Slashing)。
    
-   **优势**：能耗降低 99.95%，经济安全性更高。
    

## **4\. 以太坊生态分层**

| 层级 | 功能 | 组件举例 |
| --- | --- | --- |
| 应用层 | 用户直接交互的应用 | Uniswap (DeFi), OpenSea (NFT), MetaMask (钱包) |
| 协议层 | 以太坊核心基础设施 | Geth, Lighthouse (客户端), EVM |
| 扩展层 | 提升性能、降低成本 | Arbitrum, Optimism (L2 Rollups), Polygon PoS (侧链) |

## **5\. 核心机制**

### **（1）账户系统**：

-   **外部账户 (EOA)**：由用户私钥控制，可主动发起交易。
    
-   **合约账户 (CA)**：由智能合约代码驱动，不能主动发起交易。
    
-   每个账户包含 Nonce、余额、CodeHash、StorageRoot。
    

### **（2）Gas 模型**：

-   **Gas Fee = Gas Limit x Gas Price**。
    
-   激励矿工/验证者，防止资源滥用。
    
-   EIP-1559 后，费用分为 **基础费用**（销毁）和 **小费**（给验证者）。
    

### **（3）以太坊虚拟机 (EVM)**

-   图灵完备的分布式计算机，全局同步且隔离安全地执行智能合约代码。
    

# **6.核心文化与价值观**

-   **去中心化治理**：社区通过 EIP 机制共同决策。
    
-   **无需许可与开放性**：任何人都可使用、开发并部署。
    
-   **抗审查性**：交易不受机构干预。
    
-   **密码朋克精神**：代码即法律，用数学构建信任。
    
-   **公共物品导向**：优先考虑生态系统整体利益。
    
-   **可持续发展**：从 PoW 转向 PoS 体现环保承诺。
