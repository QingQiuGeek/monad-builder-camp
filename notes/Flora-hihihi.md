# Web3 安全与合规

## 一、中国 Web3 监管基本态度

### 1\. 总体监管基调

**全面封堵金融属性，有限容忍技术创新。**

中国目前重点禁止：

-   ICO、IEO、IDO 等代币融资
    
-   虚拟货币交易所及撮合服务
    
-   将虚拟货币作为支付工具
    
-   境外交易平台向中国境内居民提供服务
    
-   利用虚拟货币进行非法集资、传销、洗钱、赌博等活动
    

**核心判断：**  
Web3 技术本身不违法，但一旦涉及 **Token 发行、融资、交易、矿池运营** 等金融活动，法律风险显著上升。

* * *

# 二、Web3 常见法律风险

## 1\. Token 发行与交易风险

不论名称是：

-   Token
    
-   积分
    
-   凭证
    
-   治理币
    

只要具有：

**融资功能 + 可流通性**

就可能被认定为非法金融活动。

常见高风险行为：

-   ICO / IEO / IDO
    
-   面向公众募资
    
-   以“空投”“白名单”名义变相融资
    

### 技术人员也可能承担责任

参与以下工作也可能被认定为共同参与者：

-   Token 模型设计
    
-   空投机制设计
    
-   智能合约部署
    
-   技术开发与维护
    

**不能简单以“我只是写代码”免责。**

* * *

## 2\. 赌博风险

高风险模式：

**充值 → 抽奖 / 随机结果 → 获得 Token 或 NFT → 提现**

例如：

-   盲盒
    
-   转盘
    
-   随机 NFT
    
-   链游抽奖
    

判断核心：

**下注 + 随机输赢 + 可兑现**

如果项目具有提现功能，需要格外警惕。

* * *

## 3\. 传销风险

常见高风险机制：

-   邀请返利
    
-   多级推广
    
-   团队计酬
    
-   多层级返佣
    
-   算力挂靠
    

可能涉及：

**组织、领导传销活动罪。**

* * *

## 4\. 洗钱与非法经营风险

### 地下换汇模式

常见路径：

**人民币 → USDT / ETH → 境外兑换美元等外币**

如果长期、反复组织：

-   人民币与虚拟货币兑换
    
-   虚拟货币与外币兑换
    
-   跨境资金转移
    

可能涉及：

**非法经营外汇业务。**

### 黑钱风险

交易资金可能来自：

-   电信诈骗
    
-   网络赌博
    
-   毒品交易
    
-   其他犯罪活动
    

可能导致：

-   银行卡被冻结
    
-   被要求退赔资金
    
-   涉嫌洗钱罪
    
-   掩饰、隐瞒犯罪所得罪
    
-   帮信罪
    

**核心原则：KYT / KYC，审查交易对手和资金来源。**

* * *

## 5\. 民商事风险

### 虚拟货币交易

在中国司法实践中，虚拟货币相关交易可能被认定：

-   合同无效
    
-   风险自担
    
-   损失难以获得法院支持
    

### 帮别人“代投”

主要风险：

-   投资亏损引发赔偿
    
-   资金去向不清
    
-   被认定为非法经营
    
-   面向公众募资可能涉及非法集资
    

### 风险控制

委托方：

-   明确投资范围
    
-   签订协议
    
-   保留操作凭证
    
-   监督资金流向
    

受托方：

-   不得超越授权范围
    
-   不随意接受陌生人的资金
    
-   不公开募集公众资金

# 四、FATF

## 1\. FATF 是什么？

**Financial Action Task Force**  
中文：**金融行动特别工作组**

主要负责制定全球：

-   AML：反洗钱标准
    
-   CFT：反恐怖融资标准
    

主要职能：

1.  制定国际标准
    
2.  评估成员国
    
3.  发布灰名单、黑名单
    
4.  协调全球金融犯罪治理
    

* * *

## 2\. Travel Rule 旅行规则

核心要求：

虚拟资产服务提供商 VASP 在转账时需要收集并传输：

-   发送方信息
    
-   接收方信息
    

对 Web3 的影响：

-   增加交易透明度
    
-   加强 AML
    
-   对 DeFi、DEX 的匿名模式形成挑战
    

* * *

# 五、为什么要监管稳定币？

## 1\. 系统性风险

稳定币是 DeFi 的重要基础设施。

如果主要稳定币发生：

-   脱锚
    
-   挤兑
    
-   储备不足
    

可能引发系统性风险。

典型案例：

**2022 年 Terra UST 崩盘。**

* * *

## 2\. 对货币体系的影响

稳定币可能：

-   影响货币供应
    
-   冲击传统银行体系
    
-   与 CBDC 形成竞争
    
-   绕过传统跨境支付体系
    

* * *

## 3\. 稳定币监管目标

主要包括：

-   **保护投资者**：保证储备资产充足
    
-   **维护金融稳定**：防止风险传染
    
-   **反洗钱**：防止稳定币用于非法活动

# 七、Web3 求职与入职风险

## 1\. 雇佣关系风险

很多 Web3 公司注册在：

-   新加坡
    
-   开曼群岛
    
-   BVI
    

特点：

-   远程办公

-   工伤保障不足

社保、公积金可能影响：

-   落户

-   子女教育
    
-   婚育福利

## 2\. EOR 模式

**EOR = Employer of Record，名义雇主。**

模式：

境外 Web3 公司  
→ 委托国内合规公司  
→ 国内公司与员工签劳动合同

可以部分解决：

-   劳动合同
    
-   社保
    
-   公积金
    

但发生纠纷时，仍需关注：

**真正的劳动关系主体是谁。**

# 八、Web3 薪酬风险

常见模式：

-   人民币 + Token
    
-   人民币 + USDT
    
-   全 USDT
    
-   公司自发 Token
    

### 风险 1：工资支付问题

中国劳动法原则上要求：

**工资使用人民币支付。**

Token 或 USDT 作为工资可能存在法律风险。

### 风险 2：影响工资基数

如果 Token 不计入正式工资：

可能影响：

-   社保缴费基数
    
-   经济补偿金
    
-   劳动权益
    

### 风险 3：Token 价格波动

公司自发 Token 可能：

-   大幅贬值
    
-   无法交易
    
-   项目失败后归零
    

* * *

# 九、虚拟货币“出金”风险

常见方式：

-   C2C
    
-   OTC
    
-   场外交易群
    
-   平台挂单
    

最大风险：

**收到涉诈、涉赌等犯罪资金。**

后果可能包括：

-   银行卡冻结
    
-   账户调查
    
-   退赔涉案资金
    
-   刑事风险
    

### 安全原则

-   尽量保留部分法币工资
    
-   使用可信渠道
    
-   不参与高价收 U / 出 U
    
-   不帮助陌生人转账
    
-   保存工资、交易、资金来源证明

# 十一、刑事管辖：公司在海外 ≠ 没有风险

Web3 项目常见模式：

**境外做敏感业务 + 境内做技术研发**

但中国刑法不仅有：

-   属地管辖
    

还可能有：

-   属人管辖
    

因此，即使业务发生在境外，中国境内员工仍可能因：

-   共同犯罪
    
-   帮助犯罪
    

承担责任。

**“公司注册在海外”并不是绝对免责理由。**

* * *

# 十二、重点刑事风险

## 1\. 开设赌场罪

高风险特征：

**投注 + 随机输赢 + 提现**

网络赌博常见行为：

-   建立赌博网站
    
-   提供赌博平台
    
-   担任代理
    
-   接受投注
    
-   参与利润分成
    

### BigGame 案例

项目：

EOS 链上赌博平台 BigGame

游戏包括：

-   掷骰子
    
-   红黑游戏
    

参与人员包括：

-   运营
    
-   技术开发
    
-   后端开发
    
-   产品设计
    
-   智能合约开发
    
-   客户端开发
    
-   客服
    
-   推广
    

最终：

**不仅老板被判刑，技术、产品、客服、推广人员也可能承担刑事责任。**

### 核心启示

**参与违法 Web3 项目时，“只是技术人员”不能当然免责。**

* * *

## 2\. 非法经营罪

重点关注：

-   非法支付结算
    
-   非法经营外汇
    
-   利用虚拟货币进行人民币与外币兑换
    

高风险模式：

**人民币 → 虚拟货币 → 外币**

对于：

-   跨境支付
    
-   OTC
    
-   稳定币支付
    
-   虚拟币换汇
    

从业者需要特别谨慎。

* * *

# 十三、一句话风险判断框架

参与 Web3 项目前，可以先问 6 个问题：

### 1\. 有没有发币？

→ Token 发行风险

### 2\. 有没有向公众收钱？

→ 非法集资风险

### 3\. 有没有随机输赢和提现？

→ 赌博风险

### 4\. 有没有拉人头、多级返佣？

→ 传销风险

### 5\. 有没有帮人换汇、出入金？

→ 非法经营 / 洗钱风险

### 6\. 资金来源是否清楚？

→ 涉诈 / 涉赌 / 洗钱风险

* * *

# 最核心的结论

> **Web3 最大的合规误区，是认为“技术中立”“公司在海外”“只负责写代码”就没有法律责任。**

真正需要关注的是：

**项目实际做什么、资金如何流动、Token 如何使用、用户来自哪里、自己具体参与了什么。**

# 普通人如何进入以太坊协议链

## 学习路径（建地图 → 懂流程 → 找入口）

-   **第一步：建立整体认知地图**：理解以太坊的执行层、共识层、节点网络等基础架构
    
-   **第二步：阅读官方资料**：从以太坊 Wiki 和官方 specs 开始，根据兴趣选择执行层或共识层规范
    
-   **第三步：进入客户端**：推荐从 Go 客户端入手，因其使用最广泛、最成熟、相对易于理解
    

## EPF 项目介绍

-   EPF（以太坊协议奖学金）面向对以太坊**内部机制**（协议改进、扩容、ZK 证明、Gas 费降低等）感兴趣的人，而非 Solidity/DeFi 等应用层开发
    
-   主要使用语言为 **Go 和 Rust**
    
-   分为 **Provisioned** 和 **Provisionless** 两类；最终看项目成果，优秀作品即使无许可也可获得丰厚奖励
    
-   面试为**英文**，需要具备较好的英文听力
    
-   面试不是纯技术性面试（不考手写算法），更看重对以太坊协议的理解和研究能力
    

## EIP

| 类型 | 作用 | 例子 |
| --- | --- | --- |
| Core | 改以太坊底层协议、共识、EVM 等 | EIP-1559（以太坊 Gas 费机制）、EIP-4844 |
| Networking | 改 P2P 网络层 | 节点通信相关 |
| Interface | 改客户端/API/接口规范 | 钱包、RPC 接口等 |
| ERC | 应用层标准，尤其是智能合约标准 | ERC-20、ERC-721 |
