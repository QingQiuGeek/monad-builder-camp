Day4智能合约

过程：**合约源码 → 编译 → 部署 → 合约地址 → read / write 调用 → 区块浏览器验证**

**合约源码 Source Code**

AI 生成、在 Remix 里编辑的`.sol`完整 Solidity 代码（Gmonad 问候合约）

重点不是让 AI 替你完成开发，而是练习如何让 AI 辅助你理解代码、生成初稿，并由你完成最终判断。

人工检查点需要包括：

-   合约是否能编译。
    
-   函数是否符合你的预期。
    
-   是否存在明显权限问题。
    
-   是否使用了不必要的复杂逻辑。
    
-   是否有潜在安全风险。
    
-   变量和函数命名是否容易理解。
    

### ABI 应用二进制接口

源码在 Remix 编译后自动生成的「交互说明书」。

-   内容：记录合约所有函数名、参数类型、返回值、读写属性；
    
-   核心作用：钱包 / Rabby、Remix、区块浏览器**靠 ABI 识别合约功能**；
    
    没有 ABI，工具分不清哪个是读、哪个是写函数，无法交互；
    
-   来源：源码编译的产物，源码改动后必须重新编译才会更新 ABI。
    

编译后 Remix 读懂 “有一个读问候、一个改问候的函数”；

### 合约地址 Contract Address

部署合约完成后在deployed contracts下的 **0x——** 的唯一地址

-   产生时机：部署交易上链成功后，Monad 链自动分配给合约的专属 “账号”；
    
-   作用：在区块链上定位这个合约，所有读写操作都要指定目标合约地址；
    
-   特点：同一套源码部署两次，会得到两个完全不同的地址。
    

### **Read / Write Function 读写函数**

Read 只读`greeting()；`Read 只读`greeting()`

### Transaction Hash（Tx Hash 交易哈希）

一串 0x 开头唯一标识符，**只有执行写操作 / 部署合约时才会生成**。

-   产生场景：部署合约、调用`setGreeting`修改问候语、转账；
    
-   作用：链上操作唯一凭证，可在 Monadscan 浏览器查询完整操作记录；
    
-   只读操作`greeting()`不会产生任何 Tx Hash。
    

粘贴到 Monadscan，可查到：合约地址、调用函数、修改前后数据、Gas 消耗。

Day1 初步了解monad以及潮流的AI-coding

首先，我作为一个在此学习计划前基本0接触AI、编程、monad的新手而言，目前对我最重要的是了解monad大体框架，同时依靠简单易上手的AI- coding，对所学有初步的感知。

今天我跟随课程以及开营仪式的脚步，学习到了为什么我们需要monad，主要有以下几点

-   Monad 有着更高的吞吐和更低成本，既具备visa速度，又能跑完整的，规模化的应用。
    
-   如果没有monad，Ethereum吞吐量极高，力不从心；未来的算力需求必然是持续走高的，单靠ethereum跟不上，就需要monad提供基础环境
    
-   Monad是高性能的，完全兼容EVM的L1，同时不放弃Ethereum生态的保障
    
-   数据快 例如Gmonads，实时验证者性能；任何人可以运行节点，不改动EVM的前提下释放吞吐量
    
-   monadBFT RaptorCast JIT Compilation是多区块间，在共识状态下 一起执行的
    

以及AI-coding的了解感知，我明白了它是一种不需要操作者会写代码，只要操作者有一个想要做、实现的念头，把它具体地告诉AI，AI就会自动帮助操作者完整代码，后续过程中再进行不断的迭代以产出一个自己满意的产品。

我也了解到，初步的AI- coding完成不了交付、数据量大的工作，就不得不触及到monad。

今天接触到的一切对于我来说并不简单，只能从框架、基础概念的理解开始，一步一步入门monad。
