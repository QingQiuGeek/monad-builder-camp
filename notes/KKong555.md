## Building in the AI Era+Co learning  
一、从“技术概念”回归“商业常识”

无论是 AI 还是 Web3 领域，当前的行业共识都在从“炒作概念”转向“务实落地”。

### 1\. AI 的下半场：业务 > 流程 > 规则 > AI

-   阶段转变： AI 的上半场（模型层面的竞争）已基本大局已定。过去大家在质疑 AI “能做什么”，现在下半场的核心问题变成了 “要用 AI 做什么”、“给它什么样的限制” 以及 “如何应用在具体业务场景中”。
    
-   业务是核心，AI 只是壳： “AI 是赋能，不是前提”。必须先梳理并跑通整个业务流程、解决真实的业务痛点（即“有了前面的 1”），AI（“后面的 0”）才能发挥效率放大的乘数作用。
    
-   务实主义： “吹 100 个概念，不如跑通一个真实的业务”。除极少数具备顶尖背景且不需要面向用户的实验室团队外，绝大多数创业者应该沉进真实业务。
    

### 2\. 区块链（Web3）的产品方向思考

在 Web3 领域立项同样需要“带资立项”的务实考量，老师将目前链上高价值/高交易量的产品归纳为三个切入方向：

-   无痕法币价值流通： 解决跨地区、跨币种的资产无痕转化（如百万级美元与欧元的无痕兑换）。
    
-   高性能基建改进： 提升链上交易速度，解决高并发、低延迟的交易需求，以支撑频繁的数据波动。
    
-   基于智能合约的公平投机产品： 预测类、拆号码、体育竞猜等投机驱动、交易量巨大的行业。
    

## 二、 商业范式与调研思维的转变

### 1\. 走向 Agent 经济的未来范式

随着业务与流程的打通，Agent将重塑经济关系，从以前Human👉Software👉Service到现在未来Human👉Agent👉Agent👉Protocol👉Payment👉Result的新范式：

-   传统的互联网是人面向软件、软件面向服务的线性交互。
    
-   未来的商业交叉点在于，人类只需对自己的 Agent 下达意图，通过 Agent 之间相互对话、打通中间操作层，直接连接买卖双方。
    
-   _当前局限：_ 现有的链上协议（如 ERC-8804等）解决的问题较为单一，在面对真实业务的边缘情况时仍需重塑。
    

### 2\. 市场数据调研的去中心化思维

-   纠正中心化思维： 传统互联网人转型 Web3 时，容易尝试用 AI 工具（如 Gemini、GPT）去整合去中心化的代币流通数据，这在缺乏中心化统计的 Web3 行业往往流于表面。
    
-   调整调研维度： 应当转换思维，从“**市场讨论热度**”切入，通过调研不同国家（如日韩）本土论坛的话题讨论度，来曲线评估市场和产品方向。
    

## 三、 流量路径与营销推广：国内传统私域 vs 海外 Web3 社群

无论国内传统行业还是海外 Web3，其流量底层逻辑高度一致，即：**公域引流👉协同池👉私域转化**。但在合规底线和工具应用上存在显著差异：

### 1\. 法律与内容合规层面的底线

-   国内： 流程繁琐，内容首先受到《广告法》和《市场监督管理办法》的严格限制，电销和私域审查极严。
    
-   国外 / Web3： 纯技术项的内容讨论基本不受限。但若涉及交易属性，必须面对极其复杂的国际认证与牌照（类似于赌牌，如美、英、法联合认证或欧盟 MICA 等），投放时需严格防范触发地区限制的风险。
    

### 2\. 流量路径对比表

| 阶段 | 国内传统行业（以美妆/快消为例） | 国外 / Web3 行业 |
| 公域平台 | 抖音、小红书、视频号、快手、微博 | X、Facebook、海外各大论坛 |
| KOL特征 | 垂直领域达人（如美妆博主、剧情号） | 推特大V、知名行业KOL、Alpha社区 |
| 协同漏斗 | 达人内容植入👉引导进平台群👉团队私聊打粉 | KOL发文引流👉引导进入特定的TG或 Discord 协同池 |
| 私域沉淀 | 加微信、企业微信、QQ👉 归流至品牌自营方进行复购营销 | 沉淀在自己的 TG群等私域阵地，形成长效资产 |

## 四、 流量投放防骗与行业“避坑”指南

分享人根据自身“用真金白银烧出来的广告费经验”，分享了国内外流量投放的残酷现状与防骗建议：

-   数据造假的识别难点（国内）： 国内机房技术发达，能以 10 万为单位提供维持一整季度的“真人级留言僵尸号”。传统的看数据、看内容流向已很难识别造假。
    
    -   _实操对策：_ 采用“试量法”。让渠道在 2-3 天内先供 5000 粉，跑一周左右观测首单成交后“第 7 天的复购率”，以此倒推流量的真实与精准度。
        
-   最恶心的坑——掺假放大（洗粉）： 发言人曾遭遇渠道一天推送 600 个客户（单价 40 元/个），其中暗中掺杂了 30% 的机器人（180 个）。连续投放会导致极其惨烈的广告费损失，防范此类“洗粉/掺水”是推广的核心难点。
    
-   Web3 KOL 的匿名“无赖”问题： 鉴于 Web3 的去中心化与匿名特征，海外某些 KOL 收钱不干活，或在 TG 等渠道进行微信解封诈骗。由于他们可以随时换上成百上千个马甲重新出现，企业基本无法追责。
    
    -   _避坑建议：_ 初入行或在非专业领域时，必须走公认的靠谱渠道，或选择熟人已经验证过的白名单渠道，切忌盲信链上或 TG 上的陌生推介。
        

## 我的思考

### 1\. 技术祛魅：从“科技宏大叙事”回归“古典商业常识”

整场会议最大的价值，在于“降维”，把高大上的技术名词，拆解回最朴素的商业常识：

-   AI 下半场的本质是“算账”： 别再跟客户聊大模型参数和涌现能力了，客户要的是“业务 > 流程 > 规则 > AI”。如果前三者不通，AI 就是个空壳。
    
-   Web3 创新的本质是“刚需”： 剥离掉代币的泡沫，链上真正能活下来、能带资立项的产品，依然在解决人类最古老的欲望与痛点：资产怎么安全跨国转移（跨国贸易）、速度怎么更快（基建）、怎么更公平地以小博大（博弈投机）。
    
-   核心思考： 无论是做 AI 还是 Web3，“造概念的红利期”已经彻底结束了。现在拼的是谁能把脚踩在泥地里，去帮客户解决那些最脏、最累、最具体的业务痛点。
    

### 2\. 流量的宇宙终点：公私域联动与“对抗人性”的博弈

技术在变，但人性的弱点和流量的漏斗模型从来没变过。

-   同样的剧本，不同的舞台： 在国内，你是在抖音看美妆博主，然后被洗进微信群买割韭菜的护肤品；在web3，你是在推特看加密大V分析行情，然后被洗进 Telegram/Discord 群买空气币。
    
-   去中心化带来的“恶”： 传统的 Web2 营销再脏，好歹有微信、巨量引擎等中心化平台依靠实名制、主体认证进行强监管（如《广告法》）。但 Web3 走向了去中心化的极端，导致了极高的信任成本，比如KOL 可以随时换 1000 个马甲诈骗，掺水 30% 的僵尸粉让你防不胜防。
    
-   核心思考： 去中心化带来了自由，但也释放了“无成本作恶”的匿名怪兽。在 Web3 做营销，对抗的不仅是同行，还有失控的信任崩塌。熟人网络和强背书渠道，在去中心化世界里反而成了最稀缺的硬通货。
    

### 3\. 让人敬畏的“AI 原生代”与“非标未来”

最后提到的一个青少年 AI 教育营，我们这代人，带着满身的 Web2 肌肉记忆（纠结合规、纠结 Prompt 怎么写、纠结买量掺水、纠结 AI 只是个壳），在艰难地向 AI 下半场和 Web3 转型。而那群青少年，他们没有任何包袱。他们直接把 AI 当成手脚，把创意当成燃料，迅速在非标领域迭代产品。真正能把 AI 和去中心化玩出花来的，绝不是我们这群天天想着怎么用旧逻辑“降本增效”的成年人，而是那群一生下来就跟 Agent 对话、把技术当成空气和水的“AI 原生代”。

日常用钱包的安全常识。

千万别相信任何私聊你的官方客服，也别觉得“只是点个同意”没啥大不了。比如，助记词一旦截图发错群，千万别侥幸以为“撤回了就没事”，必须立刻建新钱包转走资产，因为撤回根本不能让助记词失效。另外，网站上那些零 Gas 费的免费签名，或者让你点同意的无限授权，背后都可能给黑客留下了随时掏空你钱包的权限。记住一句话，光在网站上点“断开连接”是不够的，必须去链上撤销（Revoke）授权，才算真正把“钱袋子的口子”给扎紧了。 

给技术开发人员的避坑指南，强调“前端做的再好看都没用，链上逻辑才是唯一的底线”。里面提到了好几个经典的黑客攻击套路，比如不能把敏感功能只在前端页面隐藏，必须在合约里写死权限，不然黑客绕过前端照样能随意增发代币；再比如写退款功能时，必须“先记账、后打钱”，不然黑客一招“重入攻击”就能在系统反应过来之前把合约里的钱薅得一干二净。总之，业务规则、身份校验和调用结果这些东西，只要没有白纸黑字写进链上的智能合约，在黑客眼里就形同虚设。

1\. Web3 Researcher 的成长之路

AI 时代的研究方法论：拒绝让 AI 简单地“代写”，而是要建立可靠的研究方法论，学会识别“AI 幻觉”，开展真正有深度和价值的 Web3 独立研究。

研究的公共价值与沉淀：探讨了如何通过 Public Goods（公共物品）和 Peer-to-Peer Commons（点对点公地）等实践，将个人研究转化为“公共知识”。

去中心化科学（DeSci）与平台：分享了 DeSci 自学手册的设计思路、参与路径，以及如何将研究成果贡献到Grokipedia等开放知识平台。

资金与长期支持：解答了 Web3 研究的 Funding（资助）来源，探讨研究员如何获得长期、可持续的资金资助。

2\. Web3 开源协作：从GitHub开始的实战

简历新范式：在 Web3 行业，一页含金量高的 GitHub Profile（个人主页）往往比传统简历更有说服力。

开源项目 Moss 实战案例：以面向 AI Agent 与 Web3 场景的开源框架 Moss 为例，展示了一个真实的 Web3 开源项目是如何协作、迭代和成长的。

零基础参与开源路径：拆解了最基础的 GitHub 协作流程（阅读项目、寻找 Good First Issue 、提交 Issue、创建 Pull Request），帮助新人迈出参与全球 Web3 开源生态的第一步。

深度思考与延伸复盘

知识的“公共性”与“去中心化科学 (DeSci)”是研究员的新出路

传统的学术研究往往面临壁垒高、资金难、成果私有化的问题。分享中提到的 DeSci 正在打破这种垄断。作为 Web3 Researcher，未来的价值不仅在于写出几篇报告，更在于把研究转化为“公共物品”，通过开放平台建立个人影响力，从而以去中心化的方式（如 Gitcoin、DAO 资助等）获得更自由的长期支持。

AI 是“放大器”，而不是“替代品”

在人人都能用 AI 生成万字报告的时代，拼字数和拼拼凑凑的信息毫无意义。研究员的核心竞争力变成了“如何提出好问题”和“如何穿透 AI 的幻觉去验证底层逻辑”。优质的研究依然需要严密的方法论和对真实世界的敏锐度。

“Proof of Work (工作证明)”大于一切学历

Web3 是一个极度务实且结果导向的行业。无论是 Researcher 往Grokipedia贡献知识，还是 Developer 往 Moss 提交 PR，“你在链上和 GitHub 上留下了什么”就是你最好的简历。行动起来，哪怕只是去提一个Issue或修改一行文档，都比在求职信里写一万句“热爱 Web3”更有力量。

AI Agent 高危案例解析与安全构建

关于Agent安全的分享，干货蛮多。现在的AI已经不只是陪聊了，能自己调 API、查数据库甚至直接转账，权限越来越大，安全隐患也确实越来越多。

1\. 几个印象深刻的坑（高危案例）

过度授权：权限给高了真的会出大事，有SaaS平台直接被Agent在9秒内把生产数据删得干干净净。

防不胜防的提示词注入：黑客把恶意指令藏在摩斯密码或者网页代码里，就能忽悠Agent把币转走或者套出私钥。

生态与供应链投毒：某些热门插件近20%有高风险；代码依赖包如果不锁版本，分分钟中招泄露API密钥。

低级漏洞与幻觉：像 OpenCloud网关默认不设密码直接裸奔公网；再加上AI本身的“幻觉”，有时候查资料查着查着就突然发一堆乱码失控。

2\. 怎么构建安全防线？

别太相信AI（零开发信任 Zero Trust）。AI 终究是概率模型，不能只靠模型自律，因为模型会遗忘和产生幻觉，必须从架构底层做隔离。

人类把关（Human-in-the-Loop）。凡是涉及删库、转账、发邮件这种高危动作，必须人工按一次确认，绝不能放任它自动化。

安全防护三板斧：

最小权限：只给干活必需的权限，服务设高强度密码，依赖包锁死版本。

插件走官方：安装前自动扫描后门，只用白名单工具。

上防护网（Agent Guard）：运行时实时监控，一旦检测到偷数据或删库动作立刻拦截，并保持威胁情报库动态更新。

**Jack老师分享会：AI Agent 支付基础设施与Web3融合**

AI Agent 支付的本质： 解决机器在安全合规前提下代替人类完成资金转移的问题，依赖授权、凭证、支付编排与风控四大支撑。

三种落地路径：

用户确认式： Agent 负责比价选品，用户手动确认扣款（合规风险最低，如ChatGPT结账）。

委托令牌式： 用户预先发放受限令牌（限定额度与场景），Agent 限额内自主支付（如Visa虚拟卡）。

原生链上支付： Agent 独立持有钱包公私钥，通过链上协议实现毫秒级 M2M 结算（如 Coinbase Agentic Wallets）。

Agent Economy与Web3结合： 传统支付难以满足机器间的高频交互。结合区块链技术为 Agent 赋予数字身份（DID）与链上钱包，能让其自主完成模型调用、数据采购与算力租赁。

对于非技术小白来说：

理解运作机制，掌握 Agent 的身份凭证与支付逻辑，是未来开展Web3链上社交、数据交易等新型生态运营的基础。

结合实践探索。紧跟 Monad 等新兴生态动态，持续关注去中心化身份与支付协议在实际项目中的落地，思考其与社区建设、产品推广的结合点。
